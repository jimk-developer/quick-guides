A Beginner's Guide to JavaScript Variables

üéØ What are Variables?

Variables are containers that store data values. In JavaScript, you can declare variables using var, let, or const.

let name = "John"; const age = 30; var city = "New York";

‚úÖ Valid Variable Names

JavaScript variable names must follow specific rules:

Rules for Naming Variables:

  • Must start with: a letter (a-z, A-Z), underscore (_), or dollar sign ($)
  • Can contain: letters, digits (0-9), underscores, or dollar signs
  • Cannot start with: a number
  • Case-sensitive: myVar and myvar are different
  • Cannot use: reserved keywords (like class, function, return)
// ‚úÖ Valid variable names let userName = "Alice"; let user_name = "Bob"; let _privateVar = "secret"; let $jquery = "selector"; let user123 = "Charlie"; let MAX_SIZE = 100; // ‚ùå Invalid variable names // let 123user = "Invalid"; // Starts with number // let user-name = "Invalid"; // Contains hyphen // let user name = "Invalid"; // Contains space // let class = "Invalid"; // Reserved keyword
üí° Naming Convention Tips:
  • camelCase for variables and functions: myVariableName
  • PascalCase for classes: MyClassName
  • UPPER_SNAKE_CASE for constants: MAX_VALUE
  • Use descriptive names: userName instead of un

üîë var, let, and const

var

  • Function-scoped
  • Can be redeclared
  • Can be updated
  • Hoisted to top
  • Legacy (pre-ES6)

let

  • Block-scoped
  • Cannot be redeclared
  • Can be updated
  • Not initialized when hoisted
  • Modern (ES6+)

const

  • Block-scoped
  • Cannot be redeclared
  • Cannot be updated
  • Must be initialized
  • Modern (ES6+)

Examples of var, let, and const

// var - can be redeclared and updated var x = 10; var x = 20; // ‚úÖ OK x = 30; // ‚úÖ OK // let - can be updated but not redeclared let y = 10; // let y = 20; // ‚ùå Error: Identifier 'y' has already been declared y = 20; // ‚úÖ OK // const - cannot be updated or redeclared const z = 10; // const z = 20; // ‚ùå Error: Identifier 'z' has already been declared // z = 20; // ‚ùå Error: Assignment to constant variable // const with objects - properties can be modified const person = { name: "John" }; person.name = "Jane"; // ‚úÖ OK - modifying property // person = {}; // ‚ùå Error - reassigning the variable

üéØ Variable Scope

Scope determines where variables are accessible in your code.

1. Global Scope

Variables declared outside any function or block are globally scoped.

var globalVar = "I'm global"; let globalLet = "I'm also global"; function test() { console.log(globalVar); // ‚úÖ Accessible console.log(globalLet); // ‚úÖ Accessible }

2. Function Scope (var)

Variables declared with var are scoped to the function they're declared in.

function myFunction() { var functionScoped = "I'm function scoped"; console.log(functionScoped); // ‚úÖ Accessible } // console.log(functionScoped); // ‚ùå Error: not defined

3. Block Scope (let and const)

Variables declared with let and const are scoped to the block (curly braces) they're declared in.

if (true) { var varVariable = "var"; let letVariable = "let"; const constVariable = "const"; } console.log(varVariable); // ‚úÖ "var" - accessible (function scoped) // console.log(letVariable); // ‚ùå Error - not accessible // console.log(constVariable); // ‚ùå Error - not accessible

üí° Best Practices

  1. Use const by default - Use it for values that won't be reassigned
  2. Use let when you need to reassign - Perfect for loop counters or values that change
  3. Avoid var - It's outdated and can lead to unexpected behavior
  4. Declare variables at the top - Makes code more readable
  5. Use descriptive names - userName is better than u
  6. Initialize variables when declaring - Prevents undefined values
  7. One variable per line - Easier to read and maintain
  8. Keep scope as narrow as possible - Reduces bugs and memory usage
// ‚úÖ Good practices const MAX_USERS = 100; const apiUrl = "https://api.example.com"; let currentUser = null; let userCount = 0; for (let i = 0; i < 10; i++) { // let i is block-scoped to this loop console.log(i); } // ‚ùå Bad practices var x; // Avoid var let a, b, c; // Multiple declarations - harder to read let temp; // Uninitialized - can cause bugs

üîÑ Common Patterns

Destructuring Assignment

// Array destructuring const [first, second, third] = [1, 2, 3]; // Object destructuring const { name, age } = { name: "John", age: 30 }; // With default values const { city = "Unknown" } = {};

Swapping Variables

let a = 1; let b = 2; // ES6 way (using destructuring) [a, b] = [b, a]; console.log(a, b); // 2, 1

Multiple Variable Assignment

// Assign same value to multiple variables let x = y = z = 0; // Better approach (more explicit) let x = 0; let y = 0; let z = 0;
üéì Quick Summary:
  • Use const for values that won't change
  • Use let for values that will change
  • Avoid var in modern JavaScript
  • Variable names must start with letter, $, or _
  • let and const are block-scoped
  • var is function-scoped
  • All declarations are hoisted, but only var is initialized
