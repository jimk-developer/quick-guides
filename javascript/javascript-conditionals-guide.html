üéØ What are Conditional Statements?

Conditional statements allow your program to make decisions and execute different code based on whether certain conditions are true or false. Think of them as the "if this, then that" logic in programming.

üí° Why Use Conditionals?
  • Make decisions - Execute different code based on conditions
  • Control flow - Determine which parts of code run
  • Handle different scenarios - Respond to various user inputs or situations
  • Validate data - Check if inputs meet certain criteria

üìù The if Statement

The if statement executes a block of code only if a specified condition is true.

Syntax: if (condition) { // code to execute if condition is true }
// Simple if statement let temperature = 75; if (temperature > 70) { console.log("It's warm outside!"); } // Output: It's warm outside! // Another example let age = 20; if (age >= 18) { console.log("You are an adult"); } // Output: You are an adult // If condition is false, nothing happens let score = 45; if (score >= 60) { console.log("You passed!"); } // No output - condition is false

Conditions in if Statements

Conditions are expressions that evaluate to true or false. You can use comparison operators to create conditions.

// Using different comparison operators let x = 10; if (x === 10) { console.log("x is equal to 10"); } if (x !== 5) { console.log("x is not equal to 5"); } if (x > 5) { console.log("x is greater than 5"); } if (x <= 20) { console.log("x is less than or equal to 20"); }

üîÑ The if...else Statement

The if...else statement executes one block of code if the condition is true, and another block if it's false.

Syntax: if (condition) { // code if condition is true } else { // code if condition is false }
// if...else example let age = 15; if (age >= 18) { console.log("You can vote"); } else { console.log("You cannot vote yet"); } // Output: You cannot vote yet // Another example let temperature = 55; if (temperature > 70) { console.log("Wear shorts"); } else { console.log("Wear a jacket"); } // Output: Wear a jacket // With variables let score = 85; let result; if (score >= 60) { result = "Pass"; } else { result = "Fail"; } console.log(result); // Output: Pass

üîÄ The if...else if...else Statement

Use else if to test multiple conditions in sequence. The first condition that is true will execute, and the rest will be skipped.

Syntax: if (condition1) { // code if condition1 is true } else if (condition2) { // code if condition2 is true } else { // code if all conditions are false }
// Multiple conditions example let score = 85; if (score >= 90) { console.log("Grade: A"); } else if (score >= 80) { console.log("Grade: B"); } else if (score >= 70) { console.log("Grade: C"); } else if (score >= 60) { console.log("Grade: D"); } else { console.log("Grade: F"); } // Output: Grade: B // Temperature example let temp = 45; if (temp > 80) { console.log("It's hot!"); } else if (temp > 60) { console.log("It's nice"); } else if (temp > 40) { console.log("It's cool"); } else { console.log("It's cold!"); } // Output: It's cool
üí° Tip: Conditions are checked from top to bottom. Once a condition is true, that block executes and the rest are skipped!

üéõÔ∏è Logical Operators in Conditionals

You can combine multiple conditions using logical operators.

Operator Name Description Example
&& AND True if both conditions are true x > 5 && x < 10
|| OR True if at least one condition is true x < 5 || x > 10
! NOT Reverses the boolean value !(x === 5)

AND Operator (&&)

Both conditions must be true for the entire expression to be true.

// AND operator - both must be true let age = 25; let hasLicense = true; if (age >= 16 && hasLicense === true) { console.log("You can drive"); } // Output: You can drive // Range checking let temperature = 72; if (temperature >= 60 && temperature <= 80) { console.log("Perfect weather!"); } // Output: Perfect weather! // Multiple conditions let score = 85; let attendance = 95; if (score >= 80 && attendance >= 90) { console.log("Excellent student!"); } // Output: Excellent student!

OR Operator (||)

At least one condition must be true for the entire expression to be true.

// OR operator - at least one must be true let day = "Saturday"; if (day === "Saturday" || day === "Sunday") { console.log("It's the weekend!"); } // Output: It's the weekend! // Multiple options let grade = "A"; if (grade === "A" || grade === "B" || grade === "C") { console.log("Passing grade"); } // Output: Passing grade // Age groups let age = 70; if (age < 12 || age > 65) { console.log("Discount available"); } // Output: Discount available

NOT Operator (!)

Reverses a boolean value (true becomes false, false becomes true).

// NOT operator - reverses boolean let isRaining = false; if (!isRaining) { console.log("Go for a walk!"); } // Output: Go for a walk! // Double negative let isLoggedIn = true; if (!isLoggedIn) { console.log("Please log in"); } else { console.log("Welcome back!"); } // Output: Welcome back! // Combining with other operators let age = 25; if (!(age < 18)) { console.log("Adult"); } // Output: Adult

Combining Logical Operators

// Combining AND and OR let age = 20; let hasTicket = true; let hasInvitation = false; if (age >= 18 && (hasTicket || hasInvitation)) { console.log("You can enter"); } // Output: You can enter // Complex condition let temperature = 75; let isSunny = true; let isWeekend = true; if ((temperature > 70 && isSunny) || isWeekend) { console.log("Great day for outdoor activities!"); } // Output: Great day for outdoor activities!

üîÄ The switch Statement

The switch statement is used to perform different actions based on different values. It's useful when you have many possible values to check.

Syntax: switch (expression) { case value1: // code for value1 break; case value2: // code for value2 break; default: // code if no cases match }
// Basic switch statement let day = 3; let dayName; switch (day) { case 1: dayName = "Monday"; break; case 2: dayName = "Tuesday"; break; case 3: dayName = "Wednesday"; break; case 4: dayName = "Thursday"; break; case 5: dayName = "Friday"; break; case 6: dayName = "Saturday"; break; case 7: dayName = "Sunday"; break; default: dayName = "Invalid day"; } console.log(dayName); // Output: Wednesday
‚ö†Ô∏è Important: Always use break at the end of each case! Without it, execution will "fall through" to the next case.

switch with Strings

// Switch with string values let fruit = "apple"; switch (fruit) { case "banana": console.log("Bananas are $0.50"); break; case "apple": console.log("Apples are $0.75"); break; case "orange": console.log("Oranges are $0.60"); break; default: console.log("Sorry, we don't have that fruit"); } // Output: Apples are $0.75

Multiple Cases with Same Code

// Group multiple cases together let grade = "B"; switch (grade) { case "A": case "B": case "C": console.log("You passed!"); break; case "D": case "F": console.log("You failed"); break; default: console.log("Invalid grade"); } // Output: You passed! // Day type example let day = "Saturday"; switch (day) { case "Monday": case "Tuesday": case "Wednesday": case "Thursday": case "Friday": console.log("It's a weekday"); break; case "Saturday": case "Sunday": console.log("It's the weekend!"); break; default: console.log("Invalid day"); } // Output: It's the weekend!

üí° Best Practices

  1. Use === for comparisons - Avoid == to prevent type coercion issues
    // ‚ùå Bad - can cause unexpected results if (x == "5") { } // ‚úÖ Good - strict equality if (x === 5) { }
  2. Put most likely conditions first - For better performance
  3. Use switch for many specific values - More readable than many else if statements
  4. Always include else or default - Handle unexpected cases
    // ‚úÖ Good - handles all cases if (condition) { // do something } else { // handle other cases }
  5. Keep conditions simple - Break complex conditions into separate checks
  6. Use meaningful variable names in conditions - Makes code self-documenting

üö´ Common Mistakes

1. Forgetting Braces

// ‚ùå Dangerous - only first line is in if if (age >= 18) console.log("You can vote"); console.log("This always runs!"); // ‚úÖ Correct - use braces if (age >= 18) { console.log("You can vote"); console.log("Register today!"); }

2. Using = Instead of ===

// ‚ùå Wrong - assigns instead of compares if (x = 5) { console.log("This always runs!"); } // ‚úÖ Correct - compares values if (x === 5) { console.log("x is 5"); }

3. Forgetting break in switch

// ‚ùå Wrong - falls through to next case switch (day) { case 1: console.log("Monday"); // Missing break! case 2: console.log("Tuesday"); } // ‚úÖ Correct - includes break switch (day) { case 1: console.log("Monday"); break; case 2: console.log("Tuesday"); break; }

4. Incorrect Logical Operator

// ‚ùå Wrong - this is always false! if (x > 10 && x < 5) { console.log("Never runs"); } // ‚úÖ Correct - using OR if (x > 10 || x < 5) { console.log("Outside 5-10 range"); }

üìù Key Takeaways:
  • Conditionals let your code make decisions
  • Use if for simple true/false checks
  • Use else if to check multiple conditions
  • Use else to handle all other cases
  • Use && when all conditions must be true
  • Use || when at least one condition must be true
  • Use switch for many specific value checks
  • Always use === for comparisons (strict equality)
  • Don't forget break in switch statements!
