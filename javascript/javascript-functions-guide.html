A Beginner's Guide to JavaScript Functions

üéØ What are Functions?

Functions are reusable blocks of code that perform a specific task. Think of a function as a recipe: you define the steps once, and then you can use that recipe (call the function) as many times as you need.

üí° Why Use Functions?
  • Reusability - Write code once, use it many times
  • Organization - Break large programs into smaller, manageable pieces
  • Readability - Make code easier to understand
  • Maintenance - Update code in one place instead of many

üìù Function Declaration

The most common way to create a function is using a function declaration.

Syntax: function functionName(parameters) { // code to execute return value; }
// Simple function with no parameters function sayHello() { console.log("Hello, World!"); } // Calling (executing) the function sayHello(); // Output: Hello, World! // Function with parameters function greet(name) { console.log("Hello, " + name + "!"); } greet("Alice"); // Output: Hello, Alice! greet("Bob"); // Output: Hello, Bob! // Function with multiple parameters function add(a, b) { console.log(a + b); } add(5, 3); // Output: 8 add(10, 20); // Output: 30

Anatomy of a Function

function calculateArea(width, height) { let area = width * height; return area; } /* function - keyword to declare a function calculateArea - function name (what you call it) (width, height) - parameters (inputs the function receives) { ... } - function body (code that runs) return - sends a value back to the caller */

‚¨ÖÔ∏è Return Statement

The return statement sends a value back from the function. Once a function hits a return, it stops executing.

// Function that returns a value function multiply(x, y) { return x * y; } let result = multiply(4, 5); console.log(result); // Output: 20 // You can use the returned value directly console.log(multiply(3, 7)); // Output: 21 // Function with early return function isAdult(age) { if (age >= 18) { return true; } return false; } console.log(isAdult(25)); // Output: true console.log(isAdult(15)); // Output: false
‚ö†Ô∏è Important: If a function doesn't have a return statement, it automatically returns undefined.
// Function without return function printMessage(msg) { console.log(msg); // No return statement } let result = printMessage("Hello"); console.log(result); // Output: undefined

üì• Parameters vs Arguments

Understanding the difference between parameters and arguments:

function greetPerson(name, age) { // 'name' and 'age' are PARAMETERS console.log(`Hello, ${name}! You are ${age} years old.`); } greetPerson("Sarah", 25); // "Sarah" and 25 are ARGUMENTS
Term Definition Example
Parameters Variables listed in the function definition function test(x, y) - x and y are parameters
Arguments Actual values passed to the function when called test(5, 10) - 5 and 10 are arguments

Multiple Parameters

// Function with multiple parameters function createProfile(firstName, lastName, age, city) { return `${firstName} ${lastName}, ${age} years old, from ${city}`; } let profile = createProfile("John", "Doe", 30, "New York"); console.log(profile); // Output: John Doe, 30 years old, from New York

Optional Parameters

// If you don't pass all arguments, missing ones are undefined function greet(name, greeting) { console.log(greeting + ", " + name); } greet("Alice", "Hello"); // Output: Hello, Alice greet("Bob"); // Output: undefined, Bob // Better: Use default values or check for undefined function betterGreet(name, greeting) { if (greeting === undefined) { greeting = "Hello"; } console.log(greeting + ", " + name); } betterGreet("Charlie"); // Output: Hello, Charlie

üîÑ Function Expressions

Another way to create functions is by assigning them to variables. This is called a function expression.

// Function declaration (we learned this first) function add(a, b) { return a + b; } // Function expression (function as a value) const subtract = function(a, b) { return a - b; }; // Both can be called the same way console.log(add(10, 5)); // Output: 15 console.log(subtract(10, 5)); // Output: 5

Function Declaration vs Function Expression

Feature Function Declaration Function Expression
Syntax function name() {} const name = function() {};
Hoisting ‚úÖ Can be called before it's defined ‚ùå Cannot be called before it's defined
When to Use Most common, easier to read When you need to assign to variable
// Function Declaration - Hoisted (works!) sayHi(); // Output: Hi! function sayHi() { console.log("Hi!"); } // Function Expression - NOT Hoisted (error!) // sayBye(); // ‚ùå Error: Cannot access 'sayBye' before initialization const sayBye = function() { console.log("Bye!"); }; sayBye(); // ‚úÖ Works here: Output: Bye!

üéØ Scope in Functions

Variables declared inside a function are local to that function and cannot be accessed from outside.

// Global variable let globalVar = "I'm global"; function testScope() { // Local variable let localVar = "I'm local"; console.log(globalVar); // ‚úÖ Can access global variable console.log(localVar); // ‚úÖ Can access local variable } testScope(); console.log(globalVar); // ‚úÖ Can access global variable // console.log(localVar); // ‚ùå Error: localVar is not defined
‚ö†Ô∏è Important: Variables declared inside a function only exist inside that function. This is called function scope.

Parameters are Local Variables

function example(param) { console.log(param); // param is local to this function } example("Hello"); // Output: Hello // console.log(param); // ‚ùå Error: param is not defined

üìö Practical Examples

Example 1: Calculator

function divide(a, b) { if (b === 0) { return "Cannot divide by zero"; } return a / b; } console.log(divide(10, 2)); // 5 console.log(divide(10, 0)); // Cannot divide by zero

Example 2: Temperature Converter

function celsiusToFahrenheit(celsius) { let fahrenheit = (celsius * 9/5) + 32; return fahrenheit; } console.log(celsiusToFahrenheit(0)); // 32 console.log(celsiusToFahrenheit(100)); // 212

Example 3: Age Checker

function canVote(age) { if (age >= 18) { return "You can vote!"; } else { return "Too young to vote"; } } console.log(canVote(20)); // You can vote! console.log(canVote(16)); // Too young to vote

Example 4: String Manipulation

function makeUpperCase(text) { return text.toUpperCase(); } let result = makeUpperCase("hello"); console.log(result); // HELLO

Example 5: Find Maximum

function findMax(num1, num2) { if (num1 > num2) { return num1; } else { return num2; } } console.log(findMax(10, 5)); // 10 console.log(findMax(3, 15)); // 15

Example 6: Calculate Area

function calculateRectangleArea(length, width) { let area = length * width; return area; } let myArea = calculateRectangleArea(5, 10); console.log(myArea); // 50

üí° Best Practices

  1. Use descriptive names - Function names should describe what they do
    // ‚ùå Bad function x(a, b) { return a + b; } // ‚úÖ Good function addNumbers(a, b) { return a + b; }
  2. Keep functions small - Each function should do one thing well
  3. Use meaningful parameter names - Make it clear what the function expects
    // ‚ùå Bad function calc(x, y) { return x * y; } // ‚úÖ Good function calculateArea(width, height) { return width * height; }
  4. Always return a value - If your function calculates something, return it
  5. Document complex functions - Use comments to explain what the function does
  6. Test your functions - Try different inputs to make sure they work correctly

üö´ Common Mistakes

1. Forgetting to Call the Function

function greet() { console.log("Hello!"); } // ‚ùå Wrong - just references the function, doesn't call it greet; // ‚úÖ Correct - calls the function greet();

2. Forgetting Parentheses on Parameters

// ‚ùå Wrong - missing parentheses function greet name { console.log("Hello, " + name); } // ‚úÖ Correct - has parentheses function greet(name) { console.log("Hello, " + name); }

3. Using Return Value Without Storing or Using It

function add(a, b) { return a + b; } // ‚ùå Result is lost - not stored or used add(5, 3); // ‚úÖ Store it in a variable let sum = add(5, 3); // ‚úÖ Or use it directly console.log(add(5, 3));

4. Trying to Access Local Variables Outside the Function

function calculate() { let result = 10 * 5; return result; } calculate(); // console.log(result); // ‚ùå Error: result is not defined // ‚úÖ Correct: store the returned value let myResult = calculate(); console.log(myResult); // 50

Function Cheat Sheet

// Basic function declaration function functionName(parameter1, parameter2) { // code to execute return value; } // Calling a function functionName(argument1, argument2); // Function expression const myFunction = function(parameter) { // code to execute return value; }; // Function with no parameters function doSomething() { // code here } // Function with no return (returns undefined) function logMessage(msg) { console.log(msg); }
üìù Key Takeaways:
  • Functions are reusable blocks of code
  • Use function keyword to declare a function
  • Parameters are placeholders; arguments are actual values
  • Use return to send a value back from the function
  • Variables inside functions are local (function scope)
  • Always call functions with parentheses: functionName()
  • Function declarations are hoisted; function expressions are not
  • Give functions descriptive names that explain what they do
