A Beginner's Guide to the Document Object Model

üéØ What is the DOM?

The Document Object Model (DOM) is a programming interface that represents an HTML document as a tree structure of objects. It allows JavaScript to access and manipulate the content, structure, and style of web pages dynamically.

üí° Why the DOM Matters:
  • Dynamic content - Change page content without reloading
  • Interactive apps - Respond to user actions in real-time
  • Create elements - Build page content with JavaScript
  • Manipulate styles - Change appearance dynamically

üèóÔ∏è Building a Web App

A web app consists of four main components working together:

1. HTML Code

Purpose: The app's container and structure

Defines the basic elements and layout of your application.

2. CSS Code

Purpose: The app's appearance

Controls colors, fonts, spacing, and visual design.

3. JavaScript Code

Purpose: The app's operation

Manages logic, interactivity, and dynamic content.

4. Data

Purpose: The app's content

Often comes from external files, databases, or APIs.

üîç Selecting Nodes

Before you can manipulate nodes, you need to select them. JavaScript provides several methods for this.

Basic Selection Methods

// Select by ID let element = document.getElementById("myId"); // Select by class name (returns HTMLCollection) let elements = document.getElementsByClassName("myClass"); // Select by tag name (returns HTMLCollection) let paragraphs = document.getElementsByTagName("p"); // Select first match with CSS selector let firstDiv = document.querySelector("div.container"); // Select all matches with CSS selector (returns NodeList) let allDivs = document.querySelectorAll("div.container");
üí° querySelector vs querySelectorAll:
  • querySelector() - Returns the first matching element
  • querySelectorAll() - Returns all matching elements as a NodeList
  • Both use CSS selector syntax (IDs, classes, tags, etc.)

Node Properties for Navigation

// Accessing related nodes let parent = element.parentNode; let children = element.children; // Only element children let allChildren = element.childNodes; // All nodes including text let firstChild = element.firstElementChild; let lastChild = element.lastElementChild; let nextSibling = element.nextElementSibling; let prevSibling = element.previousElementSibling;

‚ûï Creating and Adding Nodes

You can create new HTML elements dynamically and add them to the page.

Creating Elements

// Create a new element let newDiv = document.createElement("div"); let newParagraph = document.createElement("p"); let newImage = document.createElement("img"); // Create a text node let textNode = document.createTextNode("Hello World"); // Create a document fragment (temporary container) let fragment = document.createDocumentFragment();

Adding Content to Elements

// Set text content newParagraph.textContent = "This is a new paragraph"; // Set HTML content (including tags) newDiv.innerHTML = "<strong>Bold text</strong>"; // Append text node newParagraph.appendChild(textNode);

Appending Elements to the Page

// Get parent element let container = document.getElementById("container"); // Append child to end container.appendChild(newParagraph); // Insert before a specific child let firstChild = container.firstElementChild; container.insertBefore(newDiv, firstChild); // Complete example: build and append let list = document.createElement("ul"); let item1 = document.createElement("li"); let item2 = document.createElement("li"); item1.textContent = "First item"; item2.textContent = "Second item"; list.appendChild(item1); list.appendChild(item2); document.body.appendChild(list);

Using Document Fragments

A document fragment is a lightweight container for building complex structures before adding them to the DOM. This is more efficient than adding elements one at a time.

// Create fragment let fragment = document.createDocumentFragment(); // Build structure in fragment for (let i = 0; i < 5; i++) { let item = document.createElement("li"); item.textContent = "Item " + (i + 1); fragment.appendChild(item); } // Add entire fragment to DOM at once let list = document.getElementById("myList"); list.appendChild(fragment);
üí° Why Use Fragments? Adding elements one at a time causes the browser to repaint the page multiple times. Using a fragment builds the structure in memory first, then adds everything at once, improving performance.

üé® Working with Attributes and Styles

Setting Attributes

// Direct property assignment newImage.src = "photo.jpg"; newImage.alt = "A photo"; newDiv.id = "myDiv"; // Using setAttribute method newImage.setAttribute("width", "300"); newImage.setAttribute("height", "200"); // Working with classes newDiv.className = "container highlight"; // Replace all classes // Better: use classList for adding/removing classes newDiv.classList.add("active"); newDiv.classList.remove("highlight"); newDiv.classList.toggle("visible"); // Add if absent, remove if present console.log(newDiv.classList.contains("active")); // Check if class exists

Inline Styles

// Setting individual style properties newDiv.style.color = "blue"; newDiv.style.fontSize = "20px"; newDiv.style.backgroundColor = "yellow"; newDiv.style.padding = "10px"; newDiv.style.border = "1px solid black"; // Note: CSS properties with hyphens become camelCase // CSS: background-color ‚Üí JavaScript: backgroundColor // CSS: font-size ‚Üí JavaScript: fontSize
‚ö†Ô∏è Best Practice: Use CSS classes instead of inline styles when possible. Inline styles are harder to maintain and override CSS specificity rules.

üîÑ Restructuring the Node Tree

Moving Nodes with appendChild()

When you use appendChild() on an existing node, it moves that node to a new location.

// HTML: <div id="container1"><p id="para">Text</p></div> // <div id="container2"></div> let para = document.getElementById("para"); let container2 = document.getElementById("container2"); // This MOVES para from container1 to container2 container2.appendChild(para); // Real example: Moving images in a slideshow function slideNext() { let slideshow = document.getElementById("slideshow"); let firstImage = slideshow.firstElementChild; // Move first image to end (creates scrolling effect) slideshow.appendChild(firstImage); }

Inserting Nodes with insertBefore()

Use insertBefore() to insert a node before a specific child.

Syntax: parentNode.insertBefore(newNode, referenceNode);
// Insert at the beginning let container = document.getElementById("container"); let newElement = document.createElement("div"); let firstChild = container.firstElementChild; container.insertBefore(newElement, firstChild); // Move existing element to beginning function slidePrevious() { let slideshow = document.getElementById("slideshow"); let lastImage = slideshow.lastElementChild; let firstImage = slideshow.firstElementChild; // Move last image to beginning slideshow.insertBefore(lastImage, firstImage); }

Cloning Nodes

Create a copy of a node using cloneNode().

// Shallow copy (element only, no children) let original = document.getElementById("original"); let shallowCopy = original.cloneNode(false); // Deep copy (element and all descendants) let deepCopy = original.cloneNode(true); // Example: Clone a card element let card = document.querySelector(".card"); let newCard = card.cloneNode(true); newCard.querySelector(".title").textContent = "New Title"; document.body.appendChild(newCard);

Removing Nodes

// Remove a child from parent let parent = document.getElementById("parent"); let child = document.getElementById("child"); parent.removeChild(child); // Modern method: remove element directly child.remove(); // Remove all children while (parent.firstChild) { parent.removeChild(parent.firstChild); } // Or simply: parent.innerHTML = "";

‚è∞ Timed Commands

Execute code after a delay or at regular intervals.

setTimeout() - Run Once After Delay

// Basic syntax setTimeout(function, delay); // Run after 3 seconds setTimeout(function() { console.log("3 seconds passed"); }, 3000); // With a named function function showMessage() { alert("Time's up!"); } setTimeout(showMessage, 5000); // Cancel a timeout let timeoutId = setTimeout(showMessage, 5000); clearTimeout(timeoutId); // Cancels the timeout

setInterval() - Repeat at Intervals

// Basic syntax setInterval(function, interval); // Run every 2 seconds let counter = 0; let intervalId = setInterval(function() { counter++; console.log("Count: " + counter); }, 2000); // Stop the interval clearInterval(intervalId); // Example: Simple countdown timer let seconds = 10; let timer = setInterval(function() { seconds--; console.log(seconds); if (seconds <= 0) { clearInterval(timer); console.log("Done!"); } }, 1000);

Practical Timer Example

// Auto-advance slideshow let isPlaying = false; let slideshowTimer; function startSlideshow() { if (!isPlaying) { isPlaying = true; slideshowTimer = setInterval(slideNext, 3000); } } function stopSlideshow() { if (isPlaying) { isPlaying = false; clearInterval(slideshowTimer); } }

üí¨ Dialog Boxes

JavaScript provides three types of dialog boxes for user interaction.

1. Alert - Display Message

// Simple alert alert("Welcome to our site!"); // Alert with variable let username = "John"; alert("Hello, " + username + "!");

2. Confirm - Get Yes/No

// Returns true if OK, false if Cancel let result = confirm("Are you sure you want to delete?"); if (result) { console.log("Deleted"); } else { console.log("Cancelled"); } // Practical example function deleteItem() { if (confirm("Delete this item?")) { // Delete code here console.log("Item deleted"); } }

3. Prompt - Get User Input

// Returns string or null if cancelled let name = prompt("What is your name?"); if (name !== null) { console.log("Hello, " + name); } // With default value let age = prompt("How old are you?", "18"); // Validate input let quantity = prompt("How many?"); quantity = parseInt(quantity); if (isNaN(quantity)) { alert("Please enter a number"); }
‚ö†Ô∏è Dialog Box Limitations:
  • Block all page interaction until closed
  • Can't be styled with CSS
  • Can look different in different browsers
  • Consider using custom modals for better UX

ü™ü Creating Overlays

An overlay is an element that appears on top of page content, creating a modal-like experience.

Basic Overlay Example

// HTML structure // <div id="overlay" class="overlay"> // <div class="overlay-content"> // <span class="close">×</span> // <img id="overlay-image"> // </div> // </div> // CSS (add to stylesheet) /* .overlay { display: none; position: fixed; top: 0; left: 0; width: 100%; height: 100%; background: rgba(0, 0, 0, 0.8); z-index: 1000; } .overlay-content { position: relative; margin: 50px auto; max-width: 80%; } .close { position: absolute; top: 10px; right: 25px; color: white; font-size: 35px; cursor: pointer; } */ // JavaScript to show/hide overlay function showOverlay(imageSrc) { let overlay = document.getElementById("overlay"); let overlayImage = document.getElementById("overlay-image"); overlayImage.src = imageSrc; overlay.style.display = "block"; } function hideOverlay() { let overlay = document.getElementById("overlay"); overlay.style.display = "none"; } // Event listeners document.querySelector(".close").onclick = hideOverlay; document.getElementById("overlay").onclick = hideOverlay;

The this Object

The this object references the owner of the currently running code.

// In an event handler, this refers to the element let images = document.querySelectorAll("img"); images.forEach(function(img) { img.onclick = function() { // this refers to the clicked image showOverlay(this.src); }; }); // Example: Highlight clicked button let buttons = document.querySelectorAll("button"); buttons.forEach(function(btn) { btn.onclick = function() { // Remove active class from all buttons.forEach(b => b.classList.remove("active")); // Add to clicked button this.classList.add("active"); }; });

üåê Browser Object Model (BOM)

The Browser Object Model provides objects for interacting with the browser itself, not just the document.

The window Object

The global object that represents the browser window. All other BOM objects are properties of window.

// Window properties console.log(window.innerWidth); // Browser width console.log(window.innerHeight); // Browser height // Open new window let popup = window.open("page.html", "_blank", "width=500,height=400"); // Close window popup.close();

The history Object

Stores the browser's navigation history for the current session.

// Navigate history window.history.back(); // Go back one page window.history.forward(); // Go forward one page window.history.go(-2); // Go back 2 pages window.history.go(1); // Go forward 1 page // Check history length console.log(history.length); // Number of pages in history

The location Object

Contains information about the current URL.

// Current URL properties console.log(location.href); // Full URL console.log(location.hostname); // Domain name console.log(location.pathname); // Path console.log(location.protocol); // http: or https: // Navigate to new URL location.href = "https://example.com"; // Reload page location.reload();

The navigator Object

Provides information about the browser.

// Browser information console.log(navigator.userAgent); // Browser ID string console.log(navigator.language); // Browser language console.log(navigator.onLine); // Online status console.log(navigator.platform); // Operating system // Check browser features if (navigator.geolocation) { console.log("Geolocation supported"); }

The screen Object

Contains information about the user's screen.

// Screen properties console.log(screen.width); // Total screen width console.log(screen.height); // Total screen height console.log(screen.availWidth); // Available width console.log(screen.availHeight); // Available height // Example: Center a popup window function openCenteredWindow(url, width, height) { let left = (screen.width - width) / 2; let top = (screen.height - height) / 2; let options = `width=${width},height=${height},left=${left},top=${top}`; window.open(url, "_blank", options); }

üí° Best Practices

  1. Use document fragments for multiple elements
    // ‚úÖ Efficient - one DOM update let fragment = document.createDocumentFragment(); for (let i = 0; i < 100; i++) { let item = document.createElement("div"); fragment.appendChild(item); } container.appendChild(fragment);
  2. Cache DOM references
    // ‚ùå Bad - searches DOM repeatedly for (let i = 0; i < 10; i++) { document.getElementById("list").innerHTML += "<li>Item</li>"; } // ‚úÖ Good - cache reference let list = document.getElementById("list"); for (let i = 0; i < 10; i++) { let item = document.createElement("li"); list.appendChild(item); }
  3. Use classList instead of className
  4. Always clear timers when done
  5. Use textContent for plain text (safer than innerHTML)
  6. Consider accessibility when creating dynamic content

üìù Key Takeaways:
  • The DOM represents HTML as a tree of nodes
  • Use querySelector() and querySelectorAll() for flexible selection
  • Create elements with createElement() and add with appendChild()
  • Use document fragments for efficient bulk operations
  • appendChild() on existing nodes moves them
  • Use classList for managing CSS classes
  • setInterval() repeats, setTimeout() runs once
  • Dialog boxes block page interaction
  • Overlays provide better UX than popups
  • The BOM provides access to browser features
