A Beginner's Guide to Finding and Fixing Errors

üéØ What is Debugging?

Debugging is the process of finding and fixing errors (bugs) in your code. A bug is any error that causes your program to function incorrectly. Learning to debug effectively is one of the most important skills for a programmer!

üí° Why Debugging Matters:
  • Saves time - Find and fix problems quickly
  • Improves code quality - Learn from mistakes
  • Builds confidence - Understand how your code works
  • Professional skill - Essential for all programmers

üö® Types of Errors

JavaScript programs can encounter four main types of errors. Understanding each type helps you know where to look and how to fix them.

1. Load-Time Errors (Syntax Errors)

Load-Time Error

What it is: Errors that occur when the browser first loads your code and finds mistakes in the syntax (grammar rules of JavaScript).

When it happens: Before any code runs

Common causes:

  • Misspelled keywords
  • Missing semicolons
  • Missing or mismatched brackets, parentheses, or braces
  • Incorrect quotation marks
// ‚ùå Syntax Errors // Missing closing parenthesis console.log("Hello"; // Misspelled keyword funtion greet() { // should be "function" console.log("Hi"); } // Missing closing brace if (x > 5) { console.log("Greater"); // Missing } // Mismatched quotes let message = "Hello'; // starts with " ends with '

2. Runtime Errors

Runtime Error

What it is: Errors that occur when the code starts running but can't complete an operation.

When it happens: During program execution

Common causes:

  • Calling a function that doesn't exist
  • Using an undefined variable
  • Dividing by zero
  • Accessing properties of null or undefined
// ‚ùå Runtime Errors // Calling undefined function calculateTotal(); // Error: calculateTotal is not defined // Using undefined variable console.log(userName); // Error: userName is not defined // Accessing property of undefined let user; console.log(user.name); // Error: Cannot read property 'name' of undefined // Invalid math operation let result = "text" / 5; // Result: NaN (Not a Number)

3. Logic Errors

Logic Error

What it is: Errors in your program's design that cause incorrect results, even though the code runs without crashing.

When it happens: Code runs but produces wrong results

Common causes:

  • Using the wrong operator
  • Incorrect formula or calculation
  • Wrong condition in if statement
  • Off-by-one errors in loops
// ‚ùå Logic Errors // Wrong operator - should be + not * let total = price * tax; // Should be: price + tax // Wrong condition if (age > 18) { // Should be >= 18 console.log("Can vote"); } // Wrong calculation let average = (a + b + c) / 2; // Should divide by 3 // Off-by-one error for (let i = 1; i <= numbers.length; i++) { console.log(numbers[i]); // Last iteration: undefined }

4. User Errors

User Error

What it is: Errors that occur when users provide invalid input or use your program incorrectly.

When it happens: During user interaction

Common causes:

  • Entering text instead of numbers
  • Leaving required fields empty
  • Entering negative numbers where positive expected
  • Entering values outside valid range

üîç Using the Browser Console

The browser console is your first and most important debugging tool. It shows error messages and allows you to interact with your code.

Opening the Console

How to open the console in different browsers:
  • Chrome: Press F12 or Ctrl+Shift+J (Windows) / Cmd+Option+J (Mac)
  • Firefox: Press F12 or Ctrl+Shift+K (Windows) / Cmd+Option+K (Mac)
  • Safari: Enable Developer menu first, then Cmd+Option+C
  • Edge: Press F12 or Ctrl+Shift+I

Reading Error Messages

Error messages provide valuable information about what went wrong and where to look.

// Example error message: // Uncaught ReferenceError: calculateTotal is not defined // at script.js:15 // Breaking down the message: // 1. "Uncaught ReferenceError" - Type of error // 2. "calculateTotal is not defined" - What's wrong // 3. "at script.js:15" - File name and line number
üí° Reading Error Messages:
  1. Look at the error type (ReferenceError, SyntaxError, etc.)
  2. Read the error description carefully
  3. Note the file name and line number
  4. Click the line number to jump to the problematic code

üîí Strict Mode

JavaScript's strict mode makes the interpreter less forgiving and helps catch common mistakes by treating more things as errors.

// Enabling strict mode "use strict"; // Now these cause errors (without strict mode, they might not): // Using undeclared variable x = 10; // Error: x is not defined // Deleting variables let y = 5; delete y; // Error: cannot delete variable // Duplicate parameters function add(a, a) { // Error: duplicate parameter return a + a; }
üí° Best Practice: Always use "use strict"; at the beginning of your JavaScript files. It helps catch errors early and encourages better coding practices.

üîé Tracing Errors

Tracing means examining how your code executes step-by-step to find where things go wrong.

Method 1: window.alert()

The window.alert() method pauses execution and displays a message. Useful for quick checks.

// Using alert to trace values function calculateTotal(price, quantity) { alert("Price: " + price); // Check price value alert("Quantity: " + quantity); // Check quantity value let total = price * quantity; alert("Total: " + total); // Check result return total; } let result = calculateTotal(10, 5);
‚ö†Ô∏è Limitations of alert():
  • Interrupts program flow
  • User must click OK for each alert
  • Can't see multiple values at once
  • Annoying for users if left in production code

Method 2: console.log()

The console.log() method writes messages to the console. This is the preferred method for debugging.

// Using console.log to trace values function calculateTotal(price, quantity) { console.log("Price:", price); console.log("Quantity:", quantity); let total = price * quantity; console.log("Total:", total); return total; } // Log multiple values at once console.log("Values:", price, quantity, total); // Log with labels console.log("Starting calculation..."); console.log("Result:", result); console.log("Calculation complete");
üí° Advantages of console.log():
  • Doesn't interrupt program flow
  • Can view all output at once
  • Can log multiple values in one line
  • Can see the order of execution
  • Easy to remove when done debugging

Method 3: Commenting Out Code

Commenting out means temporarily disabling lines of code to isolate problems.

// Original function with error function processData() { getData(); validateData(); calculateResults(); displayResults(); } // Comment out sections to find the problem function processData() { getData(); // validateData(); // Commenting this out // calculateResults(); // displayResults(); } // If error goes away, problem is in commented code // Uncomment one function at a time to find exact issue

üõ†Ô∏è Browser Debugging Tools

Modern browsers include powerful debugging tools that let you pause code, inspect variables, and step through execution.

Setting Breakpoints

A breakpoint pauses code execution at a specific line, letting you examine the program's state.

How to set breakpoints:
  1. Open the browser's Developer Tools (F12)
  2. Go to the "Sources" or "Debugger" tab
  3. Find your JavaScript file
  4. Click the line number where you want to pause
  5. Refresh the page - code will pause at that line

Stepping Through Code

Once paused at a breakpoint, you can control execution with stepping options:

Button Name What It Does
‚ñ∂Ô∏è Resume Continue running until next breakpoint
‚Ü∑ Step Over Execute current line, skip over function calls
‚Üì Step Into Go inside function calls to see their code
‚Üë Step Out Finish current function and return to caller

Watching Variables

The debugger lets you monitor variable values as code executes:

Scope Window: Shows all variables currently in scope (local and global) and their values.

Watch Window: Lets you add specific variables or expressions to monitor continuously.

Call Stack

The call stack shows which functions are currently running, in order from most recent to first called.

// Example call stack for nested functions function first() { console.log("In first"); second(); } function second() { console.log("In second"); third(); } function third() { console.log("In third"); // If you pause here, call stack shows: // third() // second() // first() } first();

‚ö†Ô∏è Error Handling

Error handling means writing code that gracefully handles errors instead of crashing. This is called bulletproofing your code.

The try...catch Statement

The try...catch statement lets you test code for errors and handle them gracefully.

Syntax: try { // Code that might cause an error } catch (error) { // Code to handle the error }
// Basic try...catch example try { let result = riskyOperation(); console.log("Success!", result); } catch (error) { console.log("An error occurred:", error.message); } // Real example: parsing user input try { let age = parseInt(userInput); if (isNaN(age)) { throw "Invalid age entered"; } console.log("Age is:", age); } catch (error) { console.log("Error:", error); alert("Please enter a valid number"); } // Example with function that might fail function divide(a, b) { try { if (b === 0) { throw "Cannot divide by zero"; } return a / b; } catch (error) { console.log("Division error:", error); return null; } }

The throw Statement

Use throw to create your own custom errors (called exceptions).

// Throwing custom errors function validateAge(age) { if (age < 0) { throw "Age cannot be negative"; } if (age > 150) { throw "Age seems unrealistic"; } return true; } // Using the validation function try { let userAge = -5; validateAge(userAge); console.log("Age is valid"); } catch (error) { console.log("Validation error:", error); }

The try...catch...finally Statement

The finally block runs whether an error occurs or not. Useful for cleanup tasks.

Syntax: try { // Code that might cause an error } catch (error) { // Code to handle the error } finally { // Code that always runs }
// try...catch...finally example function processFile(filename) { try { console.log("Opening file..."); // Code to read file if (!filename) { throw "No filename provided"; } console.log("Processing:", filename); } catch (error) { console.log("Error:", error); } finally { console.log("Closing file..."); // This always runs, error or not } } processFile("data.txt"); // Success path processFile(""); // Error path // In both cases, "Closing file..." is logged

üìö Practical Examples

Example 1: Input Validation

function validateInput(value) { try { if (value === "") { throw "Input is empty"; } let num = parseFloat(value); if (isNaN(num)) { throw "Not a number"; } return num; } catch (error) { alert(error); return null; } }

Example 2: Safe Division

function safeDivide(a, b) { try { if (b === 0) { throw "Division by zero"; } return a / b; } catch (error) { console.log("Error:", error); return 0; } } console.log(safeDivide(10, 2)); // 5 console.log(safeDivide(10, 0)); // 0

Example 3: Form Validation

function validateForm(email, age) { try { if (!email.includes("@")) { throw "Invalid email"; } if (age < 18) { throw "Must be 18+"; } return true; } catch (error) { alert(error); return false; } }

Example 4: Array Access

function getElement(arr, index) { try { if (index < 0 || index >= arr.length) { throw "Index out of bounds"; } return arr[index]; } catch (error) { console.log("Error:", error); return undefined; } }

Example 5: File Processing

function processData(data) { try { console.log("Starting..."); if (!data) { throw "No data"; } // Process data console.log("Success"); } catch (error) { console.log("Failed:", error); } finally { console.log("Cleanup"); } }

Example 6: Multiple Checks

function validatePassword(pwd) { try { if (pwd.length < 8) { throw "Too short"; } if (!/[A-Z]/.test(pwd)) { throw "Need uppercase"; } if (!/[0-9]/.test(pwd)) { throw "Need number"; } return true; } catch (error) { alert(error); return false; } }

üí° Debugging Best Practices

  1. Read error messages carefully - They tell you what's wrong and where
    // Error: "Uncaught ReferenceError: myFunction is not defined" // This tells you: you're calling a function that doesn't exist
  2. Use strict mode - Catches more errors early
    "use strict"; // Add at the top of your files
  3. Start simple - Test small pieces before combining
    // Test functions individually first console.log(calculateTax(100)); // Test this alone console.log(calculateTotal(100)); // Then test this
  4. Use console.log() liberally - See what's happening
    console.log("Before calculation"); console.log("Value:", x); console.log("After calculation");
  5. Check your assumptions - Verify variable values
    // Don't assume - check! console.log("typeof price:", typeof price); console.log("price value:", price);
  6. Use breakpoints for complex issues - Step through execution
  7. Take breaks - Fresh eyes often spot bugs faster
  8. Search error messages - Someone else has had the same problem
  9. Comment your fixes - Explain what was wrong
    // Fixed: Was using = instead of === if (x === 5) {
  10. Test edge cases - Try unusual inputs
    // Test with 0, negative, very large numbers, empty strings calculateTotal(0); calculateTotal(-5); calculateTotal(999999);

üö´ Common Mistakes to Avoid

1. Forgetting "use strict"

// ‚ùå Bad - no strict mode x = 10; // Creates global variable accidentally // ‚úÖ Good - use strict mode "use strict"; x = 10; // Error: x is not defined

2. Ignoring Error Messages

// ‚ùå Don't ignore errors in console // Read them! They tell you what's wrong // ‚úÖ Read error messages carefully // They point you to the exact line and problem

3. Not Checking for null/undefined

// ‚ùå Bad - doesn't check let name = user.name; // Error if user is undefined // ‚úÖ Good - checks first let name = user ? user.name : "Guest";

4. Catching Errors and Doing Nothing

// ‚ùå Bad - silent failure try { riskyOperation(); } catch (error) { // Empty - error is hidden! } // ‚úÖ Good - log or handle the error try { riskyOperation(); } catch (error) { console.log("Error occurred:", error); }

5. Not Validating User Input

// ‚ùå Bad - trusts user input let age = userInput; if (age >= 18) { } // ‚úÖ Good - validates first let age = parseInt(userInput); if (isNaN(age)) { alert("Please enter a valid number"); } else if (age >= 18) { }
