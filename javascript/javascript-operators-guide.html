A Beginner's Guide to JavaScript Operators

üéØ What are Operators?

Operators are symbols that perform operations on operands (values and variables). JavaScript has a rich set of operators that allow you to perform arithmetic calculations, make comparisons, assign values, and perform logical operations.

// Examples of different operators let sum = 5 + 3; // Arithmetic operator let isEqual = sum === 8; // Comparison operator let result = isEqual && true; // Logical operator sum += 2; // Assignment operator

üìö Operator Categories

  • Arithmetic Operators - Mathematical calculations (+, -, *, /, %)
  • Assignment Operators - Assign values (=, +=, -=, *=, /=)
  • Comparison Operators - Compare values (==, ===, !=, !==, >, <)< /li>
  • Logical Operators - Boolean logic (&&, ||, !)
  • Bitwise Operators - Bit-level operations (&, |, ^, ~)
  • String Operators - String concatenation (+)
  • Conditional (Ternary) Operator - Shorthand if-else (?:)
  • Type Operators - Check types (typeof, instanceof)

// Arithmetic operators examples let a = 10; let b = 3; console.log(a + b); // 13 - Addition console.log(a - b); // 7 - Subtraction console.log(a * b); // 30 - Multiplication console.log(a / b); // 3.333... - Division console.log(a % b); // 1 - Modulus (remainder) console.log(a ** b); // 1000 - Exponentiation (10^3) // Increment and Decrement let x = 5; console.log(x++); // 5 (returns then increments) console.log(x); // 6 console.log(++x); // 7 (increments then returns) let y = 5; console.log(y--); // 5 (returns then decrements) console.log(y); // 4 console.log(--y); // 3 (decrements then returns)
‚ö†Ô∏è Prefix vs Postfix:
  • x++ (postfix) - Returns current value, then increments
  • ++x (prefix) - Increments first, then returns new value

‚öñÔ∏è Comparison Operators

Used to compare two values and return a boolean result.

Operator Name Example Result Description

=== Strict equal to 5 === "5" false Checks value and type equality
!= Not equal to 5 != "5" false Checks value inequality (with type coercion)
!== Strict not equal to 5 !== "5" true Checks value and type inequality

// Comparison operators examples console.log(5 == "5"); // true - loose equality (type coercion) console.log(5 === "5"); // false - strict equality (no coercion) console.log(5 != "5"); // false - loose inequality console.log(5 !== "5"); // true - strict inequality console.log(10 > 5); // true - greater than console.log(10 < 5); // false - less than console.log(10 >= 10); // true - greater than or equal console.log(10 <= 5); // false - less than or equal // Special cases console.log(null == undefined); // true (special case) console.log(null === undefined); // false (different types) console.log(NaN === NaN); // false (NaN never equals itself!)
‚úÖ Best Practice: Always use strict equality (===) and strict inequality (!==) to avoid unexpected type coercion behavior.

üîÄ Logical Operators

Used to combine or invert boolean values.

Operator Name Example Description
&& Logical AND x && y Returns true if both operands are true
|| Logical OR x || y Returns true if at least one operand is true
! Logical NOT !x Inverts the boolean value
// Logical AND (&&) - returns first falsy or last value console.log(true && true); // true console.log(true && false); // false console.log(false && true); // false console.log(5 && 10); // 10 (both truthy, returns last) console.log(0 && 10); // 0 (first is falsy, returns it) // Logical OR (||) - returns first truthy or last value console.log(true || false); // true console.log(false || true); // true console.log(false || false); // false console.log(0 || 10); // 10 (first is falsy, returns second) console.log(5 || 10); // 5 (first is truthy, returns it) // Logical NOT (!) - inverts boolean console.log(!true); // false console.log(!false); // true console.log(!0); // true (0 is falsy) console.log(!""); // true (empty string is falsy) console.log(!!"hello"); // true (double negation converts to boolean)

Short-Circuit Evaluation

// AND (&&) short-circuits if first operand is falsy let user = null; let name = user && user.name; // null (doesn't evaluate user.name) // OR (||) short-circuits if first operand is truthy let defaultName = ""; let displayName = defaultName || "Guest"; // "Guest" // Practical examples const isLoggedIn = true; const isAdmin = false; isLoggedIn && console.log("Welcome!"); // Executes if isLoggedIn is true isAdmin || console.log("Access denied"); // Executes if isAdmin is false

‚ùì Ternary (Conditional) Operator

A shorthand for if-else statements. The only JavaScript operator that takes three operands.

Syntax: condition ? expressionIfTrue : expressionIfFalse
// Basic ternary operator let age = 18; let status = age >= 18 ? "Adult" : "Minor"; console.log(status); // "Adult" // Instead of if-else let message; if (age >= 18) { message = "Can vote"; } else { message = "Cannot vote"; } // Ternary equivalent (more concise) let voteStatus = age >= 18 ? "Can vote" : "Cannot vote"; // Nested ternary (use sparingly!) let score = 85; let grade = score >= 90 ? "A" : score >= 80 ? "B" : score >= 70 ? "C" : score >= 60 ? "D" : "F"; // Ternary in JSX/React (common use case) // return isLoggedIn ? <Dashboard /> : <Login />;
‚ö†Ô∏è Readability Warning: While ternary operators are concise, nested ternaries can become hard to read. Use them for simple conditions only.

üî£ String Operators

The + operator is used for string concatenation.

// String concatenation with + let firstName = "John"; let lastName = "Doe"; let fullName = firstName + " " + lastName; // "John Doe" // String + Number = String (coercion) let result1 = "Score: " + 100; // "Score: 100" let result2 = "5" + 3; // "53" (not 8!) // += with strings let message = "Hello"; message += " "; message += "World"; console.log(message); // "Hello World" // Template literals (better alternative) let name = "Alice"; let age = 25; let greeting = `Hello, ${name}! You are ${age} years old.`; console.log(greeting); // "Hello, Alice! You are 25 years old."

üîç Type Operators

typeof Operator

Returns a string indicating the type of a variable.

// typeof examples console.log(typeof "hello"); // "string" console.log(typeof 42); // "number" console.log(typeof true); // "boolean" console.log(typeof undefined); // "undefined" console.log(typeof null); // "object" ‚ö†Ô∏è (JavaScript bug!) console.log(typeof {}); // "object" console.log(typeof []); // "object" ‚ö†Ô∏è console.log(typeof function(){}); // "function" console.log(typeof Symbol()); // "symbol" console.log(typeof 123n); // "bigint"

instanceof Operator

Tests whether an object is an instance of a specific constructor.

// instanceof examples let arr = [1, 2, 3]; let date = new Date(); let regex = /hello/; console.log(arr instanceof Array); // true console.log(arr instanceof Object); // true (arrays are objects) console.log(date instanceof Date); // true console.log(regex instanceof RegExp); // true // With custom constructors function Person(name) { this.name = name; } let john = new Person("John"); console.log(john instanceof Person); // true

üí° Best Practice: When in doubt, use parentheses () to make your intentions clear and avoid relying on precedence rules.

üí° Best Practices

1. Use Strict Equality

// ‚úÖ Good if (x === 5) { } // ‚ùå Avoid if (x == 5) { }

2. Use ?? for Defaults

// ‚úÖ Good (0 is valid) const count = input ?? 0; // ‚ùå Wrong (0 becomes default) const count = input || 0;

3. Use Optional Chaining

// ‚úÖ Good const city = user?.address?.city; // ‚ùå Verbose const city = user && user.address && user.address.city;

4. Compound Assignment

// ‚úÖ Good count += 1; // ‚ùå Longer count = count + 1;

5. Avoid Nested Ternary

// ‚ùå Hard to read const x = a ? b : c ? d : e; // ‚úÖ Better const x = getResult();

6. Use Parentheses

// ‚úÖ Clear intent if ((a || b) && c) { } // ‚ùå Relies on precedence if (a || b && c) { }

üö´ Common Mistakes

1. Confusing Assignment with Comparison

// ‚ùå Wrong - assigns 5 to x, always true! if (x = 5) { } // ‚úÖ Correct - compares x with 5 if (x === 5) { }

2. String + Number Coercion

// ‚ùå Unexpected result const result = "5" + 3; // "53" (string!) // ‚úÖ Convert first const result = Number("5") + 3; // 8

3. Forgetting Operator Precedence

// ‚ùå Not what you think! const result = 2 + 3 * 4; // 14, not 20 // ‚úÖ Use parentheses const result = (2 + 3) * 4; // 20

üéì Quick Summary

  • JavaScript has many operator types for different tasks
  • Use === for comparisons, not ==
  • Modern operators: ??, ?., ... improve code quality
  • Understand operator precedence or use parentheses
  • Compound assignment operators (+=, -=) are more efficient
  • Be aware of type coercion with operators
  • Logical operators use short-circuit evaluation
  • Ternary operator is great for simple conditionals
