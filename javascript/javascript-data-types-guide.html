üìö What are Data Types?

Data types specify what kind of data can be stored and manipulated within a program. JavaScript is a dynamically typed language, meaning variables can hold values of any type without explicit type declaration.

let value = 42; // Number value = "Hello"; // Now it's a String value = true; // Now it's a Boolean

üî§ Primitive Data Types

Primitive types are immutable (cannot be changed) and are compared by value.

üîÑ Type Conversion

Implicit Conversion (Coercion)

JavaScript automatically converts types in certain situations.

// String coercion "5" + 3; // "53" (number to string) "Hello" + true; // "Hellotrue" // Number coercion "5" - 3; // 2 (string to number) "5" * "2"; // 10 true + 1; // 2 (true becomes 1) false + 1; // 1 (false becomes 0) // Boolean coercion if ("hello") {} // true (non-empty string) if (0) {} // false if ([]) {} // true (empty array is truthy!)

Explicit Conversion

// To String String(123); // "123" (123).toString(); // "123" 123 + ""; // "123" // To Number Number("123"); // 123 parseInt("123"); // 123 parseFloat("123.45"); // 123.45 +"123"; // 123 (unary plus) // To Boolean Boolean(0); // false Boolean(1); // true Boolean(""); // false Boolean("hello"); // true !!"hello"; // true (double negation)

Truthy and Falsy Values

‚ùå Falsy Values (convert to false)

  • false
  • 0
  • -0
  • 0n (BigInt zero)
  • "" (empty string)
  • null
  • undefined
  • NaN

‚úÖ Truthy Values (convert to true)

  • true
  • Any non-zero number
  • Any non-empty string
  • "0" (string)
  • "false" (string)
  • [] (empty array)
  • {} (empty object)
  • All functions

üí° Best Practice: Always use strict equality (===) and strict inequality (!==) to avoid unexpected type coercion bugs.

üí° Best Practices

  1. Use const and let - Avoid var for better scoping
  2. Use strict equality (===) - Prevents type coercion bugs
  3. Be explicit with type conversion - Don't rely on implicit coercion
  4. Check types when necessary - Use typeof for type checking
  5. Remember primitives are immutable - String methods return new strings
  6. Handle null and undefined - Check before using values
  7. Be aware of floating-point precision - Use libraries for precise calculations
  8. Use descriptive variable names - Indicate the type (e.g., userName, isActive, totalCount)
  9. Use Number.isNaN() - More reliable than isNaN() for checking NaN
  10. Always initialize variables - Avoid undefined values when possible

üéØ Common Pitfalls

1. typeof null returns "object"

// Wrong way if (typeof value === "object") { /* Could be null! */ } // Right way if (value !== null && typeof value === "object") { /* Safe */ }

2. NaN is not equal to itself

NaN === NaN; // false! // Use Number.isNaN() instead Number.isNaN(NaN); // true

3. String concatenation vs addition

"5" + 3; // "53" (string concatenation) 5 + 3; // 8 (addition) "5" - 3; // 2 (coercion to number)

4. Truthy and falsy confusion

if ("0") { console.log("This runs!"); // "0" is truthy! } if (0) { console.log("This doesn't run"); // 0 is falsy }
