🔧 9 — @apply & @layer (Reusable Classes)

The .btn, .btn-primary, and .btn-outline classes below are defined using @apply in the <style> block at the top of this file — not with Tailwind directly in markup.

@layer components {
  .btn {
    @apply inline-flex items-center justify-center rounded-lg px-5 py-2.5 text-sm font-semibold transition-colors duration-200;
  }
  .btn-primary {
    @apply btn bg-blue-600 text-white hover:bg-blue-700 focus:ring-4 focus:ring-blue-300;
  }
}
⚠️ Don't overuse @apply. Use utilities directly in markup whenever possible. Reserve @apply for third-party elements or CMS-generated content you can't add classes to directly.

🔍 11 — Class Detection & The Dynamic Class Gotcha

Tailwind scans your source files for complete class strings. Dynamically constructed class names won't be detected and will be missing from the compiled output.

❌ Dynamic Construction (Broken)

// Compiler CANNOT detect these
const color = "red";
const shade = "500";

<div class={`bg-${color}-${shade}`}>
  Unstyled! 😱
</div>

✅ Complete Strings (Works)

// Full class names ARE detected
const colorClass =
  isError
    ? "bg-red-500"
    : "bg-green-500";

<div class={colorClass}>
  Styled! ✅
</div>
