Note: ASP.NET Core MVC is for web development, not Windows desktop apps!

üó£Ô∏è .NET Programming Languages

Several programming languages can be used to develop .NET applications:

C#

The most popular .NET language. Modern, object-oriented, and what we use in this course.

Visual Basic

Easier syntax, great for beginners. Still supported but less common today.

F#

Functional programming language. Used for data science and complex algorithms.

JavaScript is NOT a .NET language! It's a web scripting language used in browsers.

üì¶ .NET Framework vs .NET Core (.NET)

Aspect .NET Framework .NET Core / .NET 5+
Age Older (2002) Newer (2016+)
Platform Windows only Cross-platform (Windows, Mac, Linux)
Status Maintenance mode Actively developed
Current Version 4.8 (final) .NET 8, .NET 9...
.NET Core is newer than .NET Framework, and it is cross-platform. New projects should target .NET (not .NET Framework) when possible.

üß© Two Main Components of .NET

üìö Class Library

Provides pre-written code that can be used by .NET applications. Contains thousands of classes for common tasks like file I/O, networking, and UI.

‚öôÔ∏è Common Language Runtime (CLR)

Manages the execution of .NET applications. Handles memory management, security, and converts IL to machine code.

üìÇ Solutions and Projects

üìÅ Solution

A container that holds projects. Can contain one or more related projects. Has a .sln file.

üìÑ Project

Contains the actual code files, forms, and resources for one application. Has a .csproj file.

üìÇ Opening an Existing Project

There are several ways to open an existing project in Visual Studio:

Ways to Open a Project

  • Start Window: Click on a recent project, or click "Open a project or solution"
  • File Menu: File ‚Üí Open ‚Üí Project/Solution
  • Recent Projects: File ‚Üí Recent Projects and Solutions
Note: The "File ‚Üí Recent Files" submenu is for individual files, NOT for opening projects.

ü™ü Managing Windows

Windows in Visual Studio can be positioned in different ways. A window that appears attached to the edge of the IDE is called a docked window.

Action How To
Open a window Use the View menu (e.g., View ‚Üí Solution Explorer)
Hide a window Click the X button or use Auto Hide (üìå pin icon)
Auto Hide Window slides to edge; appears when you hover over it
Dock a window Drag window by title bar to edge of IDE (becomes a docked window)
Float a window Drag window away from docking area
Docked Window: A window attached to the edge of the Visual Studio IDE. Most tool windows (Solution Explorer, Toolbox, Properties) are docked by default.

‚ñ∂Ô∏è Building and Running Projects

üî® Build

Compiles your C# code into an executable assembly. Checks for syntax errors.

Menu: Build ‚Üí Build Solution

Shortcut: Ctrl+Shift+B

‚ñ∂Ô∏è Run (Start)

Builds the project (if needed) AND runs it. Opens the application's first form.

Menu: Debug ‚Üí Start Debugging

Shortcut: F5 or click Start button

When you run a project: Visual Studio first builds it (compiles to IL), then runs the executable. The project's first form is displayed.

üéØ Targeting a .NET Version

You can change which version of .NET your project targets using the Project Properties window.

How to Change Target Framework

  • Right-click your project in Solution Explorer
  • Select Properties
  • In the Application tab, find Target framework
  • Select the desired .NET version from the dropdown
