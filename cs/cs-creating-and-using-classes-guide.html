Guide #12: How to Create and Use Classes

Classes, Objects, Properties, Constructors, and OOP Fundamentals

üèóÔ∏è Object-Oriented Programming Concepts

Object-Oriented Programming (OOP) organizes code around objects ‚Äî self-contained units that bundle data and behavior together. A class is a blueprint that defines what properties and methods an object will have. An object is a specific instance created from that class.

üî® Instantiation

The process of creating an object from a class. You use the new keyword to instantiate an object.

üì¶ Encapsulation

Hiding the internal details of how a class works. You can call Convert.ToInt32() without knowing how it works inside.

üß¨ Inheritance

A class can be based on another class, gaining its properties and methods while adding its own.

üß© Composition

Building complex objects by combining simpler objects as fields. For example, a Car class might contain an Engine object.

Three-Layer Architecture

Well-organized applications separate code into three layers:

Layer Purpose Example
Presentation Layer The user interface ‚Äî what the user sees and interacts with Your Windows Form (.cs form file)
Middle Layer Business logic ‚Äî the rules and calculations of the application Your custom classes (e.g., Pet.cs)
Database Layer Data storage and retrieval Code that reads/writes to a database or file
Key distinction: Two objects created from the same class share the same fields, methods, and properties ‚Äî but each holds its own data. Think of a class like a cookie cutter and objects like the cookies: same shape, different sprinkles.

üîß Creating a Class File

A class is stored in its own .cs file. To add a new class to your project in Visual Studio: right-click the project in Solution Explorer ‚Üí Add ‚Üí Class. Name the file to match the class name (e.g., Pet.cs).

üîí Fields, Properties, and Access Modifiers

Fields (Instance Variables)

A field (also called a member variable or instance variable) stores data inside an object. Fields are almost always declared private so that outside code cannot directly access or change them. This is the "information hiding" part of encapsulation.

private string _name;
private decimal _price;
private int _quantity;
Naming convention: Private fields in C# typically start with an underscore: _name, _price, _quantity. This helps distinguish them from parameters and local variables.

Getter and Setter Methods

Since fields are private, you need public methods that let outside code read or change those values. A getter returns the field's value. A setter accepts a new value and assigns it to the field.

// Getter ‚Äî returns the value
public string GetName()
{
    return _name;
}

// Setter ‚Äî accepts a value and assigns it
public void SetName(string name)
{
    _name = name;
}

Properties (The C# Way)

C# provides properties as a cleaner alternative to getter/setter methods. A property looks like a field to the outside world but uses get and set accessors behind the scenes.

Syntax: Property with Backing Field

private string _name;

public string Name
{
    get { return _name; }
    set { _name = value; }
}

Auto-Implemented Properties

When the getter and setter simply return and set a value with no extra logic, you can use an auto-implemented property. C# creates the backing field for you automatically behind the scenes.

// Auto-implemented: C# creates the hidden backing field
public string Name { get; set; }

// Read-only auto-implemented (no set accessor)
public decimal Total { get; }

// Read-only with initial value
public string Title { get; } = "Untitled";

Special Property Accessors

Declaration Meaning
{ get; set; } Read/write ‚Äî anyone can get or set the value
{ get; } Read-only ‚Äî value can only be set in the constructor
{ get; init; } Read-only, but can be set by a constructor or an object initializer
required keyword The property must be assigned a value when the object is created 
// init accessor ‚Äî settable in constructor or object initializer only
public string Code { get; init; }

// required ‚Äî must be assigned when creating the object
public required string Code { get; set; }

üè≠ Constructors

A constructor is a special method that runs when an object is created. It has the same name as the class and no return type (not even void). Its purpose is to initialize the object's fields.

Syntax: Constructor

public class Bike
{
    private string _brand;
    private int _gears;

    // Constructor ‚Äî same name as the class, no return type
    public Bike(string brand, int gears)
    {
        _brand = brand;
        _gears = gears;
    }
}

Default Constructor

If you don't write any constructors, C# provides a default constructor (no parameters) automatically. However, as soon as you write any constructor, the default one goes away unless you explicitly add it back.

Overloaded Constructors

You can write multiple constructors with different parameter lists. This is called overloading.

public class Bike
{
    private string _brand;
    private int _gears;

    // Default constructor
    public Bike()
    {
        _brand = "Unknown";
        _gears = 1;
    }

    // Parameterized constructor
    public Bike(string brand, int gears)
    {
        _brand = brand;
        _gears = gears;
    }
}

‚ú® Creating Objects (Instantiation)

Once you have a class, you create objects from it using the new keyword. There are several valid syntaxes:

Ways to Instantiate an Object

// Standard syntax
Bike myBike = new Bike("Trek", 21);

// Using var (type inferred)
var myBike = new Bike("Trek", 21);

// Target-typed new (type inferred from variable)
Bike myBike = new("Trek", 21);
Invalid syntax: Bike myBike = new; ‚Äî you must always include the parentheses () after new, even for a default constructor.

Object Initializers

An object initializer lets you set properties at the time of creation using curly braces. Note the syntax: property names (not variable names), equals signs, and commas between assignments.

// Without object initializer
Bike myBike = new Bike();
myBike.Brand = "Giant";
myBike.Gears = 18;

// With object initializer ‚Äî more concise
Bike myBike = new Bike
{
    Brand = "Giant",
    Gears = 18
};

üéØ Using Objects in Your Form

Accessing Properties and Methods

Once you have an object, use the dot operator (.) to access its public properties and methods.

// Setting a property
myBike.Brand = "Cannondale";

// Getting a property
string brand = myBike.Brand;

// Calling a method
string name = myPet.GetName();
myPet.HadBirthday();

Storing Objects in Arrays

Many of your labs require storing multiple objects in an array. Declare an array of your class type, then assign objects to each index.

// Declare an array that holds up to 20 Pet objects
Pet[] pets = new Pet[20];
int petCount = 0;

// Add a new pet
pets[petCount] = new Pet("Buddy", "Dog", 3);
petCount++;

// Search the array
for (int i = 0; i < petCount; i++)
{
    if (pets[i].GetName().ToLower() == searchTerm)
    {
        // Found it ‚Äî display info
        lblResult.Text = pets[i].GetName() + 
            " is a " + pets[i].GetSpecies();
    }
}
Pattern for your labs: Declare the array and a counter as class-level variables in your form (outside any method but inside the form class) so they persist across button clicks.

‚ö° Static vs. Instance Members

A static method or field belongs to the class itself, not to any individual object. You call static members using the class name. An instance method belongs to a specific object.

‚úÖ Static (use class name)

// Call using the class name
decimal val = Convert.ToDecimal("9.99");
double root = Math.Sqrt(25);

‚úÖ Instance (use object name)

// Call using the object variable
Pet p = new Pet("Max", "Cat", 5);
string name = p.GetName();

‚û°Ô∏è Expression-Bodied Members

When a property or method only contains a single expression, you can use the lambda operator (=>) as a shorthand. This is called an expression body.

// Standard read-only property
public decimal TotalCost
{
    get
    {
        return _quantity * _price;
    }
}

// Same thing as an expression-bodied property
public decimal TotalCost => _quantity * _price;

// Expression-bodied method
public string GetSummary() => _name + ": $" + _price;
Limitation: You cannot use an expression body when a method or property executes multiple statements. Use the full block syntax with curly braces in that case.

üîÑ Method Overloading

Overloading means having multiple methods with the same name but different parameter lists. The parameter list must differ in the number of parameters or the data types of parameters ‚Äî just changing the parameter names or the return type does not count.

public string FormatLabel(string text)
{
    return text.ToUpper();
}

// Overloaded: same name, different parameter list
public string FormatLabel(string text, int maxLength)
{
    if (text.Length > maxLength)
        return text.Substring(0, maxLength) + "...";
    return text.ToUpper();
}

üîç Type Checking with is

The is keyword lets you check whether an object is a specific type and cast it in one step. This is especially useful when working with the sender parameter in event handlers.

// Check type and cast in one step
if (someObject is Pet p)
{
    // p is now a Pet variable you can use
    lblOutput.Text = p.GetName();
}

üìã Structures, Records, and Record Structs

C# offers alternatives to classes for specific scenarios:

Type Value or Reference? Equality Comparison Best For
class Reference type Compares references (are they the same object?) Complex objects with behavior, inheritance
struct Value type Compares references by default Small, simple data; less memory overhead
record Reference type Compares property values Data-focused objects where you want value-based comparison
record struct Value type Compares property values Small data objects with value-based comparison

When to Choose What

Class: Your object is complex, has many methods, or you need inheritance.

Struct: Your object primarily stores data and is small enough to copy efficiently.

Record: Your object primarily stores data, you want a reference type, and you want to compare objects based on their property values.

Record struct: Your object primarily stores data, is small enough to copy efficiently, and you want improved equality comparison.

üß™ Putting It All Together ‚Äî Complete Example

Here is a complete example showing a class and the form code that uses it. This is not from any lab ‚Äî it demonstrates the patterns you'll use across all assignments.

Snack.cs ‚Äî The Class

public class Snack
{
    private string _name;
    private decimal _price;
    private int _calories;

    // Constructor
    public Snack(string name, decimal price, int calories)
    {
        _name = name;
        _price = price;
        _calories = calories;
    }

    // Getter methods
    public string GetName() => _name;
    public decimal GetPrice() => _price;
    public int GetCalories() => _calories;

    // Setter method
    public void SetPrice(decimal price)
    {
        _price = price;
    }

    // Display method
    public string Display()
    {
        return _name + " ($" + _price + ") - " 
            + _calories + " cal";
    }
}

Form Code ‚Äî Using the Snack Class

public partial class Form1 : Form
{
    // Class-level: array and counter persist across button clicks
    Snack[] snacks = new Snack[20];
    int snackCount = 0;

    public Form1()
    {
        InitializeComponent();

        // Pre-populate with some data
        snacks[0] = new Snack("Pretzels", 2.50m, 210);
        snacks[1] = new Snack("Trail Mix", 4.00m, 350);
        snacks[2] = new Snack("Granola Bar", 1.75m, 180);
        snackCount = 3;
    }

    private void btnSearch_Click(object sender, EventArgs e)
    {
        string search = txtSearch.Text.ToLower();
        bool found = false;

        for (int i = 0; i < snackCount; i++)
        {
            if (snacks[i].GetName().ToLower() == search)
            {
                lblResult.Text = snacks[i].Display();
                found = true;
                break;
            }
        }

        if (!found)
        {
            lblResult.Text = "Snack not found.";
        }
    }

    private void btnAdd_Click(object sender, EventArgs e)
    {
        if (snackCount < 20)
        {
            string name = txtName.Text;
            decimal price = Convert.ToDecimal(txtPrice.Text);
            int cal = Convert.ToInt32(txtCalories.Text);

            snacks[snackCount] = new Snack(name, price, cal);
            snackCount++;

            txtName.Clear();
            txtPrice.Clear();
            txtCalories.Clear();
            txtName.Focus();
        }
    }
}
