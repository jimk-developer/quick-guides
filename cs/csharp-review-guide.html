This week you learned object-oriented programming concepts, how to write event handlers, work with data types, and debug your applications.

Object-Oriented Programming Concepts

üéØ Core OOP Terms

  • Class ‚Äî A blueprint that defines an object's characteristics (properties, methods, events)
  • Object ‚Äî An instance of a class
  • Instantiation ‚Äî The process of creating an object from a class
  • Property ‚Äî A characteristic/data of an object (e.g., txtName.Text)
  • Method ‚Äî An operation an object can perform (e.g., txtName.Focus())
  • Event ‚Äî A signal that something has happened (e.g., Click, Load)

Dot Notation

Access properties and methods using: objectName.memberName

// Getting a property value
string name = txtName.Text;

// Setting a property value
lblResult.Text = "Hello!";

// Calling a method
txtMonthlyInvestment.Focus();

// Close the current form
this.Close();

Event Handlers

Event handlers are methods that respond to events. Double-click a control in the designer to create its default event handler.

// Event handler naming: ControlName_EventName
private void btnCalculate_Click(object sender, EventArgs e)
{
    // Your code goes here
}

private void btnExit_Click(object sender, EventArgs e)
{
    this.Close();
}

Loops

while Loop

Condition tested before loop body. May execute zero times.

while (count < 10)
{
    total += count;
    count++;
}

do-while Loop

Condition tested after loop body. Always executes at least once.

do
{
    total += count;
    count++;
} while (count < 10);

for Loop

Best for counter-controlled loops where you know the number of iterations.

// for (initializer; condition; iterator)
for (int i = 0; i < 10; i++)
{
    total += i;
}

Loop Control Statements

StatementPurpose
breakExit the loop immediately
continueSkip to next iteration

Conditional Operator (Ternary)

// condition ? trueValue : falseValue
string status = (score >= 60) ? "Pass" : "Fail";

üìã When to Use Which Loop

  • for ‚Äî Known number of iterations, need a counter
  • while ‚Äî Unknown iterations, condition checked first
  • do-while ‚Äî Must execute at least once (e.g., menu prompts)
Variables declared inside a loop or if block have block scope ‚Äî they only exist within those curly braces.
Week 4

Arrays and Collections ‚Äî Chapter 8

This week you learned to work with arrays and generic collections to store and manipulate groups of related data.

Array Fundamentals

An array stores multiple values of the same type using zero-based indexing.

// Declare with size
int[] scores = new int[5];

// Declare and initialize
string[] names = {"Alice", "Bob", "Carol"};

// Decimal array - use 'm' suffix!
decimal[] prices = {19.99m, 29.99m, 39.99m};

// Access elements
string first = names[0];    // "Alice"
names[1] = "Robert";        // Change "Bob" to "Robert"

// Get array length
int count = names.Length;   // 3
Arrays use zero-based indexing. The 9th element is at index [8], not [9]!

Rectangular Arrays (2D)

// Declare with dimensions [rows, columns]
string[,] grid = new string[3, 4];  // 3 rows, 4 columns

// Initialize with values
int[,] matrix = {
    {1, 2, 3},
    {4, 5, 6},
    {7, 8, 9}
};

// Access: array[row, column]
int value = matrix[1, 0];  // Row 1, Col 0 = 4

// Get dimensions
int rows = matrix.GetLength(0);    // 3
int cols = matrix.GetLength(1);    // 3

for Loop vs foreach Loop

for Loop

Use when you need the index or want to modify elements.

for (int i = 0; i < arr.Length; i++)
{
    arr[i] = arr[i] * 2;  // Can modify
}

foreach Loop

Simpler syntax for read-only iteration.

foreach (int num in arr)
{
    Console.WriteLine(num);
    // Cannot modify num
}

Generic Collections: List<T>

Lists are dynamic ‚Äî they grow and shrink automatically. Much more flexible than arrays!

// Create a List
List<string> names = new List<string>();

// Add items
names.Add("Alice");
names.Add("Bob");

// Insert at specific position
names.Insert(1, "Carol");  // Insert at index 1

// Remove items
names.Remove("Bob");         // Remove by value
names.RemoveAt(0);            // Remove by index

// Get count (not Length!)
int count = names.Count;

Queue and Stack

üì¨ Queue<T> ‚Äî FIFO

First-In, First-Out ‚Äî like a line at a store.

Queue<string> line = new();
line.Enqueue("First");
line.Enqueue("Second");
string next = line.Dequeue();
// next = "First"

üìö Stack<T> ‚Äî LIFO

Last-In, First-Out ‚Äî like a stack of plates.

Stack<string> plates = new();
plates.Push("First");
plates.Push("Second");
string top = plates.Pop();
// top = "Second"

üìã Collection Comparison

  • Array ‚Äî Fixed size, use .Length
  • List<T> ‚Äî Dynamic size, use .Count, Add/Remove methods
  • Queue<T> ‚Äî FIFO, Enqueue/Dequeue
  • Stack<T> ‚Äî LIFO, Push/Pop
  • Peek() ‚Äî View next item without removing it (both Queue and Stack)
