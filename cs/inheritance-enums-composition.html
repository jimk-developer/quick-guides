üè∑Ô∏è Enums (Enumerations)

An enum defines a set of named constants that represent a fixed group of related values. Instead of using "magic numbers" or raw strings to represent options like room numbers, flavors, or categories, you create a named type with specific allowed values.

Declaring an Enum

An enum is declared outside of any class (typically in its own .cs file, or at the top of a file before the class). Each value in an enum is automatically assigned an integer starting from 0 unless you specify otherwise.

Syntax: Basic Enum

// Simple enum ‚Äî values are 0, 1, 2, 3 automatically
public enum Season
{
    SPRING,      // = 0
    SUMMER,      // = 1
    FALL,        // = 2
    WINTER       // = 3
}

Syntax: Enum with Custom Values

// Enum with explicit integer values
public enum Building
{
    MAIN_HALL   = 100,
    ANNEX_A     = 200,
    ANNEX_B     = 201,
    WAREHOUSE   = 300
}

Using an Enum

Once declared, you use an enum like any other data type ‚Äî as a field type, a property type, a parameter type, or a local variable type. You access a specific value using the enum name, a dot, and the value name.

// Declare a variable of the enum type
Season current = Season.SUMMER;

// Use as a field in a class
private Building _location;

// Use as an auto-implemented property
public Season FavoriteSeason { get; set; }

// Use in a constructor parameter
public Event(string name, Building location)
{
    _location = location;
}

// Compare enum values
if (current == Season.WINTER)
{
    lblMessage.Text = "Bundle up!";
}

Naming convention: Enum type names are PascalCase (e.g., Season, Building). Enum values are typically UPPER_CASE (e.g., SPRING, MAIN_HALL).

Using Enums with Properties

A property's type can be an enum. This is extremely common ‚Äî it lets a class store a value that is restricted to a known set of options.

public class Event
{
    // Auto-implemented property with enum type
    public string Name { get; set; }
    public Building Location { get; set; }
    public int Capacity { get; set; }

    public string Display()
    {
        return Name + " in " + Location + 
               " (capacity: " + Capacity + ")";
    }
}

// Creating an Event with an enum value
Event meeting = new Event();
meeting.Name = "Staff Meeting";
meeting.Location = Building.MAIN_HALL;
meeting.Capacity = 50;
When you display an enum value in a string (using + or string interpolation), C# automatically converts it to its name. So Building.MAIN_HALL becomes the string "MAIN_HALL". If you want its integer value, cast it: (int)Building.MAIN_HALL gives you 100.

üß© Composition ("Has-A" Relationships)

Composition is when one class contains an object of another class as a field. This models "has-a" relationships: a Car has an Engine. The contained object is part of the containing object.

How It Works

You simply declare a field or property whose type is another class (or enum). The containing class "owns" or "uses" the contained object.

Example: A Classroom Has a Projector

public class Projector
{
    private string _brand;
    private int _lumens;

    public Projector(string brand, int lumens)
    {
        _brand = brand;
        _lumens = lumens;
    }

    public string GetBrand() => _brand;
    public int GetLumens() => _lumens;
}

public class Classroom
{
    private string _roomNumber;
    private Projector _projector;    // ‚Üê Composition!

    public Classroom(string roomNumber, Projector projector)
    {
        _roomNumber = roomNumber;
        _projector = projector;
    }

    public string GetInfo()
    {
        return "Room " + _roomNumber + " - Projector: " + 
               _projector.GetBrand() + " (" + 
               _projector.GetLumens() + " lumens)";
    }
}
// Creating composed objects from the form
Projector proj = new Projector("Epson", 3200);
Classroom room = new Classroom("A204", proj);

lblInfo.Text = room.GetInfo();
// Output: "Room A204 - Projector: Epson (3200 lumens)"

Composition with Enums

Composition doesn't only work with classes ‚Äî a class can also "contain" enum values as fields or properties. In fact, you'll frequently combine both in a single class.

public class Classroom
{
    private string _roomNumber;
    private Building _building;        // ‚Üê Enum field
    private Projector _projector;      // ‚Üê Object field
    // ...
}

Composition with Arrays of Objects

A class can also hold an array of objects. This is useful when one object needs to contain a variable number of related items ‚Äî for example, a sundae that can have multiple toppings.

public class Playlist
{
    private string _name;
    private Song[] _songs;           // ‚Üê Array of objects
    private int _songCount;

    public Playlist(string name)
    {
        _name = name;
        _songs = new Song[50];
        _songCount = 0;
    }

    public void AddSong(Song song)
    {
        if (_songCount < _songs.Length)
        {
            _songs[_songCount] = song;
            _songCount++;
        }
    }
}

Composition vs. Inheritance ‚Äî Know the Difference

Composition = "has-a" ‚Üí A Classroom has a Projector. The Classroom contains a Projector object as a field.

Inheritance = "is-a" ‚Üí A Dog is an Animal. Dog extends Animal and gains all of its members.

If you're unsure which to use, ask yourself: "Is X a kind of Y?" (inheritance) or "Does X contain Y?" (composition).

üìå Constants in Classes

A constant is a field whose value is set once and can never change. In UML diagrams, constants are marked with ¬´const¬ª. In C#, you use the const keyword. Constants are always static (they belong to the class, not to individual objects).

public class Delivery
{
    public const double BASE_FEE = 5.99;
    public const double PER_MILE_RATE = 0.75;

    private double _miles;

    public Delivery(double miles)
    {
        _miles = miles;
    }

    public double CalcTotal()
    {
        return BASE_FEE + (_miles * PER_MILE_RATE);
    }
}

// Access from outside the class using the class name
double fee = Delivery.BASE_FEE;  // 5.99
UML notation: When you see + ¬´const¬ª BASE_FEE : double = 5.99 in a UML diagram, translate it to public const double BASE_FEE = 5.99; in your class.

üß¨ Inheritance ("Is-A" Relationships)

Inheritance lets you create a new class (the derived class or child class) that is based on an existing class (the base class or parent class). The derived class automatically inherits all public and protected members of the base class, and can add its own unique members.

Basic Syntax

Use a colon (:) after the derived class name, followed by the base class name.

Syntax: Inheritance

// Base class
public class Vehicle
{
    public string Make { get; set; }
    public int Year { get; set; }

    public string GetBasicInfo()
    {
        return Year + " " + Make;
    }
}

// Derived class ‚Äî inherits Make, Year, and GetBasicInfo()
public class Truck : Vehicle       // ‚Üê Truck IS A Vehicle
{
    public double TowingCapacity { get; set; }
}
// Truck has its own TowingCapacity PLUS inherited Make, Year, GetBasicInfo()
Truck t = new Truck();
t.Make = "Ford";           // inherited from Vehicle
t.Year = 2024;             // inherited from Vehicle
t.TowingCapacity = 12000;  // unique to Truck

lblInfo.Text = t.GetBasicInfo();  // inherited method works!

Calling the Base Constructor

When a derived class has a constructor, it can pass values up to the base class constructor using : base(...).

public class Vehicle
{
    public string Make { get; set; }
    public int Year { get; set; }

    public Vehicle(string make, int year)
    {
        Make = make;
        Year = year;
    }
}

public class Truck : Vehicle
{
    public double TowingCapacity { get; set; }

    // Pass make and year up to Vehicle's constructor
    public Truck(string make, int year, double towCap)
        : base(make, year)         // ‚Üê calls Vehicle(make, year)
    {
        TowingCapacity = towCap;   // handle Truck-specific field
    }
}

üé® Abstract Classes and Methods

An abstract class is a class that cannot be instantiated directly. It serves as a base class only ‚Äî you must create a derived class from it. Abstract classes can contain both regular methods (with code) and abstract methods (with no code ‚Äî just a signature that derived classes must implement).

Syntax: Abstract Class

// Abstract base class ‚Äî cannot do: new Appliance()
public abstract class Appliance
{
    public string Name { get; set; }
    public double Watts { get; set; }

    public Appliance(string name, double watts)
    {
        Name = name;
        Watts = watts;
    }

    // Regular method ‚Äî shared code, inherited as-is
    public string GetBasicInfo()
    {
        return Name + " (" + Watts + "W)";
    }

    // Abstract method ‚Äî NO body, derived classes MUST override this
    public abstract double CalcMonthlyCost();
}

Overriding Abstract Methods

Every derived class must provide its own implementation for each abstract method using the override keyword. Each derived class can have a completely different formula or logic. 

public class Refrigerator : Appliance
{
    public double HoursPerDay { get; set; }

    public Refrigerator(string name, double watts, double hours)
        : base(name, watts)
    {
        HoursPerDay = hours;
    }

    // MUST override the abstract method
    public override double CalcMonthlyCost()
    {
        // Runs 24/7 ‚Äî different formula than other appliances
        return (Watts / 1000.0) * HoursPerDay * 30 * 0.12;
    }
}

public class WashingMachine : Appliance
{
    public int LoadsPerWeek { get; set; }
    public double MinutesPerLoad { get; set; }

    public WashingMachine(string name, double watts, 
        int loads, double minutes)
        : base(name, watts)
    {
        LoadsPerWeek = loads;
        MinutesPerLoad = minutes;
    }

    // Different formula ‚Äî based on loads per week
    public override double CalcMonthlyCost()
    {
        double hoursPerMonth = (LoadsPerWeek * MinutesPerLoad / 60.0) * 4;
        return (Watts / 1000.0) * hoursPerMonth * 0.12;
    }
}
You cannot instantiate an abstract class directly. Appliance a = new Appliance(...); will cause a compile error. You must create a Refrigerator or WashingMachine instead.

üé≠ Polymorphism

Polymorphism means "many forms." It's what happens when you call a method on a base class variable, but the actual derived class determines which version of the method runs. This is the most powerful concept in OOP.

How It Works

You can store a derived class object in a variable of the base class type. When you call an overridden method on that variable, C# automatically calls the correct derived class version.

// Array of the BASE class type ‚Äî holds ANY derived type
Appliance[] appliances = new Appliance[10];
int count = 0;

// Store different derived types in the same array
appliances[0] = new Refrigerator("Kitchen Fridge", 150, 8);
appliances[1] = new WashingMachine("Front Loader", 500, 5, 45);
count = 2;

// Polymorphism: the SAME method call produces DIFFERENT results
for (int i = 0; i < count; i++)
{
    double cost = appliances[i].CalcMonthlyCost();  // ‚Üê correct formula runs!
    lstOutput.Items.Add(appliances[i].GetBasicInfo() + 
        " - $" + cost.ToString("F2") + "/month");
}
This is the core pattern in your demos and labs: declare an array of the base type, fill it with objects of various derived types, then loop through and call methods. Each object responds with its own behavior. See Demo 1 (Animal[] holding Dog and Cat), Demo 2 (Workout[] holding CardioWorkout and StrengthWorkout), and Demo 4 (Container[] holding BoxContainer, CylinderContainer, and ConeContainer).

üîç Type Checking with the is Keyword

When you have an array of base class type, sometimes you need to know the actual derived type of a specific object. The is keyword checks the type and casts in one step.

// Check what type an object actually is
for (int i = 0; i < count; i++)
{
    if (appliances[i] is Refrigerator fridge)
    {
        // fridge is now a Refrigerator variable ‚Äî access Refrigerator-only members
        lblMessage.Text = fridge.Name + " runs " + fridge.HoursPerDay + " hrs/day";
    }
    else if (appliances[i] is WashingMachine washer)
    {
        lblMessage.Text = washer.Name + ": " + washer.LoadsPerWeek + " loads/week";
    }
}
When to use is: Use it when you need to access a property or method that only exists on a specific derived class. If you only need to call methods that are defined on the base class, you don't need is ‚Äî polymorphism handles it. See Demo 1's btnSearch_Click for a real example using is Dog dog and is Cat cat.

üìÖ Working with DateTime

The DateTime type in C# represents a date and time. It's useful when your classes need to store dates ‚Äî for events, logs, records, etc.

Creating a DateTime

// From a constructor (year, month, day)
DateTime eventDate = new DateTime(2026, 3, 15);

// Current date and time
DateTime now = DateTime.Now;

// Parsing from a text box string
DateTime parsed = DateTime.Parse(txtDate.Text);

// Safer parsing with TryParse
if (DateTime.TryParse(txtDate.Text, out DateTime result))
{
    // result now holds the parsed date
}

Comparing Dates

You can compare DateTime values using the standard comparison operators. This is essential for searching by date range.

DateTime start = DateTime.Parse(txtBegin.Text);
DateTime end = DateTime.Parse(txtEnd.Text);

// Check if a date falls within a range
if (eventDate >= start && eventDate <= end)
{
    // This date is within the search range
}

Formatting Dates

DateTime d = new DateTime(2026, 10, 16);

d.ToString("MM/dd/yyyy")     // "10/16/2026"
d.ToString("MMM dd, yyyy")   // "Oct 16, 2026"
d.ToShortDateString()       // "10/16/2026"

public partial class Form1 : Form
{
    // Array of ABSTRACT BASE type ‚Äî holds ANY derived type
    Appliance[] appliances = new Appliance[20];
    int count = 0;

    public Form1()
    {
        InitializeComponent();

        // Pre-load with different derived types
        appliances[0] = new Refrigerator("Kitchen Fridge", 150, 8);

    private void btnDisplayAll_Click(object sender, EventArgs e)
    {
        lstOutput.Items.Clear();

        for (int i = 0; i < count; i++)
        {
            // Polymorphism ‚Äî correct CalcMonthlyCost() runs for each type
            string info = appliances[i].GetBasicInfo() + 
                " - $" + appliances[i].CalcMonthlyCost().ToString("F2");
            lstOutput.Items.Add(info);
        }
    }

    private void btnAdd_Click(object sender, EventArgs e)
    {
        // Determine which derived type to create
        if (rdoFridge.Checked)
        {
            appliances[count] = new Refrigerator(
                txtName.Text, 
                double.Parse(txtWatts.Text),
                double.Parse(txtExtra.Text));
        }
        else if (rdoWasher.Checked)
        {
            appliances[count] = new WashingMachine(
                txtName.Text, 
                double.Parse(txtWatts.Text),
                int.Parse(txtExtra.Text),
                double.Parse(txtExtra2.Text));
        }
        count++;
    }
}
