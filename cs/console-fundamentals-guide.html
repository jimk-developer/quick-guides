C# Console Application Fundamentals

A Comprehensive Guide for Beginners

This guide covers essential concepts for building C# console applications in Visual Studio 2026. You'll learn about variables, constants, data types, user input/output, calculations, and formatting ‚Äì everything you need to create interactive command-line programs.

üéØ What is a Console Application?

A console application is a text-based program that runs in a command-line window (also called a terminal or console). Unlike graphical applications with buttons and windows, console apps interact with users through text input and output.

üí° Why Start with Console Apps?
  • Simple interface - Focus on logic, not graphics
  • Quick feedback - Test your code immediately
  • Foundation skills - Learn core programming concepts
  • Professional tools - Many utilities are console-based

Modern C# uses top-level statements, which means you can write code directly without a class or Main method. This makes simple programs cleaner and easier to read.

üì¶ Variables and Constants

What is a Variable?

A variable is a named storage location in memory that holds a value. Variables can change during program execution.

Variable Declaration Syntax:
dataType variableName;

Variable Declaration with Initialization:
dataType variableName = value;
Declaring and Using Variables
// Declaration only
int age;

// Declaration with initialization
int count = 10;
double price = 19.99;
string name = "Alice";

// Assignment (changing the value)
age = 25;
count = count + 1;

What is a Constant?

A constant is a named value that cannot change after it's initialized. Constants are declared using the const keyword and must be assigned a value immediately.

Declaring and Using Constants
// Constant declaration - value cannot be changed
const double KM_PER_MILE = 1.6;
const double PI = 3.14159;
const double PERCENT_RAISE = 0.04;

// Use the constant in calculations
double kilometers = miles * KM_PER_MILE;
Naming Conventions: By convention, constants are named in ALL_CAPS with underscores separating words (e.g., KM_PER_MILE). Variables use camelCase (e.g., userName, totalPrice).
Common Mistake: Trying to change a constant's value will cause a compile error. 
const double TAX_RATE = 0.08;
TAX_RATE = 0.09; // ERROR! Cannot modify a constant

üî¢ Data Types

C# is a strongly-typed language, meaning every variable must have a specific data type. Data types determine what kind of values a variable can hold and what operations can be performed on it.

Intrinsic (Built-in) Data Types

int Whole numbers (integers) 42, -17, 0, 1000 -2.1 billion to 2.1 billion
double Decimal numbers (floating-point) 3.14, -0.5, 19.99 ¬±5.0 √ó 10-324 to ¬±1.7 √ó 10308
decimal Precise decimal numbers (for money) 19.99m, 100.00m ¬±1.0 √ó 10-28 to ¬±7.9 √ó 1028
string Text (sequence of characters) "Hello", "John Doe" Any text
bool Boolean (true or false) true, false true or false only

Using Different Data Types
// Integer - whole numbers
int studentCount = 30;
int temperature = -5;

// Double - decimal numbers (most common for decimals)
double price = 29.99;
double distance = 3.5;

// Decimal - precise decimal (use for money)
decimal salary = 50000.00m; // Note the 'm' suffix

// String - text (use double quotes)
string firstName = "John";
string message = "Welcome to C#!";

// Boolean - true or false
bool isStudent = true;
bool hasLicense = false;

// Char - single character (use single quotes)
char grade = 'A';
char initial = 'J';
Choosing Between double and decimal: Use double for most mathematical calculations and scientific data. Use decimal when working with money, as it provides exact decimal representation without floating-point errors.

üìù Literals

A literal is a fixed value written directly in code. Different types of literals represent different kinds of data.

Integer Literals

-17     // negative int
1000    // int literal

Floating-Point Literals

3.14    // double literal

Character and Boolean Literals

'A'      // char literal
'x'      // char literal

String vs Char: Strings use double quotes ("text") and can contain multiple characters or none. Chars use single quotes ('A') and must contain exactly one character.

‚å®Ô∏è Console Input and Output

Console.WriteLine() - Display Output

The Console.WriteLine() method displays text to the console and moves to a new line.

Basic Output
// Display a simple message
Console.WriteLine("Welcome to C#!");

// Display multiple lines
Console.WriteLine("Hello");
Console.WriteLine("World");

// Display an empty line
Console.WriteLine();

Console.Write() - Display Without New Line

The Console.Write() method displays text without moving to a new line.

Write Without Line Break
Console.Write("Enter your name: ");  // Stays on same line
string name = Console.ReadLine();      // User types here

Displaying Variables

Output with Variables
string name = "Alice";
int age = 25;
double salary = 50000.00;

// Method 1: Concatenation with +
Console.WriteLine("Name: " + name);
Console.WriteLine("Age: " + age);

// Method 2: String interpolation (recommended)
Console.WriteLine($"Name: {name}");
Console.WriteLine($"{name} is {age} years old.");
Console.WriteLine($"Salary: ${salary}");
String Interpolation: Using $ before a string and {variableName} inside curly braces is the modern, preferred way to combine text and variables. It's more readable than concatenation.

Console.ReadLine() - Get User Input

The Console.ReadLine() method reads a line of text entered by the user. It always returns a string.

Reading User Input
// Prompt and read input
Console.Write("Enter your name: ");
string name = Console.ReadLine();

Console.Write("Enter your city: ");
string city = Console.ReadLine();

// Display what was entered
Console.WriteLine($"Hello, {name} from {city}!");

Converting String Input to Numbers

Since Console.ReadLine() returns a string, you must convert it to a number type before performing calculations.

Converting Input to int
// Read and convert to integer
Console.Write("Enter your age: ");
string input = Console.ReadLine();
int age = int.Parse(input);

// Shorter version (combined)
Console.Write("Enter a number: ");
int number = int.Parse(Console.ReadLine());
Converting Input to double
// Read and convert to double
Console.Write("Enter distance in miles: ");
double miles = double.Parse(Console.ReadLine());

Console.Write("Enter the price: ");
double price = double.Parse(Console.ReadLine());
Input Validation: If the user enters non-numeric text when you expect a number, .Parse() will crash your program. For learning purposes, we assume users enter valid data. Later, you'll learn about TryParse() for safer input handling.

‚ûï Arithmetic Operators

Arithmetic operators perform mathematical calculations on numeric values.

Binary Operators (Two Operands)

Operator Name Example Result
+ Addition 10 + 5 15
- Subtraction 10 - 5 5
* Multiplication 10 * 5 50
/ Division 10.0 / 4.0 2.5
% Modulus (remainder) 10 % 3 1
Using Arithmetic Operators
int a = 10;
int b = 3;

int sum = a + b;           // 13
int difference = a - b;   // 7
int product = a * b;      // 30
double quotient = a / (double)b;  // 3.333...

Console.WriteLine($"Sum: {sum}");
Console.WriteLine($"Difference: {difference}");
Console.WriteLine($"Product: {product}");
Console.WriteLine($"Quotient: {quotient}");
Console.WriteLine($"Remainder: {remainder}");
Integer Division: When dividing two integers, C# performs integer division and discards the decimal part. 
int result1 = 10 / 3;        // result1 = 3 (not 3.333)
double result2 = 10.0 / 3.0;  // result2 = 3.333...

Unary Operators (One Operand)

Operator Name Example Explanation
++ Increment x++ or ++x Adds 1 to x
-- Decrement x-- or --x Subtracts 1 from x
- Negation -x Changes sign of x
Using Unary Operators
int count = 5;

count++;              // count is now 6 (same as count = count + 1)
count--;              // count is now 5 (same as count = count - 1)

int x = 10;
int y = -x;           // y is -10

Compound Assignment Operators

These operators combine an arithmetic operation with assignment.

Operator Example Equivalent To
+= x += 5 x = x + 5
-= x -= 3 x = x - 3
*= x *= 2 x = x * 2
/= x /= 4 x = x / 4
Calculating Raises Example
const double PERCENT_RAISE = 0.04;
double salary = 50000;

// Year 1: Apply 4% raise
salary *= (1 + PERCENT_RAISE);    // salary = 52,000

// Year 2: Apply another 4% raise
salary *= (1 + PERCENT_RAISE);    // salary = 54,080

Console.WriteLine($"Salary after 2 years: ${salary:N2}");

üîÑ Type Conversion (Casting)

Type conversion (also called casting) is the process of converting a value from one data type to another.

Implicit Casting (Automatic)

C# automatically converts smaller types to larger types when there's no risk of data loss.

Implicit Conversions
int wholeNumber = 10;
double decimalNumber = wholeNumber;  // Automatic: int ‚Üí double
// decimalNumber is now 10.0

int a = 5;
int b = 2;
double result = a / b;  // WARNING: Still integer division!
// result is 2.0 (not 2.5) because 5/2 = 2 before converting to double

Explicit Casting (Manual)

When converting from a larger type to a smaller type, or when precision might be lost, you must explicitly cast.

Explicit Conversions
double price = 19.99;
int dollars = (int)price;  // Explicit cast: double ‚Üí int
// dollars is 19 (decimal part is truncated, not rounded)

// Force floating-point division
int a = 10;
int b = 3;
double result = (double)a / b;  // Cast one operand to double
// result is 3.333... (true division)

// Alternative: multiply by 1.0
double result2 = a * 1.0 / b;  // Forces double calculation
Truncation vs Rounding: Casting a double to an int truncates (cuts off) the decimal part, it doesn't round. 
(int)19.99  // Results in 19, not 20
(int)19.1   // Results in 19

Using Parse() Methods

Convert strings to numeric types using Parse() methods.

String to Number Conversions
string text1 = "42";
string text2 = "3.14";
string text3 = "100.50";

int number = int.Parse(text1);           // 42
double pi = double.Parse(text2);        // 3.14
decimal money = decimal.Parse(text3);   // 100.50

üé® Formatted Output

C# provides powerful string formatting capabilities to control how values are displayed.

String Interpolation with Formatting

Format Syntax: {variable:formatSpecifier}
  • Placeholder: {variable} marks where to insert a value
  • Format specifier: Letter indicating format type (C, N, F, etc.)
  • Precision specifier: Number indicating decimal places

Common Format Specifiers

Specifier Name Example Output
C or C2 Currency {price:C2} $19.99
N or N2 Number (with commas) {salary:N2} 50,000.00
F or F2 Fixed-point {value:F2} 19.99
P or P2 Percent {rate:P2} 45.00%
D or D4 Decimal (integers only) {id:D4} 0042
Currency Formatting
double price = 19.99;
double salary = 50000;

// Currency format (C2 = 2 decimal places)
Console.WriteLine($"Price: {price:C2}");      // Price: $19.99
Console.WriteLine($"Salary: {salary:C2}");   // Salary: $50,000.00

// Without format specifier (default)
Console.WriteLine($"Price: ${price}");      // Price: $19.99
Number Formatting (with commas)
double population = 1234567.89;
double distance = 42.7;

// Number format with 2 decimal places
Console.WriteLine($"Population: {population:N2}");
// Output: Population: 1,234,567.89

Console.WriteLine($"Distance: {distance:N2} km");
// Output: Distance: 42.70 km
Fixed-Point Formatting
double average = 87.65432;

// Fixed-point with different decimal places
Console.WriteLine($"Average: {average:F2}");  // Average: 87.65
Console.WriteLine($"Average: {average:F3}");  // Average: 87.654
Console.WriteLine($"Average: {average:F0}");  // Average: 88 (rounded)
Creating Formatted Tables
string name1 = "Alice";
string name2 = "Bob";
double salary1 = 50000;
double salary2 = 65000;

// Table header
Console.WriteLine("Name        Salary");
Console.WriteLine("--------------------");

// Table rows with aligned columns
Console.WriteLine($"{name1,-10}  {salary1:C2}");
Console.WriteLine($"{name2,-10}  {salary2:C2}");

/* Output:
Name        Salary
--------------------
Alice       $50,000.00
Bob         $65,000.00
*/
Alignment: Use {variable,-10} for left-aligned (10 characters wide) or {variable,10} for right-aligned. This is useful for creating neat columns in console output.

‚öñÔ∏è Comparison Operators

Comparison operators compare two values and return a boolean result (true or false).

Operator Name Example Returns true when...
== Equal to x == y x equals y
!= Not equal to x != y x does not equal y
> Greater than x > y x is greater than y
< Less than x < y x is less than y
>= Greater than or equal x >= y x is greater than or equal to y
<= Less than or equal x <= y x is less than or equal to y
Using Comparison Operators
int age = 25;
int minAge = 18;
double gpa = 3.5;

bool isAdult = age >= minAge;           // true
bool isTeenager = age >= 13 && age <= 19;  // false
bool hasHonors = gpa >= 3.5;          // true

Console.WriteLine($"Is adult? {isAdult}");        // Is adult? True
Console.WriteLine($"Is teenager? {isTeenager}");  // Is teenager? False
Common Mistake: Don't confuse = (assignment) with == (comparison). 
x = 5;     // Assignment: sets x to 5
x == 5;    // Comparison: checks if x equals 5 (returns true/false)

üí° Best Practices

‚úÖ DO

  • Use meaningful variable names (totalPrice not tp)
  • Use constants for fixed values
  • Name constants in ALL_CAPS
  • Use string interpolation ($"{x}")
  • Include comments to explain complex logic
  • Format currency and numbers appropriately
  • Use blank lines to organize code sections

‚ùå DON'T

  • Use single letters except in loops (x, y)
  • Try to change constant values
  • Forget to cast when dividing integers
  • Confuse = with ==
  • Leave magic numbers unexplained
  • Display unformatted money values
  • Write all code in one long block

üéì Key Takeaways

  • Variables store data that can change; constants store fixed values
  • Every variable must have a data type (int, double, string, bool, etc.)
  • Console.ReadLine() always returns a string ‚Äì use Parse() to convert to numbers
  • Use const for values that never change (like conversion rates or tax rates)
  • Arithmetic operators (+, -, *, /, %) perform calculations on numbers
  • Integer division truncates decimals ‚Äì cast to double for true division
  • String interpolation ($"...{variable}...") is the modern way to combine text and values
  • Format specifiers (C2, N2, F2) control how numbers are displayed
  • Comparison operators (==, !=, >, <, >=, <=) compare values and return true/false
  • Always use meaningful variable names and follow naming conventions
