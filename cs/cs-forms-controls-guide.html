Guide #10: More Skills for Windows Forms and Controls

Combo Boxes, List Boxes, Radio Buttons, Check Boxes, Group Boxes, Custom Dialogs, and Multi-Form Apps

🎛️ Additional Windows Forms Controls

Beyond labels, text boxes, and buttons, Windows Forms provides several controls that let users select from predefined options rather than typing free-form input. These controls reduce input errors and make your forms more user-friendly.

📋 ComboBox

A drop-down list that lets the user pick one item. Can also allow typed input depending on its DropDownStyle property. Great for long lists of choices like states or categories.

📜 ListBox

Displays a scrollable list of items. The user clicks to select. Unlike a combo box, the list is always visible — ideal when you want users to see all options at once.

🔘 RadioButton

Used in a group where only one option can be selected at a time. Think "pick one" — like choosing a shirt size (S, M, L, XL).

☑️ CheckBox

An independent on/off toggle. Multiple check boxes can be selected simultaneously. Think "pick any that apply" — like choosing pizza toppings.

📦 GroupBox

A container that visually groups related controls. When radio buttons are placed inside a group box, they form an independent selection group.

Radio buttons vs. check boxes: Radio buttons work as a group — selecting one deselects the others. Check boxes are independent — each one can be toggled on or off without affecting the others.

📋 Working with Combo Boxes

Key Properties and Methods

Property / Method Description
Items The collection of items in the combo box
Items.Add(item) Adds a single item to the end of the list
Items.Clear() Removes all items from the list
Items.Count Returns the number of items in the list
Items[index] Gets or sets the item at a specific index (zero-based)
SelectedIndex Gets or sets the index of the currently selected item (−1 if none)
SelectedItem Gets or sets the currently selected item as an object
Text Gets or sets the text displayed in the text portion of the combo box
DropDownStyle DropDown (editable), DropDownList (selection only), or Simple

Loading Items at Startup

To fill a combo box when the form first appears, add items inside the form's Load event handler. This ensures the list is ready before the user interacts with the form.

Loading a combo box in the Load event

private void OrderForm_Load(object sender, EventArgs e)
{
    // Clear any existing items first
    cboSize.Items.Clear();

    // Add items one at a time
    cboSize.Items.Add("Small");
    cboSize.Items.Add("Medium");
    cboSize.Items.Add("Large");
    cboSize.Items.Add("Extra Large");

    // Optionally select the first item
    cboSize.SelectedIndex = 0;
}

Loading Items with a Loop

private void PaymentForm_Load(object sender, EventArgs e)
{
    // Fill a combo box with years
    int currentYear = DateTime.Now.Year;
    for (int i = 0; i < 10; i++)
    {
        cboYear.Items.Add(currentYear + i);
    }
    cboYear.SelectedIndex = 0;
}

Responding to Selection Changes

The SelectedIndexChanged event fires whenever the user picks a different item from the drop-down list. Use this event to respond to the user's selection.

private void cboCategory_SelectedIndexChanged(object sender, EventArgs e)
{
    string selected = cboCategory.SelectedItem.ToString();
    lblStatus.Text = "You selected: " + selected;
}
Watch the Clear() placement! If you call Items.Clear() inside a loop that also calls Items.Add(), the list gets cleared on every iteration. Move the Clear() call before the loop so it runs only once.

📜 Working with List Boxes

A ListBox works similarly to a combo box, but the items are always visible in a scrollable list. It shares many of the same properties: Items, SelectedIndex, SelectedItem, and Items.Add().

Populating and reading a list box

// Add items to a list box
lstColors.Items.Add("Red");
lstColors.Items.Add("Green");
lstColors.Items.Add("Blue");

// Select the first item by index
lstColors.SelectedIndex = 0;

// Retrieve the selected item
string color = lstColors.SelectedItem.ToString();
Combo box vs. list box: Use a combo box when space is limited and you have many options. Use a list box when you want the user to see all (or most) options at a glance without clicking a drop-down.

🔘 Radio Buttons and Check Boxes

Radio Buttons

Radio buttons let the user choose exactly one option from a group. All radio buttons placed directly on a form (or inside the same group box) belong to the same group — selecting one automatically deselects the others.

Reading radio button selections

private void btnCalculate_Click(object sender, EventArgs e)
{
    decimal shippingCost;

    if (rdoStandard.Checked)
    {
        shippingCost = 5.99m;
    }
    else if (rdoExpress.Checked)
    {
        shippingCost = 12.99m;
    }
    else  // rdoOvernight.Checked
    {
        shippingCost = 24.99m;
    }

    lblShipping.Text = shippingCost.ToString("c");
}

Group Boxes and Independent Groups

If you need multiple independent sets of radio buttons on the same form, place each set inside its own GroupBox. Selecting a radio button in one group box does not affect radio buttons in another group box.

What about radio buttons NOT in a group box?

Radio buttons placed directly on the form (not inside any group box) automatically form their own group. They function as a single selection set — selecting one deselects all others on the form that are not inside a group box.

Check Boxes

Unlike radio buttons, each check box operates independently. Use the Checked property (a bool) to determine whether the box is checked or to set its state in code.

// Reading a check box
bool wantsNewsletter = chkNewsletter.Checked;

// Setting a check box in code
chkAgreeToTerms.Checked = true;

// Using a check box in a calculation
decimal total = subtotal;
if (chkGiftWrap.Checked)
{
    total += 3.50m;
}

⌨️ Setting Tab Order

The tab order determines the order in which controls receive focus when the user presses the Tab key. A logical tab order makes your form easier to navigate, especially for keyboard users.

Two Ways to Set Tab Order

Method How It Works
Properties Window Select each control and set its TabIndex property to a number. Lower numbers receive focus first.
Tab Order View Go to View → Tab Order in the menu. The designer shows numbered badges on each control. Click the controls in the order you want them to receive focus. Click outside the form or press Escape to exit Tab Order view.
Tab Order view is the fastest way to set tab order for multiple controls. Just click each control in sequence and Visual Studio renumbers them automatically.

🏗️ The Program Class and Global Using Directives

The Program Class

Every Windows Forms app has a Program class with a Main() method — this is the entry point of the application. The Main() method calls Application.Run() and passes in a new instance of the startup form, which causes that form to be displayed.

A typical Program.cs file

namespace OrderEntryApp
{
    internal static class Program
    {
        [STAThread]
        static void Main()
        {
            ApplicationConfiguration.Initialize();
            Application.Run(new OrderForm());
        }
    }
}

If you want to change which form appears at startup, change the form name inside Application.Run(). For example, replacing new OrderForm() with new LoginForm() would make the login form appear first.

Global Using Directives

A global using directive makes a namespace available to every file in the project — not just the file where the directive is written. This eliminates the need to add the same using statement at the top of every .cs file.

// In Usings.cs (or any file — the convention is Usings.cs)
global using System;
global using System.Windows.Forms;
global using System.Drawing;

Where do global usings go?

Most developers store global using directives in a file named Usings.cs at the root of the project. You can name this file anything, but Usings.cs is the common convention. The directives apply project-wide regardless of which file they're in.

✏️ Renaming Forms and Refactoring

When you rename a form's file in Solution Explorer, Visual Studio's refactoring feature automatically updates most references to the old form name throughout your code — including the class declaration, the constructor, and calls like Application.Run().

Event handler names are NOT automatically updated. If your form was originally named Form1 and had an event handler called Form1_Load, renaming the form to OrderForm does not rename the handler to OrderForm_Load. You need to use the Rename refactoring command (right-click the method name → Rename, or press F2) to update the handler name and its wiring in the designer-generated code.

🪟 Multi-Form Apps and Custom Dialogs

Modal vs. Modeless Forms

A modal form (also called a dialog) must be closed before the user can interact with any other form in the application. A modeless form allows the user to switch between open forms freely.

Type Method Behavior
Modal form.ShowDialog() Blocks until the dialog is closed; code after ShowDialog() waits
Modeless form.Show() Opens the form and immediately continues executing the next line of code

Displaying a Custom Dialog

Showing a modal dialog and checking the result

private void btnPreferences_Click(object sender, EventArgs e)
{
    // Create an instance of the dialog form
    var prefsDialog = new PreferencesForm();

    // Show it as a modal dialog
    DialogResult result = prefsDialog.ShowDialog();

    // Check how the user closed it
    if (result == DialogResult.OK)
    {
        lblStatus.Text = "Preferences saved.";
    }
    else
    {
        lblStatus.Text = "Preferences cancelled.";
    }
}

The DialogResult Enumeration

When a modal dialog closes, it returns a DialogResult value that tells the calling form how the user dismissed it. You set this on the dialog's button or directly on the form.

Value Typical Use
DialogResult.OK User confirmed the action (e.g., clicked "Save" or "OK")
DialogResult.Cancel User cancelled the action (e.g., clicked "Cancel" or the X button)
DialogResult.Yes User responded "Yes" to a yes/no prompt
DialogResult.No User responded "No" to a yes/no prompt
DialogResult.Abort User chose to abort an operation
DialogResult.Retry User chose to retry an operation

Setting DialogResult on a Button

The quickest way to make a dialog return a result is to set the DialogResult property of a button in the Properties window. When the user clicks that button, the form automatically closes and returns that result — no extra code required.

// Or set it in code inside the dialog form:
private void btnSave_Click(object sender, EventArgs e)
{
    // Validate data, then close with OK result
    this.DialogResult = DialogResult.OK;
}

Passing Data with the Tag Property

The Tag property is available on every form and control. It's typed as object, which means you can store any type of data in it — a string, a number, a custom object, or anything else. This makes it a convenient way to pass data between a form and a custom dialog.

Using Tag to pass data to and from a dialog

// CALLING FORM: Pass data to the dialog via Tag before showing it
var settingsDialog = new SettingsForm();
settingsDialog.Tag = currentTheme;   // store a string
DialogResult result = settingsDialog.ShowDialog();

if (result == DialogResult.OK)
{
    // Retrieve updated data from Tag
    string newTheme = settingsDialog.Tag.ToString();
    ApplyTheme(newTheme);
}

// DIALOG FORM: Read incoming data and update Tag before closing
private void SettingsForm_Load(object sender, EventArgs e)
{
    string theme = this.Tag.ToString();
    cboTheme.Text = theme;
}

private void btnApply_Click(object sender, EventArgs e)
{
    this.Tag = cboTheme.Text;            // store updated value
    this.DialogResult = DialogResult.OK;   // close the dialog
}
Tag holds object. Because Tag is typed as object, you can store anything — but you'll need to cast it back to the correct type when you retrieve it.

💬 Standard Dialogs with MessageBox

The MessageBox.Show() method displays a standard Windows dialog with a message, title, buttons, and an icon. It returns a DialogResult value indicating which button the user clicked.

MessageBox.Show() with full parameters

DialogResult answer = MessageBox.Show(
    "Do you want to save your changes?",       // message text
    "Save Changes",                            // title bar text
    MessageBoxButtons.YesNoCancel,              // which buttons
    MessageBoxIcon.Question,                    // which icon
    MessageBoxDefaultButton.Button1             // default button
);

if (answer == DialogResult.Yes)
{
    SaveData();
}
else if (answer == DialogResult.No)
{
    DiscardChanges();
}
// Cancel → do nothing, return to the form

Common MessageBoxButtons Values

Value Buttons Shown
MessageBoxButtons.OK OK
MessageBoxButtons.OKCancel OK, Cancel
MessageBoxButtons.YesNo Yes, No
MessageBoxButtons.YesNoCancel Yes, No, Cancel
MessageBoxButtons.RetryCancel Retry, Cancel
MessageBoxButtons.AbortRetryIgnore Abort, Retry, Ignore
MessageBoxDefaultButton determines which button is focused when the dialog appears. Button1 is the first button, Button2 is the second, and so on. This controls what happens if the user presses Enter without clicking.

🚪 The FormClosing Event

The FormClosing event fires just before a form closes — whether the user clicked the X button, called this.Close(), or the application is shutting down. You can use this event to confirm the close or cancel it entirely.

Preventing a form from closing

private void OrderForm_FormClosing(object sender,
    FormClosingEventArgs e)
{
    DialogResult answer = MessageBox.Show(
        "Are you sure you want to exit?",
        "Confirm Exit",
        MessageBoxButtons.YesNo,
        MessageBoxIcon.Question,
        MessageBoxDefaultButton.Button2);   // default to "No"

    if (answer == DialogResult.No)
    {
        e.Cancel = true;  // stop the form from closing
    }
}

Setting e.Cancel = true on the FormClosingEventArgs object cancels the close operation and the form remains open. If you don't set it (or set it to false), the form closes normally.

Be careful with always-cancel logic. If your FormClosing handler always sets e.Cancel = true with no way out, the user won't be able to close the form at all! Always provide a path that allows closing.

❌ Common Mistakes to Avoid

Wrong ❌ Right ✓
cboItems.Add("Apples") cboItems.Items.Add("Apples") — must go through the Items collection
Calling Items.Clear() inside the Add loop Call Items.Clear() before the loop, not inside it
Using chkOption.IsChecked Use chkOption.Checked — there is no IsChecked in WinForms
Using form.ShowModal() Use form.ShowDialog() — ShowModal does not exist
Expecting renamed form to update event handler names Use Rename refactoring (F2) to manually update handler names and wiring
Storing global usings in GlobalUsings.cs Convention is Usings.cs — the file name doesn't matter technically, but follow the convention
Forgetting to cast Tag back to the correct type Tag is object — cast it: (string)form.Tag or use .ToString()
Using Application.Run(Form1) (no new) Application.Run(new Form1()) — must pass a new instance
