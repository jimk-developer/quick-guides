Boolean Expressions, Conditionals, Loops, and Pattern Matching in C#

‚úì Boolean Expressions

A Boolean expression evaluates to either true or false. You create Boolean expressions using relational operators.

Relational Operators

Operator Meaning Example Result (if x=5)
== Equal to x == 5 true
!= Not equal to x != 5 false
> Greater than x > 3 true
< Less than x < 3 false
>= Greater than or equal x >= 5 true
<= Less than or equal x <= 5 true
Common Mistake: The single = is the assignment operator, NOT a relational operator. Use == to compare values!

Logical Operators

Use logical operators to combine two or more Boolean expressions into a single expression.

Operator Name Description Example
&& AND True if BOTH are true x > 0 && x < 10
|| OR True if EITHER is true x < 0 || x > 100
! NOT Reverses the value !isValid 
// Combined Boolean expressions
double percent = 0.54;
bool valid = true;

if (percent > 0.50 && valid)   // Both must be true
{
    // This code runs because 0.54 > 0.50 AND valid is true
}

if (percent < 0.25 || !valid)  // Either can be true
{
    // This code does NOT run
}

üîÄ The if Statement

The if statement implements the selection structure - it lets your program choose which code to execute based on a condition.

Basic if Statement

if (subtotal >= 100)
{
    discountPercent = 0.2m;
}

if-else Statement

if (subtotal >= 100)
{
    discountPercent = 0.2m;
}
else
{
    discountPercent = 0.1m;
}

if-else-if Chain

if (subtotal >= 200)
{
    discountPercent = 0.25m;
}
else if (subtotal >= 100)
{
    discountPercent = 0.2m;
}
else if (subtotal >= 50)
{
    discountPercent = 0.1m;
}
else
{
    discountPercent = 0m;
}
Order Matters: In an if-else-if chain, conditions are evaluated from top to bottom. Once a condition is true, the rest are skipped.

‚ùì The Conditional Operator

The conditional operator (? :) is a concise alternative to simple if-else statements.

Syntax

// condition ? valueIfTrue : valueIfFalse
decimal salePrice = (price > 100.0m) ? price * .7m : price * .8m;

if-else Statement

decimal salePrice;
if (price > 100.0m)
    salePrice = price * .7m;
else
    salePrice = price * .8m;

Conditional Operator

decimal salePrice = 
    (price > 100.0m) 
    ? price * .7m 
    : price * .8m;

üîÑ The switch Statement

The switch statement implements the case structure - it's useful when comparing a variable against multiple specific values.

switch (employeeType)
{
    case 1:
        description = "Management";
        break;
    case 2:
        description = "Administration";
        break;
    case 3:
        description = "Marketing";
        break;
    default:
        description = "Development";
        break;
}

switch Statement Rules

  • Each case must end with break (or return)
  • The default case handles unmatched values
  • Cases can be combined: case 1: case 2:

‚ö° Switch Expressions (Modern C#)

Switch expressions provide a more concise syntax than switch statements. They return a value directly.

string employeeDescr = employeeType switch
{
    1 => "Management",
    2 => "Administration",
    3 => "Marketing",
    _ => "Development"       // _ is the discard pattern (default)
};

switch Statement

Uses case and break keywords. Each case contains statements to execute.

switch Expression

Uses => arrows. Returns a value directly. Uses _ for default. More concise.

üéØ Pattern Matching

Pattern matching allows you to test values against patterns and extract information. Use the is keyword for type and value patterns.

Type Pattern with if Statement

var message = data.GetMessage();
if (message is not null && message is string s)
{
    MessageBox.Show(s);   // s is the extracted string
}
else
{
    MessageBox.Show("No message received.");
}

Relational Patterns in switch

decimal discount = quantity switch
{
    1 or 2 => 0,                              // or pattern
    >= 3 and < 10 => orderTotal * .1m,       // and pattern with relational
    >= 10 and < 25 => orderTotal * .2m,
    _ => orderTotal * .3m
};
Pattern Keywords: Use and, or, and not to combine patterns. Use relational operators (>, >=, <, <=) for range checking.

üîÅ Loops (Iteration Structures)

Loops allow you to repeat a block of code multiple times. C# provides three main loop types.

while Loop

Tests the condition before executing the loop body. May execute zero times.

int i = 1;
while (i <= 10)
{
    total += i;
    i++;
}
// If i starts at 11, the loop never executes

do-while Loop

Tests the condition after executing the loop body. Always executes at least once.

int i = 1;
do
{
    total += i;
    i++;
} while (i <= 10);
// Always runs at least once, even if condition is initially false

for Loop

Best when you know the number of iterations. Combines initialization, condition, and increment.

for (int i = 1; i <= 10; i++)
{
    total += i;
}
// Ideal for counting loops

üîÑ while

Condition tested before loop. Use when iterations unknown. May run 0 times.

üîÑ do-while

Condition tested after loop. Always runs at least once.

üî¢ for

Best for counter-controlled loops when you know how many iterations.

Loop Comparison Example

// Calculate future value with interest
decimal futureValue = 1000m;
decimal monthlyRate = .01m;
int months = 10;

// Using for loop (preferred for counting)
for (int i = 0; i < months; i++)
{
    futureValue = futureValue * (1 + monthlyRate);
}

// Using while loop
int i = 0;
while (i < months)
{
    futureValue = futureValue * (1 + monthlyRate);
    i++;
}

‚èπÔ∏è break and continue Statements

Statement Effect Use Case
break Exits the loop immediately (jumps to end) Stop searching when item found
continue Skips to next iteration (jumps to beginning) Skip certain values 
// break example - exit loop early
for (int i = 0; i < items.Length; i++)
{
    if (items[i] == searchValue)
    {
        foundIndex = i;
        break;    // Stop searching, exit loop
    }
}

// continue example - skip to next iteration
for (int i = 0; i < 10; i++)
{
    if (i % 2 == 0)
        continue;  // Skip even numbers
    
    Console.WriteLine(i);  // Only prints odd numbers
}

üì¶ Block Scope

Variables declared inside a block (between { }) are only accessible within that block. This is called block scope.

for (int i = 0; i < 10; i++)
{
    int temp = i * 2;     // temp exists only inside this loop
    total += temp;
}
// Console.WriteLine(temp);  // ERROR! temp doesn't exist here
// Console.WriteLine(i);     // ERROR! i doesn't exist here

if (isValid)
{
    string message = "Valid!";
    lblStatus.Text = message;
}
// message is not accessible here
Block Scope Rule: Variables declared inside if, for, while, or any { } block are not accessible outside that block.

üìä Comparing Control Structures

if-else vs switch

Use if-else When... Use switch When...
Testing ranges or complex conditions Comparing against specific discrete values
You have 2-3 simple conditions You have many possible values to check
Conditions are unrelated All cases test the same variable

Loop Selection Guide

Loop Type Best For
for Known number of iterations, counter-based loops
while Unknown iterations, condition checked first
do-while Must execute at least once (e.g., menu prompts)
