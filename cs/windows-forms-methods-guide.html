Method Signatures, Parameters, Return Values, and Event Wiring

üì¶ Method Basics

A method is a block of code that performs a specific task. Methods help organize code, make it reusable, and easier to maintain.

Method Signature

The signature of a method consists of: the method name and the parameter list (types and order of parameters). The signature uniquely identifies a method.

‚úçÔ∏è Writing Methods

Methods That Return a Value

Use the return statement to send a value back to the calling code. The return type in the declaration must match the type being returned.

Syntax: Method with Return Value

private decimal GetTotal(decimal sub, decimal tax)
{
    decimal total = sub + tax;
    return total;
}

Methods That Don't Return a Value (void)

Use void as the return type when the method performs an action but doesn't return data.

Syntax: Void Method

private void ClearTextBoxes()
{
    txtName.Text = "";
    txtAmount.Text = "";
}

üìû Calling Methods

Calling a Method That Returns a Value

// Store return value in a variable
decimal total = GetTotal(subtotal, taxAmt);

// Or use directly in an expression
lblTotal.Text = GetTotal(subtotal, taxAmt)
    .ToString("c");

Calling a Void Method

// Just call the method - no return value
ClearTextBoxes();

// Can also use 'this' keyword (optional)
this.ClearTextBoxes();
Calling Syntax: To call a void method with no parameters, use: MethodName(); ‚Äî don't include void in the call!

‚û°Ô∏è Expression-Bodied Methods

An expression-bodied method is a shorter way to write a method that contains a single expression. Use the lambda operator (=>) instead of braces and return.

Traditional Method

private decimal GetTotal(decimal sub, decimal tax)
{
    return sub + tax;
}

Expression-Bodied Method ‚úì

private decimal GetTotal(decimal sub, decimal tax) =>
    sub + tax;
Expression-bodied syntax: Replace { return expression; } with => expression;

üîÑ Passing Arguments: By Value vs By Reference

üì• Pass by Value (Default)

A copy of the variable's value is passed. Changes inside the method do NOT affect the original variable.

// Copy passed
Calculate(amount);

üîó Pass by Reference (ref/out)

A reference to the original variable is passed. Changes inside the method DO affect the original variable.

// Reference passed
Calculate(ref amount);

Using the ref Keyword

Use ref when you want to pass a variable by reference. The ref keyword must appear in both the method declaration AND the method call.

// Method declaration with ref parameter
private void DoubleValue(ref int number)
{
    // Changes the original variable
    number = number * 2;
}

// Calling with ref keyword
int value = 5;
DoubleValue(ref value);
// value is now 10

Using the out Keyword

Use out when you want the method to assign a value to the variable. The variable doesn't need to be initialized before the call.

// Method with out parameters
private void GetValues(out int min, out int max)
{
    min = 0;
    max = 100;
}

// Calling with out keyword
GetValues(out int minimum, out int maximum);

‚ùì Optional Parameters

Optional parameters have a default value and can be omitted when calling the method. They must be coded after all required parameters.

Syntax: Optional Parameters

private decimal CalcTotal(decimal sub, decimal rate = 0.08m)
{
    return sub + (sub * rate);
}

// Uses default rate of 0.08
decimal total1 = CalcTotal(100);

// Uses specified rate of 0.10
decimal total2 = CalcTotal(100, 0.10m);
Rule: Optional parameters must be assigned a default value in the declaration, and they must come after required parameters.

üè∑Ô∏è Named Arguments

Named arguments let you specify arguments by parameter name rather than position. Use a colon (:) to separate the name from the value.

Syntax: Named Arguments

// Method declaration
private decimal GetInterest(int years, decimal rate, 
    decimal principal)

// Calling with named arguments (use colon :)
decimal interest = GetInterest(
    years: 5,
    rate: 0.05m,
    principal: 1000m
);

Advantages of Named Arguments

Advantage Description
Any order You can pass arguments in any order
Skip optional params No need to indicate when optional parameters are omitted
Mix styles Can mix positional and named arguments in the same call
Self-documenting Code is clearer about what each argument represents
Note: To use named arguments, you must know the parameter names from the method declaration.

üì¶ Returning Multiple Values with Tuples

A tuple allows a method to return multiple values. Define the tuple return type using parentheses () with named members.

Syntax: Tuple Return Type

// Method that returns a tuple with two values
private (int TotalQty, decimal TotalAmt) GetTotals(int year)
{
    int qty = 100;
    decimal amt = 5000m;
    return (qty, amt);
}

// Using the tuple return value
var totals = GetTotals(2024);
lblQty.Text = totals.TotalQty.ToString();
lblAmt.Text = totals.TotalAmt.ToString("c");
Tuple Syntax: Use parentheses () to define tuple members, NOT brackets [] or braces {}.

An event handler is a method that responds to events like button clicks, text changes, or form loading.

Generating Event Handlers

Event Wiring

Event wiring connects an event to an event handler. This is done automatically when you generate an event handler, but you can also do it manually in code.

Syntax: Manual Event Wiring

// Wire an event to a handler using +=
btnClear.Click += ClearControls;

// The event handler method
private void ClearControls(object sender, EventArgs e)
{
    // Code to handle the event
}
Wiring syntax: controlName.EventName += HandlerMethodName;

One Handler for Multiple Events

You can wire multiple controls to the same event handler. Use the sender parameter to determine which control triggered the event.

// Wire multiple buttons to the same handler
btnOne.Click += NumberButton_Click;
btnTwo.Click += NumberButton_Click;
btnThree.Click += NumberButton_Click;

// Handler that works for all buttons
private void NumberButton_Click(object sender, 
    EventArgs e)
{
    // Cast sender to Button type
    Button btn = (Button)sender;
    // Use the button's text
    txtDisplay.Text += btn.Text;
}

üîß Refactoring: Extract Method

Refactoring restructures existing code without changing its behavior. The Extract Method feature creates a new method from selected code.

How to Use Extract Method

  1. Select the code you want to extract
  2. Right-click and choose Quick Actions and Refactorings
  3. Select Extract Method
  4. Enter a name for the new method
  5. Visual Studio creates the method and replaces the selected code with a method call
Keyboard shortcut: Select code, then press Ctrl+. to access Quick Actions.
