Guide #8: How to Use Arrays and Collections

One-Dimensional Arrays, Rectangular Arrays, Lists, Queues, Stacks, and More

üì¶ Array Fundamentals

An array is a data structure that stores multiple values of the same type. When you create an array, you're creating an instance of the Array class. Arrays use zero-based indexing.

Array Index Visualization

10[0]
20[1]
30[2]
40[3]
50[4]
Zero-Based Indexing: The first element is at index [0], the second at [1], etc. To access the 9th element, use index [8].

Declaring and Initializing Arrays

Syntax: One-Dimensional Array

// Declare with size (elements initialized to default)
int[] scores = new int[5];

// Declare and initialize with values
string[] names = {"Alice", "Bob", "Carol"};

// Decimal array - use 'm' suffix for literals
decimal[] prices = {19.99m, 29.99m, 39.99m};
Decimal literals: When initializing a decimal array, you must use the m suffix on each value (e.g., 27.5m).

Accessing Array Elements

string[] colors = {"Red", "Green", "Blue"};

// Get element by index
string first = colors[0];   // "Red"
string second = colors[1];  // "Green"

// Set element by index
colors[2] = "Yellow";       // Changes "Blue" to "Yellow"

// Get the number of elements
int count = colors.Length;   // 3

üìä Rectangular Arrays (2D Arrays)

A rectangular array stores data in rows and columns, like a table. Use a comma inside the brackets to separate dimensions.

Syntax: Rectangular Array

// Declare with dimensions [rows, columns]
string[,] grid = new string[3, 4];  // 3 rows, 4 columns

// Declare and initialize
int[,] matrix = {
    {1, 2, 3},
    {4, 5, 6},
    {7, 8, 9}
};

Accessing Rectangular Array Elements

decimal[,] sales = {
    {100.0m, 150.0m},   // Row 0
    {200.0m, 250.0m},   // Row 1
    {300.0m, 350.0m}    // Row 2
};

// Access: array[row, column]
decimal value = sales[1, 0];  // Row 1, Col 0 = 200.0

// Get length of a dimension
int rows = sales.GetLength(0);    // 3 rows
int cols = sales.GetLength(1);    // 2 columns
Row first, then column: To access row 4, column 2 of array vendors, use vendors[3, 1] (remember zero-based indexing!).

üîÑ for Loop vs foreach Loop

üî¢ for Loop

Use when you need the index, want to modify elements, or need to iterate in a specific order/range.

for (int i = 0; i < arr.Length; i++)
{
    arr[i] = arr[i] * 2;
}

üîÅ foreach Loop

Use when you just need to read each element. Simpler syntax, but you can't modify elements or access the index.

foreach (int num in arr)
{
    Console.WriteLine(num);
}

Loop Syntax Comparison

Feature for Loop foreach Loop
Access index Yes - use i No
Modify elements Yes - arr[i] = value No - read-only
Syntax keyword for foreach ... in
Uses Length Yes - i < arr.Length No - automatic
foreach syntax: foreach (type variable in collection) ‚Äî Note the keyword in, not a colon!

üõ†Ô∏è The Array Class

The Array class provides static methods for common array operations like sorting and searching.

Method Description Example
Array.Sort() Sorts array in ascending order (modifies original) Array.Sort(numbers);
Array.Reverse() Reverses the order of elements Array.Reverse(names);
Array.BinarySearch() Finds index of value in sorted array int i = Array.BinarySearch(arr, 5);
Array.Copy() Copies elements to another array Array.Copy(src, dest, length); 
int[] values = {5, 2, 8, 1, 9};

// Sort the array (modifies original)
Array.Sort(values);
// values is now {1, 2, 5, 8, 9}

// Search for a value (array must be sorted first!)
int index = Array.BinarySearch(values, 5);
// index is 2 (position of 5 in sorted array)
BinarySearch requires sorted array: Always call Array.Sort() before Array.BinarySearch(). BinarySearch returns the index of the found value, not a boolean.

‚ùì Null-Conditional Operator

The null-conditional operator (?. or ?[]) safely accesses members or elements when the object might be null. It prevents NullReferenceException.

Syntax: Null-Conditional with Arrays

string[] names = null;

// Without null-conditional - throws exception!
// string first = names[0];

// With null-conditional - returns null safely
string first = names?[0];  // first is null

How It Works

Scenario Result
Array is null Returns null, no exception
Array exists but element is null Returns null (the element's value)
Array and element both exist Returns the element's value
Key benefits: Prevents NullReferenceException, stops further operations if null is found, and works with both value types and reference types.

üéØ List Patterns (Pattern Matching)

List patterns let you match arrays or lists against specific patterns of elements. Use brackets [] to define the pattern.

int[] numbers = {1, 2, 3};

// Match exact sequence
if (numbers is [1, 2, 3])
    Console.WriteLine("Exact match!");

// Match with wildcards
if (numbers is [1, _, _])
    Console.WriteLine("Starts with 1");

// Match first and last, ignore middle
if (numbers is [var first, .., var last])
    Console.WriteLine($"First: {first}, Last: {last}");
Pattern elements: Use _ to match any single element, .. to match zero or more elements, and var name to capture a value.

üìö Collections Overview

Collections are more flexible than arrays - they can grow and shrink dynamically. There are two types: typed (generic) and untyped.

‚úÖ Typed Collections (Generic)

Use generics to specify data type. Found in System.Collections.Generic. No casting required, compile-time type checking.

List<int> nums = new List<int>();

‚ö†Ô∏è Untyped Collections

Store objects of any type. Found in System.Collections. Require casting, runtime type checking only.

ArrayList items = new ArrayList();
Prefer typed collections: They reduce runtime errors and eliminate the need for casting when retrieving elements.

üìã List<T> Collection

The List<T> is the most commonly used typed collection. It's like a dynamic array that can grow and shrink.

Creating and Using a List

// Create a typed list of integers
List<int> scores = new List<int>();

// Add items to end of list
scores.Add(85);
scores.Add(92);
scores.Add(78);

// Insert at specific index
scores.Insert(1, 88);  // Insert 88 at index 1

// Access by index (like array)
int first = scores[0];  // 85

// Get number of items
int count = scores.Count;

// Remove items
scores.Remove(78);       // Remove by value
scores.RemoveAt(0);      // Remove by index
Method/Property Description
Add(item) Adds item to end of list
Insert(index, item) Inserts item at specified index
Remove(item) Removes first occurrence of item
RemoveAt(index) Removes item at specified index
Count Number of items in list
Contains(item) Returns true if item exists
Clear() Removes all items

üî§ SortedList<TKey, TValue>

A SortedList stores key-value pairs, automatically sorted by key. Each element consists of a key and its associated value.

// Create sorted list: key=string, value=decimal
SortedList<string, decimal> products = 
    new SortedList<string, decimal>();

// Add key-value pairs (auto-sorted by key)
products.Add("Laptop", 999.99m);
products.Add("Mouse", 29.99m);
products.Add("Keyboard", 79.99m);

// Access value by key
decimal price = products["Laptop"];  // 999.99

// Iterate using KeyValuePair
foreach (KeyValuePair<string, decimal> item in products)
{
    Console.WriteLine($"{item.Key}: {item.Value}");
}
Keys don't need to be added in order: SortedList automatically maintains sorted order by key, regardless of insertion order.

üì§ Queue and Stack

üì• Queue<T> ‚Äî FIFO

First-In, First-Out ‚Äî like a line at a store. First item added is first item removed.

Queue<string> line = new();
line.Enqueue("First");
line.Enqueue("Second");
string next = line.Dequeue();
// next = "First"

üìö Stack<T> ‚Äî LIFO

Last-In, First-Out ‚Äî like a stack of plates. Last item added is first item removed.

Stack<string> plates = new();
plates.Push("First");
plates.Push("Second");
string top = plates.Pop();
// top = "Second"
Collection Add Method Remove Method Peek Method Order
Queue Enqueue() Dequeue() Peek() FIFO
Stack Push() Pop() Peek() LIFO
Peek vs Dequeue/Pop: Peek() returns the next item without removing it. Dequeue() and Pop() return the item and remove it.

üìä Collection Types Comparison

Collection Type Description Use Case

Stack<T> Typed LIFO (last-in, first-out) Undo operations, parsing
ArrayList Untyped Dynamic array, stores objects Legacy code (avoid)
