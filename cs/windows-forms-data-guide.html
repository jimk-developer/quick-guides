Guide #4: How to Work with Numeric and String Data

Variables, Data Types, Operators, and Type Conversion in C#

üì¶ Variables and Constants

Variables and constants are used to store data in your applications. Understanding the difference between them is fundamental to programming.

üìù Variable

Stores data that can change as the program executes. Use camelCase naming (e.g., totalAmount, firstName).

üîí Constant

Stores data that does not change as the program executes. Use PascalCase naming (e.g., TaxRate, MaxValue).

Declaring Variables

// Declare and initialize in one statement
int quantity = 10;
decimal price = 19.99m;
string name = "John";
bool isValid = true;

// Declare first, initialize later
int count;
count = 5;

Declaring Constants

// Use the const keyword - must be initialized when declared
const decimal TaxRate = 0.075m;
const int MaxItems = 100;
const string CompanyName = "Acme Corp";

üéØ Common Data Types

Type Description Example Category
int Whole numbers (no decimals) 42, -17, 0 Value
double Floating-point numbers (less precise) 3.14, -0.5 Value
decimal High-precision numbers (best for money) 19.99m, 100.00m Value
bool True or false value true, false Value
string Text (sequence of characters) "Hello", "" Reference
Best Practice: Use decimal for monetary values because it's the most precise type for financial calculations. Never use double for money!

Value Types vs Reference Types

Value Type

Stores actual data directly. Defined by a structure. Examples: int, double, decimal, bool

Reference Type

Stores a reference (pointer) to data in memory. Defined by a class. Examples: string, arrays, objects

‚ú® Literals and Special Values

Literal Values

A literal value is a value that is coded directly into your source code.

// Numeric literals
int count = 42;              // integer literal
double rate = 3.14;           // double literal
decimal price = 19.99m;       // decimal literal (note the 'm' suffix)

// String literal
string greeting = "Hello, World!";

// Boolean literals
bool isActive = true;
bool isComplete = false;

Special String Values

String Literal Types

// Regular string literal

// Raw string literal (C# 11+) - use """

‚ûï Arithmetic Operators

Binary Operators

Binary operators work with two operands.

Operator Name Example Result
+ Addition 5 + 3 8
- Subtraction 5 - 3 2
* Multiplication 5 * 3 15
/ Division 15 / 3 5
% Modulus (remainder) 17 % 5 2

Unary Operators

Unary operators work with one operand. The increment (++) and decrement (--) operators are NOT binary operators.

int x = 5;
x++;        // x is now 6 (increment)
x--;        // x is now 5 (decrement)

// Prefix vs Postfix
int a = 5;
int b = ++a;    // a=6, b=6 (increment first, then assign)
int c = a++;    // c=6, a=7 (assign first, then increment)

Compound Assignment Operators

int total = 100;
total += 50;     // Same as: total = total + 50;  ‚Üí 150
total -= 25;     // Same as: total = total - 25;  ‚Üí 125
total *= 2;      // Same as: total = total * 2;   ‚Üí 250
total /= 5;      // Same as: total = total / 5;   ‚Üí 50

üìä Order of Precedence

When an expression has multiple operators, C# evaluates them in a specific order:

Priority Operators Description
1 (Highest) ( ) Parentheses - evaluated first
2 ++ -- Increment, Decrement
3 * / % Multiplication, Division, Modulus
4 (Lowest) + - Addition, Subtraction
Use parentheses to override the default order of precedence and make your code clearer! 
// Without parentheses: multiplication happens first
int result = 5 + 3 * 2;       // = 5 + 6 = 11

// With parentheses: addition happens first
int result = (5 + 3) * 2;     // = 8 * 2 = 16

‚ö†Ô∏è Integer Division

When you divide two integers, the result is also an integer (the decimal part is truncated).

int i = 4;
int j = 8;

decimal x = i / j;           // x = 0 (not 0.5!) - integer division

// To get decimal result, cast at least one operand
decimal x = (decimal)i / j;  // x = 0.5 ‚úì
decimal x = (decimal)i / (decimal)j;  // Also works
Common Mistake: Dividing two integers always gives an integer result, even if you assign it to a decimal variable. You must cast BEFORE the division occurs!

üé≤ Generating Random Numbers

// Create a Random object
Random number = new Random();

// Generate random integer from 1 to 10 (inclusive)
int i = number.Next(1, 11);    // Note: upper bound is exclusive!

// Generate random double from 0.0 to less than 1.0
double d = number.NextDouble();
Remember: The Next(min, max) method's upper bound is exclusive. To get numbers 1-10, use Next(1, 11).

üìù Working with Strings

String Concatenation

Use the + operator to concatenate (join) strings together.

string firstName = "John";
string lastName = "Doe";
string fullName = firstName + " " + lastName;  // "John Doe"

// Append to existing string
string cityState = "Milwaukee";
cityState += ", " + "Wisconsin";  // "Milwaukee, Wisconsin"

String Interpolation

string name = "Alice";
int age = 25;
string message = $"Hello, {name}! You are {age} years old.";

üîÑ Type Conversion

Implicit vs Explicit Casting

Implicit Casting

Automatic conversion from less precise to more precise type. No data loss possible.

int i = 5;
double d = i;  // OK

Explicit Casting

Manual conversion from more precise to less precise type. Potential data loss.

double d = 9.7;
int i = (int)d;  // i = 9

Conversion Methods Comparison

Method Use Case Example
(type) cast Convert between numeric types (int)3.7 ‚Üí 3
ToString() Convert any type to string 42.ToString() ‚Üí "42"
Parse() Convert string to numeric (throws error if invalid) int.Parse("42") ‚Üí 42
TryParse() Safely convert string to numeric (returns bool) int.TryParse("42", out x)
Convert class Convert between various types Convert.ToInt32("42") 
// ToString() - convert to string with formatting
decimal total = 1234.56m;
string amount = total.ToString("c");  // "$1,234.56" (currency format)

// Parse() - string to number (throws exception if invalid)
int num = int.Parse("42");

// TryParse() - safe conversion (returns true/false)
if (int.TryParse(txtInput.Text, out int result))
{
    // Conversion succeeded, use result
}

// Convert class methods
int x = Convert.ToInt32("42");
decimal d = Convert.ToDecimal(txtAmount.Text);

üî≠ Variable Scope

Scope determines where a variable can be accessed in your code.

Class Scope (Field)

Declared at class level, accessible from any method in the class. Also called instance variables or fields.

Method Scope (Local)

Declared inside a method, accessible only within that method. Also called local variables. 

public partial class Form1 : Form
{
    // Class scope - accessible from any method
    private decimal totalSales = 0;
    
    private void btnCalculate_Click(...)
    {
        // Method scope - only accessible in this method
        decimal price = 19.99m;
        totalSales += price;  // Can access class-level variable
    }
    
    private void btnReset_Click(...)
    {
        totalSales = 0;       // Can access class-level variable
        // price = 10;        // ERROR! price not accessible here
    }
}

üìã Enumerations

An enumeration defines a set of related constants. It creates a custom data type with a fixed set of valid values.

// Define an enumeration
enum DayOfWeek
{
    Sunday,     // 0
    Monday,     // 1
    Tuesday,    // 2
    Wednesday,  // 3
    Thursday,   // 4
    Friday,     // 5
    Saturday    // 6
}

// Use the enumeration
DayOfWeek today = DayOfWeek.Wednesday;

if (today == DayOfWeek.Saturday || today == DayOfWeek.Sunday)
{
    lblMessage.Text = "It's the weekend!";
}

‚ùì Nullable Types

Nullable types allow value types to hold null in addition to their normal values.

Declaring Nullable Types

// Add ? after the type to make it nullable
int? quantity = null;       // Can be null or an integer
decimal? price = 19.99m;    // Can be null or a decimal
bool? isActive = null;      // Can be null, true, or false

Null Operators

Operator Name Purpose
?? Null-coalescing Provide default value if null
??= Null-coalescing assignment Assign only if null
! Null-forgiving Suppress null warnings 
// Null-coalescing operator (??) - provide default if null
decimal? paymentAmount = null;
decimal payment = paymentAmount ?? 9.99m;  // payment = 9.99

// If paymentAmount had a value:
paymentAmount = 25.00m;
payment = paymentAmount ?? 9.99m;          // payment = 25.00

// Null-forgiving operator (!) - tells compiler "trust me, not null"
string? name = GetName();
string displayName = name!;  // Suppresses null warning
Null-forgiving operator (!) is used to suppress null reference compiler warnings when you're certain a value won't be null.
