üìÅ Creating a New Project

To create a new Windows Forms application, you'll use the New Project dialog box.

  1. Open Visual Studio and select File ‚Üí New ‚Üí Project
  2. Select a Template - Choose "Windows Forms App (.NET)" from the available templates. A template specifies the type of project you want to create.
  3. Configure Your Project - Enter a name for your project and choose a location to save it
  4. Create the Project - Click Create to generate the solution, project, and form files
Understanding Solutions: A solution contains one or more projects, and each project can contain multiple forms. Visual Studio creates a solution file (.sln) and project files automatically.

üè∑Ô∏è Essential Properties

Every form and control has properties that determine its appearance and behavior. Two properties are common to both forms and controls:

Name Property

The Name property specifies the name you use to refer to the control in your C# code.

  • Used in code to reference the control
  • Should follow naming conventions
  • Must be unique within the form
  • Default names: textBox1, button1, etc.

Text Property

The Text property determines the text displayed in or on the control.

  • For forms: appears in the title bar
  • For labels: the displayed text
  • For buttons: the button caption
  • For text boxes: the content

Additional Control Properties

Enabled Property Boolean

Determines whether a control is enabled or disabled. When Enabled is set to false, the control appears grayed out and users cannot interact with it.

// Disable a button
btnCalculate.Enabled = false;

// Enable it later
btnCalculate.Enabled = true;

ReadOnly Property TextBox Only

For TextBox controls, the ReadOnly property prevents users from entering or editing data while still allowing them to select and copy the text. The control appears with a gray background.

// Make a text box read-only for displaying results
txtResult.ReadOnly = true;

TextAlign Property Alignment

Sets the alignment for text displayed in a control. Common values include Left, Center, and Right.

// Right-align text in a text box (useful for numbers)
txtAmount.TextAlign = HorizontalAlignment.Right;

‚å®Ô∏è Tab Order

The tab order determines the order in which controls receive focus when the user presses the Tab key. This is crucial for keyboard navigation and accessibility.

How Tab Order Works

  • Each control has a TabIndex property (0, 1, 2, etc.)
  • Controls receive focus in ascending TabIndex order
  • Lower numbers receive focus first
  • Controls with the same TabIndex follow their order in the form

Setting Tab Order

There are two ways to set the tab order:

Method 1: View Tab Order

  1. Select View ‚Üí Tab Order
  2. Numbers appear on each control
  3. Click controls in desired order
  4. Click View ‚Üí Tab Order again to exit

Method 2: Properties Window

  1. Select a control
  2. Find TabIndex property
  3. Enter the desired number
  4. Repeat for each control

TabStop Property Boolean

To prevent a control from receiving focus when the user presses Tab, set its TabStop property to false. This is useful for read-only text boxes or labels.

// Remove from tab order
txtOutput.TabStop = false;

üîë Access Keys

Access keys let users activate a control by pressing Alt plus another key. They provide quick keyboard shortcuts for common actions.

Creating Access Keys

To create an access key, place an ampersand (&) before the letter you want as the access key in the control's Text property.

Text Property Value Displayed Text Access Key
&Calculate Calculate Alt + C
E&xit Exit Alt + X
&Invoice number Invoice number Alt + I
&Save Save Alt + S
Access Keys for TextBoxes: TextBox controls cannot display access keys directly. To create an access key for a text box, set the access key in the Text property of the label immediately before the text box in the tab order. When the user presses the access key, focus moves to the next control (the text box).
Remember: The ampersand (&) character is used to define access keys. To display a literal ampersand in text, use two ampersands (&&).

‚èé Default Buttons

Forms have two special button properties that respond to keyboard shortcuts:

AcceptButton Property

The button that's activated when the user presses the Enter key.

Typically set to the primary action button like "OK" or "Calculate".

Set in: Form's Properties

CancelButton Property

The button that's activated when the user presses the Esc key.

Typically set to "Cancel" or "Exit" buttons.

Set in: Form's Properties

Setting Default Buttons: Select the form (click on the form's title bar or background), then find the AcceptButton and CancelButton properties in the Properties window. Select the appropriate button from the dropdown list.

üé® Designing the Form

Follow these steps to design a professional-looking form:

  1. Add Controls from Toolbox - Drag controls from the Toolbox onto the form, or double-click a control to add it
  2. Position and Size Controls - Use the mouse to move and resize controls. Hold Ctrl and click to select multiple controls
  3. Align Controls - Select multiple controls, then use Format ‚Üí Align to align them. Alignments are made relative to the primary control (shown with white handles)
  4. Set Properties - Use the Properties window to set Name, Text, and other properties for each control
  5. Set Tab Order - Arrange the tab order logically from top-left to bottom-right
  6. Add Access Keys - Define access keys for buttons and labels
  7. Set Default Buttons - Configure AcceptButton and CancelButton properties on the form

Selecting Multiple Controls

To select two or more controls, hold down the Ctrl key while clicking on each control. You can also draw a selection rectangle around multiple controls.

Primary Control: When you select multiple controls, the last control clicked becomes the primary control (shown with white handles instead of black). Alignment and sizing operations are made relative to the primary control.

‚ñ∂Ô∏è Running Your Form

When you run a form after designing it in the Form Designer (but before adding any code):

  • The form is displayed with all controls visible
  • You can type in text boxes
  • Tab navigation works
  • Access keys work
  • Buttons don't cause anything to happen (no event handlers yet)
  • The form can be moved, minimized, maximized, and closed
Press F5 to run your application in debug mode, or Ctrl+F5 to run without debugging.

üìù Renaming Files

Use the Solution Explorer to rename form, project, or solution files:

  1. Open Solution Explorer (View ‚Üí Solution Explorer)
  2. Right-click on the file you want to rename
  3. Select "Rename" from the context menu
  4. Type the new name and press Enter
  5. If prompted, allow Visual Studio to update all references
Important: When renaming form files, Visual Studio will offer to rename all references. Click "Yes" to keep your code synchronized with the new filename.
