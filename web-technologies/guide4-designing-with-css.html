🎨 Why CSS Exists

HTML provides the structural foundation of a webpage. It says "this is a heading", "this is a paragraph", "this is a list". CSS provides the formatting. It says "headings should be dark green", "paragraphs should use a 16-pixel font", "lists should have extra spacing between items."

Keeping these two concerns separate is one of the most important ideas in modern web design.

Separation of Content and Style

📄 Content (HTML)

The text, images, and structure that the page is fundamentally about. Authors and content editors work here.

🎨 Style (CSS)

The visual presentation: colors, fonts, spacing, layout. Designers work here.

🔄 Reskin Without Rewriting

Because style lives in a separate file, you can completely redesign a site by editing one CSS file. No HTML changes are needed. This is called reskinning.

♻️ Consistency Across Pages

One CSS file styles every page in the site. Update it once and every page updates instantly.

What CSS Stands For

CSS stands for Cascading Style Sheets. The W3C developed CSS as the official way to format webpages. The "cascading" part refers to how multiple style rules combine and resolve when they target the same element. We'll cover that in detail when we look at precedence.

📝 The Anatomy of a CSS Rule

A style rule (also called a rule-set) is the basic building block of CSS. Every rule has the same structure: a selector followed by one or more declarations in curly braces. Each declaration is a property and a value separated by a colon and ending in a semicolon.

h1 { color: darkgreen; }
Selectorwhich element(s) to style
Propertywhich aspect to change
Valuethe new setting
Punctuation{ } : ; structure the rule

Multiple Declarations

A single rule can set as many properties as you want. Each declaration ends with a semicolon. Indenting and using one declaration per line is a convention that makes rules easier to read.

A Rule with Several Declarations

h1 {
    color: darkgreen;
    font-family: Georgia, serif;
    font-size: 2rem;
    text-align: center;
}
HTML attributes vs. CSS properties. These two phrases sound similar but mean different things. An HTML attribute is set inside a tag (like src="logo.png"). A CSS property is set inside a rule and styles an element. A page can have both at the same time.

3️⃣ The Three Types of Styles

CSS can be applied to a webpage in three different ways. Each has a use case, and when more than one applies to the same element, the browser follows a strict order of precedence.

1. Inline Styles

An inline style is written directly on an HTML element using the style attribute. It affects only that one element.

An Inline Style

<p style="color: darkgreen; font-weight: bold;">
    Welcome to our site.
</p>

Inline styles are quick to write but rarely used in real projects. They mix style with content, can't be reused, and have to be edited everywhere if the design changes.

2. Embedded Styles

An embedded style sheet (also called an internal style sheet) lives inside the HTML file itself, in a <style> element placed in the <head>. The rules apply to that one HTML page.

An Embedded Style Sheet

<head>
    <title>Prairie Bakery</title>
    <style>
        h1 { color: darkgreen; }
        p { font-size: 1.1rem; }
    </style>
</head>

Embedded styles are useful for one-off pages or for testing during development. They still mix design and structure, but at least the rules are reusable across the page.

3. External Style Sheets

An external style sheet (also called a linked style sheet) is a separate file with the .css extension that contains nothing but CSS rules. The file is linked from each HTML page that should use it.

An External CSS File: styles.css

/* Site-wide styles for Prairie Bakery */

body {
    font-family: Georgia, serif;
    color: #333;
}

h1 {
    color: darkgreen;
    text-align: center;
}
External style sheets are the professional standard. One file controls the design of the entire site. New pages just need to be linked, and global changes happen in one place.

Style Precedence

When multiple style sources target the same element, the closer the style is to the element, the higher its priority. Inline beats embedded, embedded beats external.

Order of Precedence (highest first)

  1. Inline styles attached directly to an element with the style attribute.
  2. Embedded styles inside a <style> block in the page's head.
  3. External styles from a linked CSS file.
  4. Browser defaults for any property no one has set.

Two more rules apply when rules tie: more specific selectors win over general ones, and the rule that appears later in the file wins over earlier ones.

Inheritance

Inheritance means that some CSS properties pass automatically from an element to the elements inside it. Set the font on body and every paragraph, heading, and list item inside the body inherits that font, unless overridden by a more specific rule. Color and font properties are inherited; border and padding are not.

🔗 Linking an HTML Page to a CSS File

To use an external stylesheet, every HTML page that needs it must include a <link> element inside the <head>.

The Link Element

<head>
    <meta charset="UTF-8">
    <title>Prairie Bakery</title>
    <link rel="stylesheet" href="css/styles.css">
</head>

The <link> element uses two attributes:

Attribute Purpose Typical Value
rel Describes the relationship between this document and the linked file stylesheet
href The path to the CSS file css/styles.css
Confirm your styles loaded. If a page looks unstyled after you link a CSS file, the most common causes are: a typo in the href, the CSS file isn't in the path you specified, or a syntax error in the CSS file is preventing rules from being read.

🎯 Selectors

A selector is the part of a rule that decides which elements get styled. CSS provides several kinds of selectors, each with its own use.

Element Selectors

The simplest selector is the name of an HTML element. The rule applies to every instance of that element on the page.

Element Selector

/* Every paragraph on the page */
p {
    line-height: 1.6;
}

ID Selectors

An id selector targets the single element with a matching id attribute. In CSS, an id selector starts with a hash sign (#).

ID Selector

/* HTML: <div id="container"> ... </div> */

#container {
    max-width: 960px;
    margin: 0 auto;
}

Because each id is unique on a page, an id selector matches exactly one element.

Class Selectors

A class selector targets every element that has a matching class attribute. In CSS, a class selector starts with a period (.).

Class Selector

/* HTML: <a class="action" href="#">Order Now</a> */

.action {
    background-color: darkgreen;
    color: white;
    padding: 10px 20px;
    border-radius: 6px;
}

Any number of elements can share a class, which makes class selectors ideal for styling buttons, cards, or any pattern that repeats.

Descendant Selectors

A descendant selector styles an element only when it appears inside another element. Write the parent selector, a space, and then the child selector.

Descendant Selector

/* Only paragraphs inside <main> get this style */

main p {
    font-size: 1.1rem;
}

ID vs. Class: When to Use Which

🎯 ID Selector (#name)

Use for: a unique element that appears once on the page, such as the page wrapper, the main navigation, or a specific section.

Rule: only one element can have a given id.

🏷️ Class Selector (.name)

Use for: a style pattern that you'll reuse on multiple elements, such as a button style, a card style, or a callout.

Rule: any number of elements can share the same class.

Class names have rules. Any word can be a class name as long as it doesn't contain spaces. Stick to lowercase letters, numbers, and hyphens for consistency. call-to-action is a typical class name; Call To Action is not.

Font Stacks and Fallbacks

Not every computer has every font. To handle this, the font-family property accepts a list of fonts called a font stack. The browser tries each font in order and uses the first one available. The last item should be a generic family (serif, sans-serif, monospace) that always works as a final fallback.

1. Try 'Source Sans Pro' (custom web font)
↓ if not available
2. Try Arial (system font)
↓ if not available
3. Use sans-serif (always works)

A Font Stack with Fallbacks

body {
    font-family: 'Source Sans Pro', Arial, sans-serif;
}

About 140 color names work everywhere, such as red, steelblue, tomato. Easy to read but limited.

A six-digit code in pairs of red, green, and blue channels. Each channel ranges from 00 (none) to FF (full). Most precise control over exact colors.

Hue (color wheel angle), saturation, and lightness. More intuitive when adjusting a color by feel.

🧱 Block Elements vs. Inline Elements

Every HTML element is either a block element or an inline element. Knowing the difference is essential to layout and styling.

Block Elements

A block element starts on a new line and stretches to fill the full width available. Stacking block elements creates the vertical structure of a page. Common examples: div, p, h1 through h6, header, nav, main, section, article, footer, ul, ol, li.

Inline Elements

An inline element flows along with the surrounding text. It takes only as much width as its content needs and does not start on a new line. Common examples: span, a, img, strong, em, code.

Block elements stack vertically:
<p> This is a paragraph </p>
<p> Another paragraph below </p>
<p> And a third </p>
Inline elements flow with the text:

Visit our <a>menu</a> page or read about <strong>our story</strong> on the <a>about us</a> page.

The span Element

The span element is to inline elements what div is to block elements. It is a generic inline container with no meaning of its own. Use a <span> when you need to apply CSS to a piece of inline text but no semantic tag fits.

Highlighting Words with span

<!-- HTML -->
<p>Our bread is baked
   <span class="highlight">fresh daily</span>.</p>

/* CSS */
.highlight {
    background-color: yellow;
    font-weight: bold;
}

📦 The CSS Box Model

Every element on a webpage is a rectangle, even when it doesn't look like one. The CSS box model describes how four layers wrap that rectangle: content, padding, border, and margin.

margin
border
padding
content
Layer What It Is
Content The actual text, image, or other inner material of the element.
Padding Space between the content and the border. Padding is inside the box.
Border A line drawn around the padding. Can have a width, style, color, and rounded corners.
Margin Space between this element and surrounding elements. Margin is outside the box.

Common Box Properties

Property Effect Example
margin Space outside the border margin: 20px;
padding Space inside the border padding: 10px 15px;
border-style Type of border line solid, dashed, dotted
border-width Thickness of the border 2px, medium
border-color Color of the border #0E7C4A
border-radius Roundness of the corners 8px, 50%
box-shadow Drop shadow around the box 0 4px 8px rgba(0,0,0,0.1)

Putting Box Properties Together

.card {
    margin: 20px;
    padding: 15px 20px;
    border: 2px solid #0E7C4A;
    border-radius: 8px;
    box-shadow: 0 4px 10px rgba(0,0,0,0.1);
}
The shorthand pattern. When you write padding: 10px 15px;, the first value is top and bottom, the second is left and right. With four values (10px 15px 20px 5px), they go clockwise from the top: top, right, bottom, left.

🌊 Float and Clear

Before modern layout systems like Flexbox and Grid, the float property was the workhorse of CSS layout. It is still useful for wrapping text around an image or pulling an element to one side of its container.

The float Property

The float property moves an element to the left or right edge of its parent. Surrounding content flows around the floated element on the other side.

Floated Left

Text flows around a floated element on the opposite side. Notice how this paragraph wraps to the right of the box on the left. Float was the standard layout technique on the web for years before Flexbox and Grid arrived. Today, floats are mostly used for the specific case of wrapping text around an image, which is what they were originally invented to do.

Value Effect
left Pushes the element to the left; surrounding content wraps to the right.
right Pushes the element to the right; surrounding content wraps to the left.
none No floating. Default value.
inherit Use whatever float value the parent element has.

The clear Property

After a floated element, content keeps flowing around it. Sometimes you want the next element to start below the float instead. The clear property forces an element below any preceding floats.

Clearing a Float

.gallery img {
    float: left;
}

footer {
    clear: both; /* start below all floated images */
}

Centering with text-align and margin

📝 Center Text

Use text-align: center; on a block element to center the text inside it.

📦 Center a Block

Use margin: 0 auto; on a block element with a defined width to center the whole block within its parent.

📱 Responsive Images

A responsive image automatically resizes to fit the container it lives in. The image looks good on a wide desktop monitor and on a narrow phone screen without code changes.

The trick is one short CSS rule. Set the width to 100% so the image fills its container, and set height: auto so the aspect ratio is preserved.

Making Every Image Responsive

img {
    max-width: 100%;
    height: auto;
}

Using max-width instead of plain width means the image never grows beyond its natural size, only shrinks down when needed.

Hero Images

A hero image is a large, attention-grabbing image at the top of a page, often spanning the full browser width. Hero images set the visual tone of the site and frequently appear right below the header on a home page.

Remove inline width and height attributes. If you control sizing with CSS, leftover width="800" and height="450" attributes on the <img> tag can fight your responsive rules. Strip them from the HTML once your CSS is in charge.

🔄 The CSS Reset

Every browser ships with default styles. Chrome's defaults are slightly different from Firefox's, which differ from Safari's. The result: a page can look subtly different on each browser even before you apply your own CSS.

A CSS reset is a rule (or short set of rules) that levels the playing field by setting common properties to known values. The reset goes at the very top of your stylesheet, before any other rule.

A Simple Universal Reset

/* Apply to every element on the page */
* {
    margin: 0;
    padding: 0;
    box-sizing: border-box;
}

The asterisk (*) is the universal selector. It matches every element on the page. With the reset in place, your design starts from a clean slate and looks the same in Chrome, Firefox, Safari, and Edge.

💬 CSS Comments

Like HTML, CSS supports comments. CSS comments use a different syntax: they start with /* and end with */. They can span multiple lines.

Comments in a Stylesheet

/* ===========================
   Site-wide typography
   =========================== */

body {
    font-family: Georgia, serif;
}

/* Buttons use the brand green */
.action {
    background-color: darkgreen;
}

Use comments to mark major sections of a stylesheet, explain non-obvious decisions, or temporarily disable a rule while testing.

✅ Validating CSS Files

Just as the W3C provides a validator for HTML, it provides one for CSS. The W3C CSS validator reads your stylesheet, checks every rule against the official specification, and reports any errors or warnings.

Why Validate Your CSS

  • Catch typos. A misspelled property like colr is silently ignored by the browser; the validator flags it immediately.
  • Verify syntax. A missing semicolon or curly brace can break every rule that follows. The validator pinpoints exactly where the trouble starts.
  • Surface deprecated properties. Some old properties have been replaced by better alternatives. The validator points these out so your code stays modern.
Where to validate. Visit jigsaw.w3.org/css-validator. You can validate by URI, by file upload, or by pasting code directly. Errors must be fixed; warnings are advisory.

Common CSS Errors

Error What It Means
Property doesn't exist You typed a property name that isn't valid CSS. Usually a typo.
Invalid value The value doesn't fit the property. For example, color: 25px.
Missing semicolon A declaration didn't end with a semicolon. The next declaration runs into it.
Unclosed block A rule's opening curly brace has no matching closing brace.

✨ Best Practices

Habits that make CSS maintainable

  • Use external stylesheets. Keep CSS out of HTML except for true exceptions.
  • Start with a CSS reset. A consistent baseline saves cross-browser headaches.
  • One declaration per line. Easier to read, easier to track in version control.
  • Use semantic selectors. Style nav a, not .thing-with-blue-text.
  • Prefer classes over IDs for styling. Classes are reusable; IDs aren't. Save IDs for unique elements like the page wrapper.
  • Group related rules. Keep all typography rules together, all layout rules together, and so on.
  • Comment your sections. A short comment at the top of each region of the file makes long stylesheets navigable.
  • Use a font stack with fallbacks. Always include a generic family at the end.
  • Make images responsive. One short rule with max-width: 100% covers the whole site.
  • Validate before publishing. Catching errors in the validator is faster than chasing them in the browser.
