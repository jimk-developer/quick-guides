🖼️ Adding Images to a Website

Images give a website personality. A well-chosen photo, illustration, or icon draws the eye, reinforces the brand, and helps visitors understand the content faster than text alone. Images used as business logos are often the first thing a visitor remembers about a site.

Don't overuse images. Too many images slow down the page, distract from the content, and can feel like visual clutter. Every image should serve a purpose. If an image doesn't support the surrounding content, leave it out.

Pixels and Image Size

A pixel is the smallest unit of light or color on a screen. Digital images are grids of pixels arranged in rows and columns. An image listed as 800 by 600 has 800 pixels across and 600 pixels down, for a total of 480,000 pixels.

Two different measurements matter when you add an image to a webpage:

📐 Image size (dimensions)

The number of pixels wide and tall, such as 800 by 600.

💾 File size

How many kilobytes or megabytes the image takes up on disk. This is what determines how fast the image downloads.

Smaller dimensions don't shrink the file. Setting width="400" in your HTML makes a large image display smaller, but the browser still downloads the full original file. To actually speed up page loading, resize the image in a photo editor before uploading it.

Image File Formats

Four image formats dominate the web. Each has strengths and weaknesses that determine when you should use it.

.jpg JPEG

✓ Great for photos

✗ No transparency

.png PNG

✓ Transparency, crisp edges

✗ Larger than JPEG for photos

.gif GIF

✓ Animation, small size

✗ Limited to 256 colors

.svg SVG

✓ Scales perfectly at any size

✗ Not ideal for photos

Format Full Name Compression Best For
JPEG / JPG Joint Photographic Experts Group Lossy Photographs and images with many gradients
PNG Portable Network Graphics Lossless Logos, icons, screenshots, and any image that needs transparency
GIF Graphics Interchange Format Lossless Short animations and simple graphics with few colors
SVG Scalable Vector Graphics XML-based, not pixels Logos, icons, charts, and any image that must stay sharp when resized

Lossy vs. Lossless Compression

🗜️ Lossy Compression

Discards some image data to make the file smaller. The result looks almost identical to the human eye. JPEG uses lossy compression, which is why it is great for photos but not for crisp graphics.

📦 Lossless Compression

Makes the file smaller without throwing away any data. The image can be restored to its exact original form. PNG and GIF use lossless compression.

Optimizing Images for the Web

To optimize an image means to reduce its file size as much as possible without making the image look noticeably worse. A photo or graphics editor such as Photoshop, GIMP, or an online tool can optimize images by:

  • Resizing to the actual dimensions needed on the page.
  • Compressing by saving at a lower quality setting (for JPEG).
  • Stripping metadata such as camera info that the browser doesn't need.
  • Choosing the right format for the content.
Aspect ratio matters. When you resize an image, turn on "Constrain proportions" or "Maintain aspect ratio" in your editor. This changes width and height together so the image doesn't get stretched or squashed.

🏷️ The Image Tag

The image tag, written as <img>, inserts an image into a webpage. It is an empty (void) tag, meaning it has no closing tag because there is no content to wrap. All information travels in the tag's attributes.

Core Image Attributes

Attribute Required? Purpose
src Yes The source path that tells the browser where to find the image file.
alt Yes (for accessibility) Alternate text describing the image. Read aloud by screen readers and shown if the image fails to load.
width Optional The display width in pixels.
height Optional The display height in pixels.

A Basic Image Tag

<img src="images/prairie-fields.jpg"
     alt="Rolling prairie fields at sunset"
     width="800"
     height="450">

Writing Good Alt Text

Alt text serves three roles: it describes the image for blind and low-vision users, it displays if the image fails to load, and it helps search engines understand the image. Good alt text is specific, concise, and describes what the image shows and why it matters.

✅ Good Alt Text

alt="Golden retriever puppy playing in snow"

Specific and descriptive. Someone who can't see the image understands what's there.

❌ Weak Alt Text

alt="image"

Adds no information. Tells a screen reader user nothing.

Decorative images: If an image is purely decorative and adds no meaning, use an empty alt attribute (alt=""). This tells screen readers to skip it rather than announcing a filename.

Why an Image Might Not Display

When an image doesn't show up, the alt text appears in its place (or a broken image icon). The fix is almost always one of these three things:

  • Wrong file path in the src attribute. Folder names and filenames must match exactly, including capitalization.
  • File not uploaded to the server. The image works locally but not after publishing because it wasn't included in the upload.
  • Wrong file extension. The tag points to .jpg but the file is actually saved as .jpeg.

📦 Div Elements and Layout

The div element defines a generic division or section of a webpage. A div has no meaning of its own. It is a neutral container that groups related content together so you can style or position the group as a unit.

Why Use Divs in an HTML 5 Page

HTML 5 provides semantic elements (header, nav, main, footer) for common page regions. So why would anyone still use div?

🎁 Wrapping for Layout

A common pattern is to wrap everything on the page in a single <div id="container">. This gives you one element to style for the overall page width, margins, and centering.

🧩 Grouping Within Main

Inside the <main> element, div blocks can group related content that doesn't fit a semantic tag, such as a two-column layout or a card grid.

🎯 CSS Hooks

Giving a div an id or class creates a target for CSS styling and JavaScript selection.

🕰️ Legacy Code

Before HTML 5, divs were the only way to divide a page into regions. You will still see this pattern in older sites you may need to maintain.

The id Attribute

The id attribute gives an element a unique name. No two elements on the same page can share the same id. Common uses include styling with CSS, targeting with JavaScript, and creating bookmark links (covered later in this guide).

A Container Div Wrapping the Page

<body>
    <div id="container">
        <header> ... </header>
        <nav> ... </nav>
        <main> ... </main>
        <footer> ... </footer>
    </div>
</body>
Why "container"? The word "container" is a convention. It describes the role of the div (it contains everything) rather than how it looks. Conventions like this make code easier to read because other developers recognize the pattern.

Fixed vs. Fluid Layouts

📏 Fixed Layout

Page width is a specific pixel value, such as 960 pixels. The layout stays the same size regardless of the visitor's screen width. Simple to design, but leaves wasted space on wide screens and requires horizontal scrolling on small ones.

💧 Fluid Layout

Page width is a percentage, such as 90% of the browser window. The layout expands and contracts with the window. More flexible and the foundation of modern responsive design.

🔗 Adding Links to a Webpage

Hyperlinks are the defining feature of the web. A hyperlink (or link) is text, image, or other content that a visitor can click to jump somewhere else. That somewhere might be another page on the same site, a page on a different site, a spot on the current page, an email address, or a phone number.

The Anchor Element

All hyperlinks use the anchor element, written as <a>. The text or image between the opening and closing anchor tags becomes clickable. The destination is set with the href attribute (short for "hypertext reference").

Anchor Element Structure

<a href="destination">Clickable text</a>

Default Link States

Browsers display links in distinct colors depending on their state. These default colors have been standard for decades so visitors instinctively recognize clickable text.

normal Unvisited link (typically blue)
visited Visited link (typically purple)
active Active link being clicked (typically red)
hover Mouse hovering over the link

You can override these colors with CSS to match your site's design, but keep the principle: links should look clearly different from regular text.

Six Types of Hyperlinks

The href attribute can point to many kinds of destinations. Here are the six common types.

1. Relative Links

A relative link points to another page on the same website. The path is relative to the current page's location. Because the link stays within the site, you only need to write the part of the path that differs.

Relative Link Examples

<!-- File in the same folder -->
<a href="about.html">About Us</a>

<!-- File in a subfolder -->
<a href="menu/breakfast.html">Breakfast Menu</a>

<!-- File in the parent folder -->
<a href="../index.html">Home</a>
📁 my-website/ ├── 📄 index.html # link with href="index.html" ├── 📄 about.html # same-folder: href="about.html" ├── 📁 menu/ │ └── 📄 breakfast.html # from index: href="menu/breakfast.html" └── 📁 images/ └── 🖼️ logo.png # from index: href="images/logo.png"

2. Absolute Links

An absolute link points to a page on a different website. The full URL including the protocol (https://) and domain name must be included.

Absolute Link Example

<a href="https://www.ranken.edu">
    Visit Ranken Technical College
</a>
Open external links in a new tab. Add target="_blank" to keep visitors on your site when they click an absolute link. Pair it with rel="noopener" for security.

3. Bookmark Links

A bookmark link (or anchor link) jumps to a specific spot on a page. You first mark the destination with an id attribute, then link to that id using a hash symbol.

Creating and Linking to a Bookmark

<!-- Step 1: mark the destination with an id -->
<h2 id="hours">Our Hours</h2>

<!-- Step 2: link to it from elsewhere on the page -->
<a href="#hours">Jump to Hours</a>

<!-- You can also link to a bookmark on another page -->
<a href="about.html#hours">Our Hours</a>

4. Image Links

An image link turns an image into a hyperlink by wrapping an <img> tag inside an <a> tag. Logos that link back to the home page are a classic example.

Wrapping an Image in a Link

<a href="index.html">
    <img src="images/logo.png"
         alt="Prairie Bakery home"
         width="200">
</a>
Nesting rule. The image tag goes inside the anchor tag. The opening <a> comes first, then the <img>, then the closing </a>.

5. Email Links

An email link opens the visitor's default email program with a new message addressed to you. Use mailto: followed by the email address in the href.

Email Link with Subject and Body

<!-- Simple email link -->
<a href="mailto:hello@prairiebakery.com">
    Email us
</a>

<!-- Email link with subject and body pre-filled -->
<a href="mailto:hello@prairiebakery.com?subject=Catering%20Inquiry&body=Hi,%20I'd%20like...">
    Ask about catering
</a>

6. Telephone Links

A telephone link starts a phone call when tapped on a mobile device. Use tel: followed by the phone number. These links are especially useful on mobile-first websites.

Telephone Link

<a href="tel:+15555551234">
    Call (555) 555-1234
</a>

Writing Good Link Text

The clickable text of a link should describe where the link goes. "Click here" tells the visitor nothing. Screen readers often read through a list of links in isolation, so descriptive text matters for accessibility too.

✅ Descriptive

Read our shipping policy

The link text describes the destination.

❌ Vague

For our shipping policy, click here

A screen reader user hearing "click here" in a link list has no idea where it goes.

Attribute Syntax Reminder

Every HTML attribute follows the same pattern: the attribute name, an equals sign, and the value in quotation marks.

The Shape of an Attribute

name="value"

Tags can have many attributes, separated by spaces: <img src="photo.jpg" alt="A photo" width="400">. Order doesn't matter, but consistent order makes code easier to read.

📋 Creating Lists

HTML supports three kinds of lists. Picking the right type carries meaning: a numbered list implies order and priority, a bulleted list implies a group of equal items, and a description list implies pairs of terms and definitions.

Ordered Lists

An ordered list numbers its items automatically. Use it when the sequence matters, such as steps in a recipe, ranked items, or a process.

HTML
<ol>
    <li>Preheat oven</li>
    <li>Mix ingredients</li>
    <li>Bake 20 minutes</li>
</ol>
Displays as
  1. Preheat oven
  2. Mix ingredients
  3. Bake 20 minutes

Unordered Lists

An unordered list displays items with bullets. Use it when the items are a group of equal peers and the order doesn't matter, such as features, ingredients, or navigation links.

HTML
<ul>
    <li>Sourdough loaves</li>
    <li>Croissants</li>
    <li>Fresh pastries</li>
</ul>
Displays as
  • Sourdough loaves
  • Croissants
  • Fresh pastries

Description Lists

A description list pairs terms with their descriptions. Use it for glossaries, key/value pairs, or any content where each item has a label and an explanation.

Description lists use three tags that work together:

  • <dl> wraps the whole description list
  • <dt> holds the term (the word being defined)
  • <dd> holds the description of that term
HTML
<dl>
    <dt>Baguette</dt>
    <dd>A long, thin French loaf
        with a crisp crust.</dd>

    <dt>Ciabatta</dt>
    <dd>An Italian bread with a
        soft, airy interior.</dd>
</dl>
Displays as
Baguette
A long, thin French loaf with a crisp crust.
Ciabatta
An Italian bread with a soft, airy interior.

Nested Lists

Any list item can contain another complete list. This is useful for hierarchical content such as an outline or a multi-level menu.

A Nested Unordered List

<ul>
    <li>Breads
        <ul>
            <li>Sourdough</li>
            <li>Whole wheat</li>
        </ul>
    </li>
    <li>Pastries</li>
</ul>

🗺️ Embedding a Map

Many businesses include a map on their contact page so visitors can see the location at a glance. You don't build the map yourself. You embed one from a service like Google Maps using an <iframe> element.

What an iframe Does

An iframe (inline frame) displays content from another webpage inside your own page. It is a window into someone else's content. Maps, YouTube videos, and some social media widgets are all commonly embedded this way.

Embedding from Google Maps

The Embed Process

  1. Go to maps.google.com and search for your location.
  2. Click the Share button.
  3. Choose the Embed a map tab.
  4. Pick a size, then copy the HTML code Google provides.
  5. Paste that code into your page where you want the map to appear.

The Shape of an iframe

<iframe
    src="https://www.google.com/maps/embed?pb=..."
    width="600"
    height="450"
    title="Prairie Bakery location map"
    loading="lazy">
</iframe>
Always include a title attribute. Screen readers use it to announce the purpose of the embedded content. Without it, the iframe is effectively invisible to users of assistive technology.

🧪 Testing and Validating

After adding images and links to a page, two checks matter: every link should actually work, and the page should still validate as proper HTML.

Testing Your Links

Open the page in a browser and click every link. Look for these specific problems:

  • Broken links. The target page doesn't exist or the path is wrong. You'll see a "404 Not Found" error.
  • Wrong destination. The link works but goes somewhere unexpected. Usually a typo in the href.
  • Images not loading. A broken image icon or the alt text appears in place of the image.
  • External links open in the same tab. A visitor who follows an external link may not find their way back.
  • Email links don't open an email. Usually a typo in mailto:.

Validating New Pages

Run each new page through the W3C validator at validator.w3.org. Images and links introduce new ways to make mistakes: missing alt attributes, unclosed anchor tags, and duplicate id values are all common.

✨ Best Practices

Habits for images and links

  • Always include alt text on images. Required for accessibility and valid HTML.
  • Optimize images before uploading. Resize and compress in a photo editor, not just in the HTML attributes.
  • Use descriptive link text. Avoid "click here" at all costs.
  • Prefer relative links within your site. They keep working even if you move the site to a new domain.
  • Use absolute links only for external sites. They lock you to a specific URL.
  • Open external links in a new tab. Keeps visitors on your site while letting them explore.
  • Keep filenames lowercase and use hyphens. Avoid spaces, special characters, and mixed capitalization.
  • Pick the right image format. JPEG for photos, PNG for graphics with transparency, SVG for logos and icons.
  • Test every link after adding it. A single broken link damages the visitor's trust in the whole site.
