🌐 Exploring the Internet

The Internet is the world's largest computer network. It is a global system that links billions of devices so they can share information. The World Wide Web (or simply "the web") is just one service that runs on top of the Internet. Other Internet services include email, file transfer, texting, and voice over IP (VoIP).

Common confusion: Many people use "Internet" and "web" interchangeably, but they are not the same. The Internet is the infrastructure (cables, routers, servers). The web is a collection of linked documents that travels across that infrastructure.

Internet Building Blocks

🔘 Node

Any device connected to a network, such as a computer, phone, printer, or smart thermostat.

🕸️ Network

A group of nodes connected so they can share resources and communicate.

🔌 Data Lines

The physical links (fiber optic, cable, wireless, satellite) that carry data between nodes.

🏢 ISP

An Internet Service Provider sells access to the Internet and connects homes and businesses to the Internet backbone.

🛣️ Internet Backbone

A collection of high-speed data lines that link major computer systems worldwide.

📡 Internet of Things (IoT)

Everyday objects (appliances, vehicles, wearables) equipped with sensors and Internet connections.

The Web at a Glance

Term Meaning
World Wide Web A system of interlinked documents (webpages) accessed through the Internet.
Webpage A single document on the web that may contain text, images, video, and links.
Website A group of related webpages stored under one domain.
Home page The default introductory page of a website, usually named index.
Hyperlink Clickable text, image, or region that jumps to another resource on the web.

📜 Internet Protocols

A protocol is a set of rules that computers follow to talk to each other. Without agreed-upon protocols, a laptop in Missouri and a server in Tokyo would have no way to exchange data reliably.

Protocol Full Name What It Does
HTTP Hypertext Transfer Protocol Rules for transferring webpages, images, video, and other web content between browsers and web servers.
HTTPS HTTP Secure Encrypted version of HTTP, required for any site handling logins or payment information.
FTP File Transfer Protocol Rules for uploading and downloading files between a client and a server.
TCP/IP Transmission Control Protocol / Internet Protocol The foundational protocol suite that breaks data into packets, routes them, and reassembles them on arrival.
Think of TCP/IP like the postal system. A large package (your data) gets split into smaller envelopes (packets). Each envelope is addressed, routed independently, and reassembled when they all arrive at the destination.

🖥️ Servers, Clients, and DNS

The Client–Server Model

The web runs on a simple pattern called client–server. A client (usually a web browser) sends a request. A server receives the request, looks up or builds a response, and sends it back.

💻 Client (Browser)
→ request →
🌐 Server
💻 Client (Browser)
← response ←
🌐 Server

Every webpage you visit follows this request/response cycle.

Domain Name System (DNS)

Every device on the Internet has a numeric address called an IP address (for example, 142.250.72.110). Humans find numbers hard to remember, so we use domain names like ranken.edu instead.

The Domain Name System (DNS) is the Internet's phonebook. When you type a domain name into your browser, DNS translates it into the correct IP address so your computer knows where to send its request.

Anatomy of a URL

A URL (Uniform Resource Locator) is the full web address of a resource. Every URL has several parts that each tell the browser something different:

https://support.microsoft.com/help/windows
protocol: how to communicate
subdomain: area within a domain
domain: the organization
TLD: top-level domain (.com, .edu)
path: the specific resource

🏛️ Types of Websites

Not every website is open to the public. Businesses and organizations often use restricted-access sites for internal purposes.

🌍 Internet Site

A public-facing website available to anyone with an Internet connection.

🏢 Intranet

A private network inside an organization. Accessible only to employees, it typically hosts internal resources like HR policies and directories.

🤝 Extranet

An extension of an intranet that gives limited access to trusted outsiders such as suppliers, partners, or key customers.

Within those categories, websites serve many purposes: commercial (e-commerce), informational, educational (including Learning Management Systems), entertainment, news, personal/portfolio, blogs, and social networking.

🗺️ Planning a Website

Before writing any HTML, a designer must plan. A website built without planning almost always ends up cluttered, confusing to navigate, and painful to maintain. Good planning starts with three questions:

The Three Planning Questions

  1. What is the purpose? Sell a product? Raise awareness? Entertain? The purpose often comes from the organization's mission statement.
  2. Who is the audience? Age, technical skill, devices they use, and what they want from your site.
  3. What does success look like? Purchases, sign-ups, downloads, time on site, or something else.

Wireframes and Site Maps

📐 Wireframe

A simple sketch of a single page's layout showing where the header, navigation, content, and footer go. Wireframes focus on structure, not colors or final graphics.

🗂️ Site Map

A diagram showing all the pages in a site and how they link together. The site map gives you the big-picture view before you build.

You don't need fancy software to wireframe. Pencil and paper work fine. Free tools like Pencil Project, Mockplus, and Wireframe CC are popular. Many designers use Word, PowerPoint, or Figma.

Website Structures

There are three common ways to organize the pages in a website. Picking the right one depends on how visitors will use the content.

➡️ Linear

Pages are connected in a straight sequence. Best when visitors should read in a specific order, such as tutorials, onboarding steps, or a multi-page article.

🌳 Hierarchical

A treelike structure with a home/index page at the top linking to sub-pages. Best for sites with distinct categories. It is the most common structure on the web.

🕸️ Webbed

No set order, so visitors can jump between any pages. Best for reference sites, encyclopedias, and sites where content isn't meant to be read sequentially.

🎨 Design Principles

Content is important, but how that content looks determines whether visitors stay or leave. Four design areas deserve attention on every site.

🖼️ Graphics

Images add visual appeal and communicate the look and feel of a brand. Use high-quality, relevant images, and don't overdo it. Too many graphics slow the page and distract from content.

🧭 Navigation

Clear, consistent navigation is the single most important usability feature. Every page should have a designated navigation area with links to the site's major sections.

🔤 Typography

Font choice and size affect both readability and mood. Limit yourself to one or two font families. Make sure body text is large enough to read comfortably on a phone.

🎨 Color

Pick a small, coordinated color palette that reinforces your brand. Avoid harsh combinations like red text on a yellow background because they strain the eyes and reduce readability.

White Space

Active vs. Passive White Space

White space (sometimes called negative space) is the empty area around elements on a page. It isn't wasted. It helps the eye rest and guides attention.

  • Active white space: Deliberate empty area used to emphasize an element or group related items.
  • Passive white space: Small gaps between letters, lines, and paragraphs that make text easier to read.

📱 Responsive and Multiplatform Design

Visitors arrive on desktops, laptops, tablets, phones, and even smart TVs. A website that works well on one screen size but breaks on another fails a large part of its audience.

Responsive design is an approach where a single website adapts its layout to fit the size of the visitor's screen. Images resize, columns stack, and navigation collapses into a menu, all automatically.

Mobile-first thinking: More than half of all web traffic now comes from phones. Designing for the smallest screen first forces you to prioritize what matters most, and the desktop version tends to come out cleaner as a result.

♿ Accessibility

The web should work for everyone, including people with visual, hearing, motor, or cognitive impairments. Good accessibility practice also benefits users on slow connections, older devices, or in bright sunlight.

The World Wide Web Consortium (W3C) publishes the Web Content Accessibility Guidelines. They are built around four principles. Web content should be:

Principle What It Means
Perceivable Users can perceive the content with at least one sense. Provide text alternatives for images and captions for video.
Operable Users can interact with the interface, including with only a keyboard, no mouse required.
Understandable Content and navigation are predictable and use plain language.
Robust Content works reliably across browsers, devices, and assistive technology like screen readers.
A quick accessibility habit: Always write alt text for images and make sure every interactive element is reachable with the Tab key.

🏷️ HTML Fundamentals

HTML stands for Hypertext Markup Language. It is the standard language for creating webpages. HTML is a markup language, not a programming language. It uses tags to describe the structure and meaning of content rather than instructions for a computer to execute.

Tags, Elements, and Attributes

🏷️ Tag

A special keyword surrounded by angle brackets that tells the browser how to treat content. Examples include a paragraph tag, a heading tag, or an image tag.

🧩 Element

A complete unit made up of a start tag, content, and an end tag. The element is what you actually place in the document.

⚙️ Attribute

An extra piece of information inside a tag that configures how it behaves. Examples include the language of a page or the source of an image.

Paired vs. Void Tags

Type Description Examples
Paired tag Has a start tag and a matching end tag that wraps around content. Paragraphs, headings, links, lists
Empty (void) tag Has no end tag because it has no content. It just marks a location or inserts something. Line break, horizontal rule, image, meta tag
Use lowercase. HTML 5 convention and the current W3C standard is to write all tags in lowercase letters. Mixing uppercase and lowercase still works in browsers, but lowercase is the professional standard.

HTML Versions

HTML has gone through several versions over the years. The current standard is HTML 5. HTML 5 is important for three reasons:

  • Semantic structural elements. Tags like header, nav, main, article, section, and footer describe what a region of content means, not just how it looks.
  • Native audio and video. The audio and video tags play media without requiring browser plug-ins.
  • Mobile and app friendly. HTML 5 was designed with phones, tablets, and web applications in mind.

Related Technologies

📦 XML

Extensible Markup Language is a companion to HTML designed to transport and store data rather than display it.

📋 XHTML

Extensible Hypertext Markup Language is a stricter version of HTML written using XML rules.

💻 Other Web Languages

HTML handles structure, but modern websites use several other languages to add style, interactivity, and server-side logic. You'll learn some of these in later guides.

Language Runs On What It Does
CSS Browser Controls the visual appearance, including colors, fonts, layout, and spacing.
JavaScript Browser (client-side) Adds interactivity: popups, form validation, dynamic content, animations.
jQuery Browser A JavaScript library that simplifies common tasks, though modern JavaScript has reduced the need for it.
PHP Web server An open-source server-side language used heavily for database-driven sites. Files end in .php.
ASP / ASP.NET Web server Microsoft's server-side technology. Traditional ASP files end in .asp.
Client-side vs. server-side: Client-side code runs inside the visitor's browser. Server-side code runs on the web server, and the visitor only ever sees the HTML it produces.

🛠️ Web Authoring Tools

You need software to write HTML. There are four main categories of tools, each suited to a different kind of work.

📝 Text Editor

A plain program that lets you enter, change, save, and print text, including HTML tags. Simple and fast, but lacks advanced features.

🖋️ HTML Editor

A text editor with extra features for web work: color-coded tags (syntax highlighting), tag auto-complete, spell-checking, and previews.

🎨 WYSIWYG Editor

What You See Is What You Get. It is a visual editor where you drag and arrange elements and the software writes the code for you.

☁️ Online Code Editor

Browser-based editors that let you write and preview code from anywhere, often with team collaboration built in.

Popular Tools

Tool Category Notes
Notepad++ HTML editor Free, open-source, Windows only. Popular for learning.
Visual Studio Code HTML editor Free, cross-platform. Industry favorite.
Atom HTML editor Free, open-source, cross-platform.
Brackets HTML editor Open-source editor built specifically for web design.
Adobe Dreamweaver WYSIWYG Commercial product. Visual and code modes.
Platform independent. HTML is platform independent. A file created in Windows can be viewed on Mac, Linux, iPhone, or Android browsers without modification. The tool you pick is a matter of preference.

📄 Structure of a Basic Webpage

Every HTML 5 webpage follows the same basic skeleton. The document starts with a DOCTYPE declaration, then a root element that contains two major sections: a head and a body.

<!DOCTYPE html>
↓
<html> (root)
↙ ↘
<head>
<body>

The document tree of every HTML 5 page

What Each Part Does

Section Purpose
<!DOCTYPE html> Must be the very first line. Tells the browser this is an HTML 5 document so it renders the page correctly.
<html> The root element. Wraps the entire page. A lang attribute identifies the language (for example, lang="en" for English).
<head> Information about the page that isn't directly shown, such as the title, character encoding, linked stylesheets, and meta tags.
<title> Appears in the browser tab, bookmarks, and search results.
<meta> Provides information about the page, such as encoding, description, and viewport settings. Meta tags are void (no end tag).
<body> Everything the visitor actually sees, including text, images, links, videos, and forms.

UTF-8 Character Encoding

UTF stands for Unicode Transformation Format. It is a standard that allows computers to display text from any language (English, Arabic, Chinese, emoji, mathematical symbols) using a single encoding system.

UTF-8 is the recommended encoding for HTML 5. Declaring UTF-8 in the meta section prevents strange characters (mojibake) from appearing when your page includes anything outside basic English letters.

Naming your home page: By convention, the main page of a website is named index.html. Web servers automatically serve this file when someone visits the root of a site, which is why you don't have to type it in the URL.

✨ Best Practices

Habits that will save you time in every guide to come

  • Use lowercase for all tags and attribute names. It is the modern standard.
  • Indent nested elements. Indentation is not required, but it makes the code much easier to read.
  • Always close paired tags. Unclosed tags cause layout bugs that are hard to track down.
  • Give every image meaningful alt text. Screen readers depend on it, and so does search engine optimization.
  • Save your home page as index.html. Servers look for this filename by default.
  • Test in more than one browser. Most pages look the same everywhere, but edge cases show up in testing.
  • Check on a phone. Mobile traffic is the majority, so don't ship a site you haven't seen on a small screen.
