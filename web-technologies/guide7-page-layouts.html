🌉 The Next Layer of Polish

By this point a website has structure (HTML), style (CSS), responsive behavior (media queries), and adapts to phones, tablets, and desktops. The next step is to refine the design with HTML 5 semantic elements that describe content more precisely, modern CSS layout tools that replace older float-based layouts, and visual touches that make the page feel polished.

Everything in this guide builds on what came before. The goal isn't to throw out the existing pages but to upgrade them.

🧩 More HTML 5 Semantic Elements

Earlier guides introduced the high-level semantic elements: header, nav, main, and footer. HTML 5 also provides finer-grained semantic elements that describe specific kinds of content within those regions.

Element Meaning Example Use
<article> Self-contained content that could stand on its own and be reused elsewhere Blog post, news story, product card, forum entry
<aside> Content tangentially related to its surroundings Sidebar, pull quote, related links, ad block
<section> A thematic grouping of content with its own heading A chapter, a tabbed area, a part of a longer document
<figure> Self-contained visual content referenced by the main flow Image, chart, diagram, code listing, illustration
<figcaption> The caption or legend for a figure Description text under a chart or photo

Semantic elements nest naturally. A typical layout for a content-heavy page combines them like this:

The article Element

The article element wraps content that could stand on its own. The test is simple: if you could pull this content out of the page and republish it on a different site without losing meaning, it belongs in an <article>.

Articles can be nested inside other elements (such as main or section) and can even contain other articles (such as a list of comments inside a blog post).

An article Element

<article>
    <h2>How to Choose the Right Running Shoes</h2>
    <p>The right pair of shoes makes the
       difference between an enjoyable run...</p>
    <p>Start by understanding your foot type...</p>
</article>

The aside Element

The aside element contains content tangentially related to its surroundings. Aside content adds context but isn't essential to understanding the main flow. Sidebars, "did you know" boxes, related-posts panels, and pull quotes are all good uses.

An aside Inside main Content

<main>
    <article>
        <h2>The Forest Hike</h2>
        <p>The trailhead sits at the end of a quiet road...</p>

        <aside>
            <h3>Quick Tip</h3>
            <p>Bring twice the water you think you'll need.</p>
        </aside>

        <p>The first mile is the hardest...</p>
    </article>
</main>

The section Element

The section element groups thematically related content. Each section should have its own heading. Think of a section as a chapter within a larger document.

Section is not a generic container. If you just need a div to attach styles or layout to, use <div>. Reserve <section> for content that has a clear thematic identity, ideally with its own heading.

Multiple section Elements on a Page

<main>
    <section>
        <h2>Our Story</h2>
        <p>Prairie Bakery opened in 2014...</p>
    </section>

    <section>
        <h2>Our Bakers</h2>
        <p>Each loaf is shaped by hand by one of our four bakers...</p>
    </section>

    <section>
        <h2>Visit Us</h2>
        <p>We're open Tuesday through Saturday...</p>
    </section>
</main>

The figure and figcaption Elements

The figure element wraps an image, chart, diagram, code listing, or any other self-contained visual that the main content refers to. The figcaption element holds the caption for the figure.

Together they create a single semantic unit. A screen reader announces them as a group, and you can move the entire figure (image plus caption) anywhere in the page without separating the two.

A figure with a figcaption

<figure>
    <img src="images/sourdough-loaf.jpg"
         alt="Freshly baked sourdough loaf with golden crust">
    <figcaption>
        Our signature country sourdough,
        baked in a wood-fired oven for 45 minutes.
    </figcaption>
</figure>
Each figure stands alone. If you want to caption two images separately, wrap each in its own <figure>. Don't put two images inside a single figure unless the caption applies to the pair as a unit.

🎛️ CSS Grid Layout

The CSS Grid Layout is a modern way to lay out content in rows and columns. It was created by the W3C Working Group as a more reliable replacement for the older float-based layout techniques. Grid gives you exact control over both dimensions: how wide each column is, how tall each row is, and which cell each item belongs in.

Why Grid Replaces Float

🌊 Float-Based Layout

Originally invented for wrapping text around images. Repurposed for layout, but never quite designed for it. Requires hacks like clearfix to control overflow.

🎯 Grid Layout

Designed specifically for two-dimensional layouts. Define the grid once and items snap into position. No clearing needed, no surprise overflows.

The Grid Mental Model

To use Grid, picture your page as a grid of rows and columns. The container tells the grid how many columns and rows it has, and each child element takes up one cell by default. The container is called the grid container; its children are grid items.

Activating Grid on a Container

.gallery {
    display: grid;
    grid-template-columns: 1fr 1fr 1fr;
    gap: 20px;
}

Common Grid Properties

Property Purpose
display: grid Turns the element into a grid container.
grid-template-columns Defines the columns of the grid (number, widths).
grid-template-rows Defines the rows of the grid (number, heights).
gap Sets the space between grid cells (also row-gap and column-gap).
grid-column Tells a specific item how many columns to span.
grid-row Tells a specific item how many rows to span.

Column Width Values

The values you can use for grid-template-columns determine how the columns share the available space.

Value Effect
200px A fixed-width column of exactly 200 pixels.
1fr One fraction of the remaining space. 1fr 1fr means two equal columns.
auto The column expands to fit its content.
repeat(3, 1fr) Shorthand for 1fr 1fr 1fr.
minmax(200px, 1fr) At least 200 pixels, but expands to take a fraction of remaining space.
repeat(auto-fit, minmax(200px, 1fr)) Pack as many 200-pixel-minimum columns as fit, expanding to fill the row.

A Multi-Column Layout with Grid

Three-Column Layout, Mobile-First

/* Mobile: single column */
.cards {
    display: grid;
    grid-template-columns: 1fr;
    gap: 20px;
}

/* Tablet: two columns */
@media only screen and (min-width: 768px) {
    .cards {
        grid-template-columns: 1fr 1fr;
    }
}

/* Desktop: three columns */
@media only screen and (min-width: 1024px) {
    .cards {
        grid-template-columns: repeat(3, 1fr);
    }
}

Spanning Multiple Columns

By default, each grid item takes one cell. To make an item span multiple columns or rows, use the grid-column property.

A Featured Card That Spans Two Columns

.featured {
    grid-column: span 2;
}

🌫️ Opacity

The opacity property makes an element partially transparent. The value ranges from 0 (completely invisible) to 1 (completely opaque). Useful for fade effects, hover states, and softening overlays.

opacity: 1.0
opacity: 0.75
opacity: 0.5
opacity: 0.25
opacity: 0.1

Opacity Examples

.fade-in-thumbnail {
    opacity: 0.7;
}

.fade-in-thumbnail:hover {
    opacity: 1.0;
}
Opacity affects everything inside. When you set opacity on a container, every child element inherits the same transparency. Text, images, borders, all of them. To make only the background transparent and keep text fully opaque, use a color value with alpha (such as rgba(0, 0, 0, 0.5)) instead of the opacity property.

📦 Box Shadow

The box-shadow property adds a drop shadow to an element. Shadows create depth, draw attention, and signal that an element is "above" the rest of the page. Cards, buttons, modals, and any element that should feel elevated benefit from shadow.

subtle
soft
prominent
inset
glow

Box Shadow Anatomy

The box-shadow property takes up to five values, in this order:

The Anatomy of a Box Shadow

.card {
    box-shadow: 0 4px 12px 0 rgba(0,0,0,0.15);
    /*           ↑ ↑   ↑    ↑  ↑                  */
    /*    h-offset  v-offset blur spread  color    */
}
Value What It Controls
Horizontal offset How far right (positive) or left (negative) the shadow sits.
Vertical offset How far down (positive) or up (negative) the shadow sits.
Blur radius How soft the shadow edges are. Zero means a hard edge.
Spread radius How much larger the shadow is than the element itself. Optional.
Color Any CSS color, often rgba for transparency.
inset Optional keyword that makes the shadow appear inside the element.
Subtle shadows look professional. A small offset (2 to 6 pixels), a generous blur (10 to 20 pixels), and a low-opacity black is the classic "Material design" recipe. Heavy, sharp shadows tend to look dated.

✏️ Text Shadow

The text-shadow property applies a shadow to text rather than to a box. The syntax is similar to box-shadow, but text-shadow has no spread value and no inset option.

Text Shadow Examples

/* Subtle drop shadow under the headline */
h1 {
    text-shadow: 2px 2px 4px rgba(0,0,0,0.3);
}

/* Glowing text effect */
.glow {
    text-shadow: 0 0 8px #6D28D9, 0 0 16px #6D28D9;
}

/* White outline on a dark photo overlay */
.hero-text {
    text-shadow: 1px 1px 2px black;
}
Don't overdo it. Heavy text shadows hurt readability. Use them for headings or short labels over busy backgrounds, never for body text.

📐 Box Sizing

The box-sizing property changes how the browser calculates an element's total width and height. By default, the width you set on an element only applies to the content area. Padding and border get added on top, making the element bigger than expected.

The Default Behavior is Confusing

Suppose you set width: 200px, padding: 20px, and border: 2px solid. Most people expect the element to be 200 pixels wide. The default behavior makes it 244 pixels wide (200 content + 40 padding + 4 border). That's why box-sizing exists.

📦 content-box (default)

Width refers to the content only. Padding and border are added on top.

200px content

Actual element width: 244px

✅ border-box

Width refers to the entire element. Padding and border are included.

~156px content

Actual element width: 200px

The Universal box-sizing Reset

Almost every modern site applies border-box sizing globally. The result is widths that always behave the way you'd expect them to.

The Standard Modern Reset

/* Apply border-box to every element */
* {
    box-sizing: border-box;
}
Set this once, forget about it. Putting box-sizing: border-box on every element at the top of your stylesheet eliminates an entire category of layout bugs. Writing width: 100% always means the element fills its container, no matter what padding or border it has.

🎯 Structural Pseudo-Classes

Earlier you learned dynamic pseudo-classes (:hover, :active, etc.). HTML 5 also makes heavy use of structural pseudo-classes, which target elements based on their position among siblings.

Common Structural Pseudo-Classes

Pseudo-Class Matches
:first-child The first child of its parent.
:last-child The last child of its parent.
:nth-child(n) The nth child of its parent (counting all children).
:nth-of-type(n) The nth child of its type (counting only matching elements).
:first-of-type The first element of its type within its parent.
:last-of-type The last element of its type within its parent.

nth-of-type vs nth-child

🔢 :nth-child(n)

Counts every sibling, regardless of element type. p:nth-child(2) matches a paragraph only if it happens to be the second child of its parent.

🎯 :nth-of-type(n)

Counts only siblings of the same type. p:nth-of-type(2) matches the second paragraph among the parent's children, regardless of what other elements are there.

Striped Article Backgrounds

/* Every odd-numbered article gets a tinted background */
article:nth-of-type(odd) {
    background-color: #F5F3FF;
}

/* The third article gets special emphasis */
article:nth-of-type(3) {
    border-left: 4px solid #6D28D9;
}

🌐 Adding a Favicon

A favicon (short for "favorite icon") is a small image that appears in the browser tab next to a page's title. Favicons also appear in bookmarks, history lists, and the home-screen icons of mobile bookmarks.

P
Prairie Bakery

Why Use a Favicon

🏷️ Brand Recognition

A favicon makes your site instantly identifiable in a sea of open browser tabs.

📚 Bookmark Visibility

Bookmark menus show only the favicon and a short title. A distinctive favicon helps visitors find their way back.

📱 Home Screen Icon

When a visitor saves your site to their phone home screen, the favicon (or a larger version of it) becomes the app-style icon.

✨ Professional Polish

Sites without favicons look unfinished. The blank tab icon signals an amateur build.

Favicon File Formats

Format Notes
.png Most common. Supports transparency. Pick 32 by 32 or 64 by 64 pixels for crisp display.
.gif Older format. Smaller file size but limited colors and lower quality.
.ico The classic favicon format. Can contain multiple sizes in one file.
.svg Scales perfectly at any size. Supported by modern browsers.

Adding a Favicon to a Site

The favicon is added with a <link> element in the <head>. Many sites also place a copy of the favicon file in the root folder so browsers can find it automatically.

The Favicon Link

<head>
    <meta charset="UTF-8">
    <title>Prairie Bakery</title>
    <link rel="icon"
          type="image/png"
          href="images/favicon.png">
</head>
Make your own favicon. Create a square image in any photo editor, simplify it down to a recognizable icon, and export it as PNG at 32 by 32 or 64 by 64 pixels. Several free online tools also generate favicon files in multiple sizes from a single source image.

✅ Validating the Improved Site

After adding new semantic elements, grid layouts, and visual effects, validate every page. Each new feature is another opportunity for a typo, an unclosed tag, or a duplicate id.

  • HTML files: validate at validator.w3.org.
  • CSS files: validate at jigsaw.w3.org/css-validator.
  • Mobile-friendliness: test with the search engine mobile-friendly tools.
  • Manual review: open every page in a real browser at multiple viewport sizes and click every link.

✨ Best Practices

Habits for modern page layouts

  • Reach for semantic elements first. Use article, aside, section, and figure wherever they fit. Save div for purely structural grouping.
  • Pair every figure with a figcaption when the visual needs explanation.
  • Use Grid for two-dimensional layouts. Float was for wrapping text around images, not laying out pages.
  • Apply * { box-sizing: border-box; } globally. It eliminates a whole class of width-calculation surprises.
  • Use opacity for fades, alpha colors for backgrounds. Don't put opacity on a parent if you want the children to stay opaque.
  • Keep shadows subtle. A small offset, a soft blur, and a low-opacity color outperform heavy black drops.
  • Don't apply text-shadow to body copy. It hurts readability. Reserve it for headings or labels.
  • Add a favicon to every site. A blank tab icon makes a site look unfinished.
  • Validate after major changes. Catching errors in the validator beats hunting them later.
